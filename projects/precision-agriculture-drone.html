Precision Agriculture Drone

Integrate GPS waypoint navigation and a NoIR camera with 850nm bandpass filter on a Raspberry Pi companion computer to generate NDVI crop-health maps from aerial survey footage — a real-world precision farming tool using drone-based remote sensing.

📖 Introduction

NDVI (Normalised Difference Vegetation Index) measures plant health using the difference in reflectance between near-infrared (NIR) and visible red light: NDVI = (NIR − R) / (NIR + R). Healthy, chlorophyll-rich plants reflect strongly in NIR and absorb red light (NDVI 0.6–0.9). Stressed or sparse vegetation scores 0.2–0.5. A Raspberry Pi NoIR camera with an 850nm bandpass filter (allowing NIR through but blocking visible) on a drone captures geotagged images along a grid survey flight path. These are mosaicked into a field NDVI map that reveals stressed crop zones requiring targeted intervention.

🧩 Components Required

  • 🚁
    F450 Quadcopter Frame + Pixhawk — F450 frame; Pixhawk 2.4.8 flight controller with ArduCopter firmware; GPS module; telemetry radio 915MHz
  • 📷
    Pi NoIR Camera v2 + 850nm Filter — IR pass filter blocks red; NoIR removes IR-cut glass; captures NIR in RED channel; mounted nadir (downward)
  • 🧠
    Raspberry Pi Zero 2W (Companion) — Connects to Pixhawk via UART MAVLink; controls survey mode; captures geotagged JPEG every 2s
  • 📡
    NEO-M8N GPS Module — 10Hz update; compass; plugs into Pixhawk; ±2m accuracy; good enough for 5m overlap grid survey
  • 🔋
    4S 5000mAh LiPo Battery — ~20 min flight; 14.8V; 50A peak discharge; power Pi companion via BEC 5V/3A from distribution board

⚙️ How It Works

  1. 1

    MAVLink Waypoint Upload

    Plan survey grid with Mission Planner: survey area, 30m altitude, 70% front/side overlap, grid pattern auto-generated. Upload to Pixhawk as waypoint mission. Python companion (DroneKit): from dronekit import connect; vehicle = connect('/dev/ttyS0', baud=57600). Monitor mission: vehicle.mode.name == 'AUTO'. Read GPS position every 1s: lat = vehicle.location.global_frame.lat. Trigger camera at each survey grid point using MAVLink DO_DIGICAM_CONTROL command.

  2. 2

    Geotagged NIR Image Capture

    Picamera2 captures downward-facing JPEG at each waypoint: camera.capture_file(f'nir_{lat:.6f}_{lon:.6f}.jpg'). Embed GPS EXIF into JPEG with piexif: exif_ifd = {piexif.GPSIFD.GPSLatitude: to_deg_rational(lat), ...}. Camera settings: fixed shutter 1/1000s (avoid motion blur), ISO 100, AWB locked. 850nm bandpass filter + NoIR sensor = the "red channel" of the Bayer sensor captures NIR reflectance.

  3. 3

    NDVI Computation

    Post-processing on laptop (or Pi 4): load each image with OpenCV, split channels: B, G, R = cv2.split(img_float). NIR ≈ R channel (850nm captured in Red Bayer cells). 530nm visible green ≈ B channel (post-filter blue). Compute: ndvi = (R - B) / (R + B + 1e-5). Clip to [-1,1]. Apply colour map: ndvi_colour = cm.RdYlGn((ndvi+1)/2). Values <0.4 flagged as stressed zones. Save georeferenced GeoTIFF with rasterio.

  4. 4

    Mosaic + Field Map

    Stitch geotagged images into orthomosaic with OpenDroneMap (ODM) or OpenCV feature matching. ODM command: docker run opendronemap/odm --project-path /odm_photos. Output: georeferenced orthomosaic GeoTIFF + 3D point cloud. Compute zonal statistics: average NDVI per 5×5m cell. Overlay on field boundary polygon (uploaded as GeoJSON). Identify and GPS-mark stress zones for targeted fertiliser application. Export as PDF farm report or Google My Maps KML.

💻 Core Code Snippet

import cv2, numpy as np
from dronekit import connect, VehicleMode
from picamera2 import Picamera2
import piexif, time

# Connect to Pixhawk via serial MAVLink
vehicle = connect('/dev/ttyS0', baud=57600, wait_ready=True)
cam = Picamera2()
cam.configure(cam.create_still_configuration(
    main={"size": (3280,2464), "format": "BGR888"}))
cam.start()

def capture_ndvi_image(lat, lon, idx):
    arr = cam.capture_array()
    # Embed GPS EXIF
    exif_bytes = build_gps_exif(lat, lon)
    cv2.imwrite(f'nir_{idx:04d}_{lat:.5f}_{lon:.5f}.jpg',
                arr, [cv2.IMWRITE_JPEG_QUALITY,95])

# NDVI computation (post-flight)
def compute_ndvi(img_path):
    img = cv2.imread(img_path).astype(np.float32)
    B, G, R = cv2.split(img)  # NIR≈R channel with 850nm filter
    ndvi = (R - B) / (R + B + 1e-5)
    ndvi = np.clip(ndvi, -1, 1)
    ndvi_norm = ((ndvi + 1) / 2 * 255).astype(np.uint8)
    ndvi_colour = cv2.applyColorMap(ndvi_norm, cv2.COLORMAP_RdYlGn)
    return ndvi, ndvi_colour

# Monitor mission and capture at waypoints
prev_wp = -1
while vehicle.mode.name == 'AUTO':
    wp = vehicle.commands.next
    if wp != prev_wp:
        lat = vehicle.location.global_frame.lat
        lon = vehicle.location.global_frame.lon
        capture_ndvi_image(lat, lon, wp)
        prev_wp = wp
    time.sleep(0.5)
