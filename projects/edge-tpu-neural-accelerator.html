Edge TPU Neural Accelerator Demo

Deploy INT8-fully-quantised TFLite models — MobileNetV2 image classification and SSD MobileNet object detection — onto a Google Coral USB Accelerator connected to Raspberry Pi 4. Benchmark inference latency, power efficiency, and accuracy vs. CPU, demonstrating how dedicated neural hardware transforms edge AI deployment.

📖 Introduction

The Google Coral USB Accelerator contains a dedicated Edge TPU ASIC capable of 4 TOPS (tera-operations per second) while consuming only 2W. For TFLite INT8-quantised models where all ops are TPU-compatible, it delivers 30–100× speedup over the Pi 4 CPU. This project is a structured benchmark + live demo: three models (MobileNetV2 classification, SSD-MobileNet detection, DeepLabV3 segmentation), measured on 1000 frames each, producing a definitive comparison table. Then live Pi Camera demo showing real-time 300fps classification — impossible on CPU alone.

🧩 Components Required

  • 🧠
    Google Coral USB Accelerator — Edge TPU 4 TOPS; USB 3.0 (use USB 3 port for full speed!); plug-and-play with pycoral library; <2W
  • 🧩
    Raspberry Pi 4 (4 GB) — USB 3.0 host; pycoral + tflite-runtime; picamera2; benchmark harness; matplotlib result plots
  • 📷
    Pi Camera Module v2 — Live inference demo input; 640×480 frames; compare CPU vs TPU inference pipeline visually
  • 📊
    USB Power Meter (FNIRSI or UM25C) — Measure Coral USB current draw during inference; calculate TOPS/Watt efficiency vs CPU

⚙️ How It Works

  1. 1

    Model Preparation: Full INT8 Quantisation

    The Edge TPU requires ALL operations to be INT8. Standard TFLite partial quantisation won't work. Post-training INT8 quantisation with representative dataset: converter.optimizations=[tf.lite.Optimize.DEFAULT]; converter.representative_dataset=representative_data_gen; converter.target_spec.supported_ops=[tf.lite.OpsSet.TFLITE_BUILTINS_INT8]; converter.inference_input_type=tf.uint8; converter.inference_output_type=tf.uint8. Compile for Edge TPU: edgetpu_compiler model_quant.tflite. Output: model_quant_edgetpu.tflite.

  2. 2

    pycoral Inference Setup

    Install: pip install pycoral. Detect Coral: from pycoral.utils.edgetpu import make_interpreter; interp = make_interpreter('model_edgetpu.tflite'). For CPU: import tflite_runtime.interpreter as tflite; cpu_interp = tflite.Interpreter('model_quant.tflite'). Both APIs are compatible. Warm-up: run 10 inference passes before benchmarking to avoid USB initialisation latency in timing.

  3. 3

    Structured Benchmark Harness

    For each model × hardware: run 1000 inference passes on pre-loaded batch of test images. Record latency: t0=time.perf_counter(); interp.invoke(); latency=(time.perf_counter()-t0)*1000. Compute: mean, p50, p95, p99 latency, throughput (FPS). Power measurement via USB meter read via serial. Plot: grouped bar chart of latency + separate scatter plot of accuracy vs. latency. Export results to CSV + Markdown table for project poster.

  4. 4

    Live Camera Inference Demo

    Run SSD-MobileNet on Coral: real-time bounding box detection at ~300fps. Draw detections: for obj in objs: cv2.rectangle(frame, obj.bbox[:2], obj.bbox[2:], (0,255,200), 2); cv2.putText(frame, f"{labels[obj.id]} {obj.score:.0%}", ...). Display CPU vs TPU side-by-side on HDMI (split-screen): left panel CPU at 8fps, right panel Coral at 60+fps. Dramatically demonstrates the value of dedicated ML hardware for edge deployment.

💻 Core Code Snippet

import time, numpy as np
from pycoral.utils.edgetpu import make_interpreter
from pycoral.adapters import common, classify
import tflite_runtime.interpreter as tflite

# Load models
tpu = make_interpreter('mobilenet_v2_1.0_224_quant_edgetpu.tflite')
cpu = tflite.Interpreter('mobilenet_v2_1.0_224_quant.tflite')
tpu.allocate_tensors(); cpu.allocate_tensors()

def benchmark(interp, images, n=1000):
    latencies = []
    for i in range(n):
        img = images[i % len(images)]
        common.set_input(interp, img)
        t0 = time.perf_counter()
        interp.invoke()
        latencies.append((time.perf_counter()-t0)*1000)
    return {
        'mean_ms': np.mean(latencies),
        'p95_ms':  np.percentile(latencies, 95),
        'fps':     1000 / np.mean(latencies)
    }

images = [np.random.randint(0,255,(224,224,3),dtype=np.uint8) for _ in range(100)]
cpu_stats = benchmark(cpu, images)
tpu_stats = benchmark(tpu, images)
speedup = cpu_stats['mean_ms'] / tpu_stats['mean_ms']
print(f"CPU: {cpu_stats['mean_ms']:.1f}ms | TPU: {tpu_stats['mean_ms']:.2f}ms | Speedup: {speedup:.0f}x")
