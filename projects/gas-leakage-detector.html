Gas Leakage Detector

An MQ-2 sensor detects LPG, methane, and smoke concentrations above a safe threshold and triggers a loud buzzer — a life-saving real-world sensor application.

📖 Introduction

The MQ-2 semiconductor gas sensor uses a heated tin dioxide (SnO₂) element whose electrical resistance drops when reducing gases (LPG, methane, smoke, hydrogen) are present. The module converts this resistance change into an analog voltage (0–5 V) and a digital threshold output with an onboard potentiometer you can tune.

MQ sensors require a 20–30 second warm-up time after power-on before readings stabilise — always account for this in your sketch. Calibrating in fresh air gives you a baseline Rs/Ro ratio used to calculate ppm concentrations in advanced use-cases.

⚙️ How It Works

  1. 1
    Wire and warm up — MQ-2: VCC → 5 V, GND → GND, AOUT → A0, DOUT → D2. Buzzer → D8, red LED → D7, green LED → D6. In setup() add delay(30000); or a countdown on Serial Monitor — wait for the sensor to stabilise. The heater element draws ~150 mA so use the Vin pin with a suitable adapter.
  2. 2
    Set danger threshold — In clean air, analogRead(A0) typically returns 100–200. Hold a lit lighter near the sensor (carefully — don't ignite the gas) and observe the value jump to 400–800+. Set const int GAS_THRESHOLD = 400;. Adjust with the onboard pot for digital D2 output as well.
  3. 3
    Alarm logic with levels — Read int gasVal = analogRead(A0);. Below 300: green LED on, safe. 300–500: green off, yellow warning (serial print). Above 500: red LED + buzzer alarm. Using tone(buzzer, 1500) creates a piercing 1.5 kHz siren. Call noTone(buzzer) to stop.
  4. 4
    Add I2C LCD status — Display "SAFE" in green-text context or "⚠ GAS DETECTED" with the raw ADC value on row 2. Updating the LCD at 500 ms intervals prevents flicker. Print gas percentage: int pct = map(gasVal, 0, 1023, 0, 100); for a human-readable gauge.
