Motion-Activated Night Light

A PIR sensor wakes your LEDs the moment someone enters a dark room — the LDR ensures lights only fire at night — then they auto-shut after 10 seconds.

Dark + motion detected → LEDs on → 10 s auto-off

▶ Watch on YouTube

📖 Introduction

This project combines two sensing techniques: a PIR (Passive Infrared) sensor that detects changes in the IR heat signature of moving objects, and an LDR that measures ambient light level. By requiring BOTH conditions — darkness AND motion — you build the same energy-saving logic used in hallway lights, bathroom sensors, and outdoor floodlights worldwide.

The key coding skill is the non-blocking timer using millis(). Using delay(10000) would freeze your entire sketch. Instead, record when the motion last happened with lastMotionTime = millis(), then check if (millis() - lastMotionTime > 10000) turnOff(); in every loop iteration — execution never stops.

⚙️ How It Works

  1. 1
    Wire the sensors — HC-SR501: VCC → 5 V, GND → GND, OUT → D2. Turn the sensitivity and time-delay potentiometers fully counterclockwise (minimum). LDR voltage divider: 5 V → 10 kΩ → A0 → LDR → GND. LEDs in parallel from D9 via 220 Ω to GND.
  2. 2
    Check light level — int ldrVal = analogRead(A0);. LDR reads low (~50–150) in a bright room and high (~750–950) in darkness. Set const int DARK_THRESHOLD = 500;. Only proceed to check PIR if ldrVal > DARK_THRESHOLD.
  3. 3
    Motion trigger — if (digitalRead(PIR_PIN) == HIGH) { lastMotionTime = millis(); if (!lightsOn) { digitalWrite(LED_PIN, HIGH); lightsOn = true; } }. Each new motion resets the 10-second timer.
  4. 4
    Auto-off timer — In every loop() pass: if (lightsOn && (millis() - lastMotionTime > 10000UL)) { digitalWrite(LED_PIN, LOW); lightsOn = false; }. Because no delay() is used, the PIR continues to be polled every loop cycle — motion during the countdown resets the timer seamlessly.
