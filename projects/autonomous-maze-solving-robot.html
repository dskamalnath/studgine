Autonomous Maze-Solving Robot

Build a robot that maps and solves a maze autonomously — using the flood-fill algorithm with IR wall sensors for maze discovery and encoder odometry for precise cell-to-cell movement on Raspberry Pi.

📖 Introduction

The micromouse / maze-solving robot is a classic robotics competition challenge. This implementation uses the flood-fill algorithm: think of the goal cell as a bowl of water — water (distances) flood outward, and the robot always moves to the neighbour with the smallest flood value. As the robot discovers walls, it updates the flood values and changes direction. This guarantees an optimal path once the whole maze is explored.

You'll learn cell-based maze representation (2D array of wall bitmasks), encoder odometry for accurate forward/turn movements, and PID-corrected straight-line driving. The split between exploration (first run) and speed run (second run taking the known optimal path) is a key algorithm design concept.

🧩 Components Required

  • 🧠
    Raspberry Pi Zero 2W — Small form factor fits maze robot; runs Python flood-fill + motor PID; pigpio for encoder interrupts
  • 👁️
    3× IR Wall Sensors (GP2Y0A21) — Left wall / front wall / right wall detection; analog 10–80cm range; sample 5 times for noise rejection
  • ⚙️
    2× N20 Motors with Encoders — Quadrature encoders; count clicks for distance and turn angle; essential for cell-accurate navigation
  • 🔌
    TB6612FNG Motor Driver — 1.2A continuous per channel; logic-level PWM; thermal protection for continuous maze running
  • 📐
    MPU-6050 IMU (optional) — Gyro-assisted turns: verify 90° turn accuracy using Z-axis gyro, especially on slippery surfaces

⚙️ How It Works

  1. 1

    Maze Representation

    Represent a 16×16 maze (standard micromouse) as two arrays: walls[16][16] (bitmask: NESW = bits 3,2,1,0) and flood[16][16] (distance values). Initialise: outer boundary walls set, interior unknown. Flood-fill initialisation: set flood[goal] = 0, BFS outward: for each cell with value n, unblocked neighbours get n+1. All cells start with flood value based on Manhattan distance to goal (heuristic for first move).

  2. 2

    Flood-Fill Update Algorithm

    Each time robot moves to a new cell and detects walls via IR sensors: update walls[x][y] bitmask. Check if current flood values are consistent with known walls: for each cell, if its flood value is not 1 + min(accessible neighbours), it's inconsistent. Rerun BFS from goal to recompute all flood values. Move direction: choose the accessible neighbour with the lowest flood value. If tied, prefer the direction that hasn't been visited.

  3. 3

    Precise Cell Navigation with Odometry

    Each maze cell is 180mm × 180mm (standard). Forward: drive until encoder counts = cells × counts_per_180mm. PID straight-line correction: use encoder difference between left/right wheels to correct drift. Turn 90°: spin one wheel forward, one backward until encoder counts = turn_counts (calibrated empirically). Wall-following correction: if left wall detected at <30mm during forward drive, gently steer right. Combine encoder odometry (primary) with gyro heading (secondary crosscheck).

  4. 4

    Exploration to Speed Run

    Exploration phase: robot explores maze using flood-fill, building complete wall map. Terminates when flood value at start position after reaching goal equals steps taken (optimal path confirmed). Speed run: use recorded optimal path; run at maximum speed the motors can sustain while still reliably executing turns. Speed run uses pre-compiled turn list: [F, F, R, F, L, L, F...]. No IR sensing needed during speed run (path is known); just encoder-based dead reckoning.
