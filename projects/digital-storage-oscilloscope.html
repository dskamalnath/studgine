Digital Storage Oscilloscope

📖 Introduction

A digital storage oscilloscope (DSO) captures, stores, and displays voltage waveforms. This project uses the MCP3208 — a 12-bit, 8-channel, SPI ADC that can sample at up to 100 kSPS — connected to a Raspberry Pi. The captured samples are displayed in real time using pygame on an HDMI monitor. Key oscilloscope features — timebase, voltage scaling, edge triggering — are all software-implemented.

This project teaches ADC interfacing, ring-buffer triggering algorithms, aliasing, and signal conditioning. A ×10 probe divider circuit and input protection (1MΩ + Zener clamp) make it somewhat safe for real circuit debugging up to ±15V inputs.

🧩 Components Required

  • 🔋
    MCP3208 12-bit SPI ADC — 8 channels; 100 kSPS max; 12-bit resolution; SPI 0–3.3V input range; PDIP-16 package
  • 🧠
    Raspberry Pi 4 or 5 — SPI0 port (MOSI/MISO/SCLK/CE0); pigpio SPI at 1 MHz for reliable capture; Pi 5 recommended for higher sample rate
  • 🖥️
    7" HDMI Touchscreen (1024×600) — pygame renders waveform + UI controls; touchscreen for timebase/trigger adjustment via GUI buttons
  • 🛡️
    Input Protection Circuit — 1MΩ input resistor + BZX55C3V3 Zener clamp + 100Ω series; limits input to 0–3.3V for ADC safety
  • 📉
    ×10 Voltage Divider — 9MΩ/1MΩ divider extends input range to 33V; matched to 1MΩ input impedance

⚙️ How It Works

  1. 1

    High-Speed SPI Sampling

    Use pigpio SPI: h = pi.spi_open(0, 1000000, 0). Read MCP3208 channel 0: send 3 bytes [0x06, 0x00, 0x00], read 3 bytes back, extract 12-bit value: adc = ((buf[1]&0x0F)<<8) | buf[2]. Convert to voltage: voltage = adc * 3.3 / 4096. Fill a 512-sample ring buffer at target sample rate. Use time.perf_counter() for precise inter-sample delays. Actual achievable rate: ~50 kSPS on Pi 4 (limited by SPI overhead + Python GIL).

  2. 2

    Edge Triggering Algorithm

    Sweep through ring buffer to find trigger point: scan for first sample where sample[i] >= trigger_level and sample[i-1] < trigger_level (rising edge). Start display from that index. This stabilises the waveform on screen — without triggering, the display would scroll randomly. Falling edge trigger: swap >/<. Auto trigger: if no edge found within 100ms, display anyway (prevents freeze on DC signals).

  3. 3

    pygame Waveform Rendering

    Map each sample to screen coordinates: x = i * (screen_w/num_samples), y = screen_h - (voltage - v_offset) * (screen_h / v_range). Draw with pygame.draw.lines(screen, GREEN, False, points). Draw grid: 8×10 division graticule. Render measurements: frequency (via zero-crossing count), Vmax, Vmin, Vpp. Update at 30 FPS display rate (decouple from sample rate). Add touchscreen buttons for timebase: 0.1ms/div, 0.2ms, 0.5ms, 1ms, 2ms, 5ms.

  4. 4

    Measurements + FFT

    Auto-measure: frequency = 1 / (average period of zero crossings). Duty cycle = time above 50% range / period. RMS = sqrt(mean(samples^2)). FFT: numpy.fft.rfft(samples) → plot frequency spectrum in a second window. This turns the DSO into a spectrum analyser. Useful for seeing harmonics of PWM signals, switching noise, and audio frequencies. Label peak frequency on spectrum display.

💻 Core Code Snippet

import pigpio, time, numpy as np, pygame

pi = pigpio.pi()
h = pi.spi_open(0, 1_000_000, 0)  # SPI0, 1MHz

def read_mcp3208(ch):
    cmd = [0x06|(ch>>2), (ch&3)<<6, 0]
    c, data = pi.spi_xfer(h, cmd)
    return ((data[1]&0x0F)<<8)|data[2]  # 12-bit result

SAMPLES = 512
buf = [0.0]*SAMPLES
TRIGGER_LEVEL = 1.65  # 3.3V midpoint

def capture(samples_per_sec=50000):
    dt = 1.0/samples_per_sec
    for i in range(SAMPLES):
        t = time.perf_counter()
        buf[i] = read_mcp3208(0) * 3.3/4096
        while time.perf_counter()-t < dt: pass

def find_trigger():
    for i in range(1, SAMPLES-1):
        if buf[i-1] < TRIGGER_LEVEL <= buf[i]:
            return i
    return 0

# pygame render
pygame.init(); screen = pygame.display.set_mode((800,480))
W, H = 800, 400; V_RANGE = 3.3

while True:
    capture()
    t_idx = find_trigger()
    points = [(int(i*(W/SAMPLES)), int(H-(buf[(t_idx+i)%SAMPLES]/V_RANGE)*H))
              for i in range(min(SAMPLES, SAMPLES-t_idx))]
    screen.fill((0,20,0))
    if len(points)>1: pygame.draw.lines(screen,(0,255,64),False,points,2)
    pygame.display.flip()
