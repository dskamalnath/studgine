Electronic Coin Toss

Press a button for a random heads-or-tails result shown on two LEDs. Introduces randomSeed(), the random() function, and seed entropy from an unconnected analog pin.

📖 Introduction

Arduino's random() function generates pseudo-random numbers — determined by an algorithm that produces a sequence that appears random but repeats if you start from the same seed. Without seeding, the sequence starts identically every power cycle, making results predictable. The fix is randomSeed() — feeding it an unpredictable starting value.

The classic technique: randomSeed(analogRead(A0)); — an unconnected analog pin floats at a noisy, indeterminate voltage, giving a different seed each boot. This is a real-world application of entropy harvesting — the same concept used in cryptography libraries to seed secure RNGs.

⚙️ How It Works

  1. 1
    Seed the RNG — In setup(): randomSeed(analogRead(A0)); — A0 is left unconnected. On some boards you can XOR multiple reads: randomSeed(analogRead(A0) ^ analogRead(A1) ^ millis()); for better entropy. This ensures each power-on sequence is unique.
  2. 2
    Detect button press with debounce — Button → D2 with 10 kΩ pull-down. On rising edge (after 50 ms debounce): call the flip function. Use INPUT mode (not PULLUP) with external resistor to clearly distinguish HIGH/LOW on a 5 V rail.
  3. 3
    Generate and display result — int result = random(0, 2); (returns 0 or 1). If 0: digitalWrite(LED_HEADS, HIGH); digitalWrite(LED_TAILS, LOW);. If 1: opposite. Add a 500 ms "spinning" animation by rapidly blinking both LEDs alternately 10 times before settling on the result, creating a slot-machine effect.
  4. 4
    Track statistics — Maintain counters: int headsCount=0, tailsCount=0;. Display on Serial Monitor: Serial.print("H:"); Serial.print(headsCount); Serial.print(" T:"); Serial.println(tailsCount);. After 100 tosses you should see roughly 50/50 — verifying uniform distribution and giving you a live statistics lesson.
