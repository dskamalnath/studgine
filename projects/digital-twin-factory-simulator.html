Digital Twin Factory Simulator

Physical sensors on a miniature conveyor-belt model publish real-time state over MQTT to a Raspberry Pi that runs a Flask/WebSocket server. A browser-based Three.js 3D scene renders a digital twin that mirrors the physical system state in real time — a compelling Industry 4.0 / IoT demonstration.

📖 Introduction

Digital twins are virtual replicas of physical systems that update in real time from sensor data. Major manufacturers (Siemens, GE, BMW) use them to predict failures, optimise throughput, and train operators. This project creates a miniature demonstration: a small DC motor conveyor belt with IR item-counting sensors, a temperature sensor on the motor, and a Hall-effect speed encoder. Arduino collects sensor data and publishes to Mosquitto MQTT broker on Pi. A Node.js (or Python Flask-SocketIO) server consumes MQTT and pushes via WebSocket to a browser. Three.js renders a 3D animated digital twin. Total latency: <20ms physical-to-virtual.

🧩 Components Required

  • 🏭
    DC Motor Conveyor Belt Model — DIY rubber-band or timing belt on 3D-printed rollers; 12V DC motor + L298N driver; Hall encoder; ~40cm working length
  • 📡
    Arduino Uno + Sensor Suite — 2× IR break-beam sensors (item counting); LM35 temperature sensor; HC-SR04 for jam detection; UART to Pi
  • 🧠
    Raspberry Pi 4 + Mosquitto MQTT — Receives Arduino serial; publishes MQTT topics; Flask-SocketIO WebSocket bridge; hosts Three.js digital twin web app
  • 🖥️
    Browser (PC / Tablet) — Three.js 3D digital twin; live sensor dashboard; Chart.js live plots; WebSocket auto-reconnect; responsive design

⚙️ How It Works

  1. 1

    Physical Layer: Arduino Sensor Collection

    Arduino reads at 20Hz: encoder count (speed), IR item counters, temperature. Sends UART JSON: {"belt_rpm":85,"temp_c":38.4,"items_passed":12,"jam":false,"motor_pwm":210}. Python bridge on Pi: import serial, json, paho.mqtt.client as mqtt; data=json.loads(ser.readline()); client.publish("factory/conveyor", json.dumps(data)). MQTT topic hierarchy: factory/conveyor, factory/temperature, factory/items. Mosquitto broker: sudo apt install mosquitto; systemctl start mosquitto.

  2. 2

    Flask-SocketIO WebSocket Bridge

    Flask-SocketIO server subscribes to MQTT and emits to browser: @mqtt.on_message(); def handle_message(client, userdata, msg): socketio.emit('sensor_update', json.loads(msg.payload)). Serve static Three.js front-end: @app.route('/'); return render_template('twin.html'). Browser connects: const socket = io('http://pi:5000'); socket.on('sensor_update', updateTwin). Achieved latency MQTT→WebSocket→render: ~12ms typical.

  3. 3

    Three.js Digital Twin Rendering

    Three.js scene: BoxGeometry conveyor frame, CylinderGeometry rollers, animated belt texture (UV scroll). Belt animation: belt_texture.offset.x += data.belt_rpm/1000 * dt. Items: SphereGeometry spheres moving along belt path triggered by IR sensor events. Motor: colour changes with temperature (blue→green→red). Real-time Chart.js inset: temperature plot, item rate, motor PWM. Predictive alert: if temp > 50: showAlert("Motor overheat predicted").

  4. 4

    Predictive Analytics + Anomaly Detection

    Store last 100 measurements. Moving average of motor temperature rise rate: dT_dt = (temps[-1]-temps[-10]) / (10*dt). Linear prediction: predicted_overheat_time = (50-current_temp) / dT_dt. Show countdown in twin. Jam detection: belt speed drops >20% → highlight jam zone in red in 3D scene; alert. Items rate anomaly: Z-score >2.5 → flag defective sensor. Production OEE: (actual_items / target_items) * 100%. Dashboard: rate KPIs, uptime, efficiency score.

💻 Core Code Snippet

# Raspberry Pi Bridge: Arduino→MQTT→WebSocket
import serial, json, threading, time
import paho.mqtt.client as mqtt
from flask import Flask, render_template
from flask_socketio import SocketIO, emit

app = Flask(__name__); socketio = SocketIO(app, cors_allowed_origins='*')
mqtt_client = mqtt.Client()
serial_port = serial.Serial('/dev/ttyACM0', 9600)

def read_arduino():
    while True:
        line = serial_port.readline().decode().strip()
        try:
            data = json.loads(line)
            mqtt_client.publish('factory/state', json.dumps(data))
        except: pass

def on_mqtt_message(client, userdata, msg):
    data = json.loads(msg.payload)
    socketio.emit('sensor_update', data)

mqtt_client.on_message = on_mqtt_message
mqtt_client.connect('localhost', 1883)
mqtt_client.subscribe('factory/state')
mqtt_client.loop_start()

threading.Thread(target=read_arduino, daemon=True).start()

@app.route('/')
def index(): return render_template('twin.html')

if __name__ == '__main__':
    socketio.run(app, host='0.0.0.0', port=5000)
