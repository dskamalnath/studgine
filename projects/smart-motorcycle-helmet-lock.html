Smart Motorcycle Helmet Lock

An MQ-3 gas sensor tests rider breath alcohol level; if BAC exceeds the legal limit the ignition relay stays disabled. An RFID override allows emergency sober starts logged for accountability.

📖 Introduction

Two-wheeler fatalities involving alcohol are disproportionately high globally — motorcyclist reaction time impairment begins well below the legal BAC limit. This project embeds an MQ-3 alcohol sensor inside the helmet chin-bar vent or handlebar fairing, where the rider's outward breath naturally flows. The sensor reading gates a relay in series with the ignition wire.

The RFID override adds a crucial real-world reliability feature: in emergencies (sensor malfunction, medical need to move the vehicle), a registered RFID card unlocks the ignition and logs the override with a timestamp to a microSD for post-incident accountability. This combines Batch-6's alcohol interlock knowledge with Batch-1's RFID skills into a more sophisticated dual-authentication safety system.

🧩 Components Required

  • 🍷
    MQ-3 Alcohol Sensor — Mounted in helmet breath channel or near handlebar; 60s warm-up required; 5 V heater
  • 🔑
    RC522 RFID Module + Cards — SPI interface; reads registered card UID for emergency ignition override
  • 🧠
    Arduino Nano — Compact size for handlebar mounting; ADC for MQ-3; SPI for RFID; EEPROM for calibration
  • ⚡
    12V Relay Module — Motorcycle battery is 12V; relay normally open; interrupts ignition coil signal wire
  • 💾
    MicroSD SPI Module — Logs every override event: timestamp, duration, RFID UID — accountability trail
  • 📺
    OLED 0.96″ or 7-Seg Display — Shows BAC level and lock state on handlebar dashboard

⚙️ How It Works

  1. 1

    Helmet Sensor Integration

    The MQ-3 sensor is mounted in a 3D-printed bracket clipped to the helmet chin guard vent — close to the rider's mouth exhalation path. A small 5 V USB buck converter (from the motorcycle's 12V accessory line) powers the sensor and Arduino. During warm-up (60 s after key-on), the relay remains open and the OLED displays "WARMING UP... Xs." The rider cannot start the engine during this phase — this is intentional and prevents ignition before the sensor is calibrated to operating temperature.

  2. 2

    Breath Test Sequence

    After warm-up completes, the OLED prompts "BLOW 3 SECONDS." The rider blows into the vent — the firmware captures the peak ADC reading over a 3-second window. Three consecutive peak readings are averaged and converted to BAC% using the MQ-3 calibration curve (see alcohol-interlocked-ignition project for formula). If peak BAC < 0.05%: relay closes, green LED, OLED shows "START PERMITTED — 10 MIN." If >= 0.05%: relay stays open, red LED, buzzer 5 beeps, 20-minute lockout, OLED shows "LOCKED — BAC: X.XXX%".

  3. 3

    RFID Emergency Override

    A registered RFID card (UID stored in EEPROM) can bypass the lock. Place the card near the RC522 at any time (including during lockout). Firmware reads the 4-byte UID and compares to the stored override UID. Match: relay closes for 30 minutes; OLED shows "OVERRIDE ACTIVE — LOGGED." Mismatch: buzzer double-beep, "INVALID CARD." Every override event is written to microSD: override_log.csv row with millis(), uid_hex, bac_at_override. This provides the accountability trail that makes the override morally defensible — it can't be used silently.

  4. 4

    Auto-Relock While Riding

    Once permitted, the relay remains closed. The system reassesses every 30 minutes using a timer interrupt — if the engine is still running (detected via vibration sensor SW-420 or via the relay current sense), it samples the MQ-3 again in the background (passive sampling, no blow required). If a significantly elevated BAC is detected while riding (e.g., from ambient fumes incorrectly triggering the sensor), an audible warning is issued but the ignition is NOT cut mid-ride for safety — only locked on the next start attempt. This prevents sudden engine cut at highway speed.

💻 Core Code Snippet

#include <SPI.h>
#include <MFRC522.h>
#include <EEPROM.h>
#include <SD.h>

const int MQ3 = A0, RELAY = 7, BUZZ = 6;
const int RFID_SS = 10, RFID_RST = 9;
MFRC522 rfid(RFID_SS, RFID_RST);

byte storedUID[4]; float R0;
enum State { WARMUP, TESTING, SAFE, LOCKED } state = WARMUP;
unsigned long stateStart;

bool checkRFID() {
  if(!rfid.PICC_IsNewCardPresent() || !rfid.PICC_ReadCardSerial()) return false;
  for(int i=0;i<4;i++) if(rfid.uid.uidByte[i]!=storedUID[i]) return false;
  return true;
}

float readBACPeak() {
  long mx=0; unsigned long t=millis();
  while(millis()-t<3000) { long v=analogRead(MQ3); if(v>mx) mx=v; delay(10); }
  float Rs=(5.0-(mx/1023.0*5.0))/(mx/1023.0*5.0)*10.0;
  return 0.3934*pow(Rs/R0,-1.504)/2100.0;
}

void setup() {
  SPI.begin(); rfid.PCD_Init();
  pinMode(RELAY,OUTPUT); pinMode(BUZZ,OUTPUT);
  EEPROM.get(0,R0); EEPROM.get(10,storedUID);
  if(R0<0.1) R0=9.8;
  stateStart=millis();
}

void loop() {
  if(state==WARMUP && millis()-stateStart>60000) state=TESTING;
  if(state==TESTING) {
    float bac=readBACPeak();
    state = (bac<0.05)?SAFE:LOCKED;
    if(state==SAFE) digitalWrite(RELAY,HIGH);
  }
  if(state==LOCKED && checkRFID()) {
    digitalWrite(RELAY,HIGH); state=SAFE;
    // Log override to SD
    File f=SD.open("override.csv",FILE_WRITE);
    if(f){f.println(String(millis()));f.close();}
  }
}
