📖 Introduction

Non-Intrusive Load Monitoring (NILM) uses a single current sensor on the mains to disaggregate individual appliance consumption — no per-device sensors needed. This project clamps SCT-013 current transformers (CTs) on live 240V wires (safely, non-invasively) and feeds the AC current signal through a burden resistor + DC bias circuit into an MCP3208 ADC. A Raspberry Pi samples at 5 kHz, computes RMS current and real power at 50 Hz mains frequency, then classifies the combined load signature using a pre-trained Random Forest model.

Results are published via Paho MQTT to an InfluxDB time-series database ingested by Grafana for beautiful real-time energy dashboards.

🧩 Components Required

  • 🔌
    SCT-013 Non-Invasive Current Transformer (100A) — Clamp around single live wire; outputs 0–50mA AC proportional to load current; no mains wiring required
  • 🔬
    Signal Conditioning Board — 33Ω burden resistor + 1.65V DC bias (two 10kΩ) + 10µF coupling cap; scales CT output to 0–3.3V for ADC
  • 📡
    MCP3208 ADC + Raspberry Pi 4 — SPI at 500kHz; 12-bit voltage samples fed to emonLib-style RMS calculation in Python
  • 📊
    InfluxDB + Grafana + MQTT Broker — Mosquitto MQTT broker; Paho Python client; InfluxDB 2.0 time-series DB; Grafana 10 dashboard

⚙️ How It Works

  1. 1

    RMS Power Measurement

    Sample 1000 ADC readings per 50Hz cycle (5kHz sample rate) on MCP3208. Convert to current: I_rms = sqrt(mean((V_adc - V_bias)**2)) * CT_ratio / burden. Measure voltage via a plug-in AC-AC adapter (9V) as voltage reference: V_rms_scale = 230 * (measured_ratio). Real power = sum(V_inst * I_inst) / N over one cycle. Apparent power = V_rms * I_rms. Power factor = real/apparent.

  2. 2

    Appliance Signature Feature Extraction

    Extract features from each 1-second power window: [P_rms, Q_reactive, PF, THD_3rd_harmonic, P_delta_on, P_delta_off, P_steady_state]. FFT of current waveform reveals harmonics: np.fft.rfft(current_samples). Compute 3rd/5th/7th harmonic ratios — each appliance type has a unique harmonic signature (e.g., kettle = resistive → PF≈1; fridge motor = inductive → PF≈0.7; CFL lamp → high THD).

  3. 3

    ML Classifier Training + Inference

    Record labeled training data for each appliance: label events (appliance ON/OFF transition in raw power data). Build feature matrix. Train: sklearn.ensemble.RandomForestClassifier(n_estimators=100). Save with joblib. Runtime inference: extract features from current 1s window, call model.predict(features) → returns appliance label. Disaggregate total load: sum classified appliance powers, remainder = "Other".

  4. 4

    MQTT + Grafana Dashboard

    Publish JSON payload every 5s: mqtt.publish("home/energy", json.dumps({"total_w":2650, "kettle":2400, "fridge":80})). InfluxDB Telegraf MQTT consumer writes to energy measurement bucket. Grafana panels: time-series power graph, pie chart load breakdown, stat tiles (today's kWh, estimated cost at £0.24/kWh). Set thresholds → alert if daily cost exceeds budget.

💻 Core Code Snippet

import numpy as np, spidev, time, json
import paho.mqtt.client as mqtt
import joblib

spi = spidev.SpiDev(); spi.open(0,0); spi.max_speed_hz = 500_000
model = joblib.load('energy_model.pkl')
client = mqtt.Client(); client.connect("localhost", 1883)

def read_adc(ch):
    r = spi.xfer2([0x06|(ch>>2),(ch&3)<<6,0])
    return ((r[1]&0x0F)<<8)|r[2]

SAMPLES, RATE = 100, 5000
V_BIAS = 2048; CT_RATIO = 2000; BURDEN = 33

def measure_power():
    raw = [read_adc(0) for _ in range(SAMPLES)]
    centered = np.array(raw) - V_BIAS
    I_rms = np.sqrt(np.mean(centered**2)) * CT_RATIO/4096*3.3/BURDEN
    real_power = I_rms * 230  # simplified (no phase shift)
    harmonics = np.abs(np.fft.rfft(centered))
    thd = harmonics[3]/harmonics[1] if harmonics[1]>0 else 0
    return real_power, I_rms, thd

while True:
    P, I, THD = measure_power()
    features = [[P, I, P/(I*230+0.001), THD, 0, 0, P]]
    label = model.predict(features)[0]
    payload = json.dumps({"total_w": round(P), "appliance": label,
                          "pf": round(P/(I*230+0.001), 2)})
    client.publish("home/energy", payload)
    time.sleep(5)
