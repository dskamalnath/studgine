Push-Button Score Counter

A two-team scoreboard with increment, decrement, and reset push-buttons displayed on a 16×2 LCD. Perfect for sports events and mastering multi-button debouncing.

📖 Introduction

This project teaches the most fundamental interactive pattern in embedded systems: debounced button → state change → display update. Mechanical buttons "bounce" — a single press registers dozens of rapid on/off transitions in the first ~10 ms due to contact vibration. Without debouncing, your score would jump by 5–10 for each intended +1 press.

You'll implement both hardware debouncing (100 nF capacitor across the button) and software debouncing (millis() delay flag) to understand the difference and why both are sometimes used together in professional designs.

⚙️ How It Works

  1. 1
    Wire buttons and LCD — LCD SDA → A4, SCL → A5. Buttons: A+ → D4, A– → D5, B+ → D6, B– → D7, Reset → D8. Each button: one leg to 5 V, other leg to Arduino pin and via 10 kΩ to GND. Configure as INPUT (not INPUT_PULLUP) with external pull-downs.
  2. 2
    Software debounce — For each button track: bool lastState[5], lastDebounceTime[5];. On each loop: if pin state ≠ lastState AND millis() - lastDebounceTime[i] > 50: act on state change and update lastDebounceTime[i]. This 50 ms lockout window ignores bounce transitions.
  3. 3
    Score logic — Increment/decrement int scoreA, scoreB clamped at 0 min and 99 max. scoreA = constrain(scoreA + 1, 0, 99);. Reset sets both to 0. Refresh LCD only when a score changes: lcd.setCursor(0,0); lcd.print("A: "); lcd.print(scoreA); lcd.print(" B: "); lcd.print(scoreB);
  4. 4
    Win detection — Define a winning score: const int WIN_SCORE = 10;. When a team reaches it, display "TEAM A WINS!" on row 2, flash LCD backlight 3 times with a 500 ms tone, then hold the display until Reset is pressed.
