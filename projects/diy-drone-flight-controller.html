DIY Drone Flight Controller

Design and program a quadcopter flight controller from scratch using a Raspberry Pi, MPU-6050 6-DOF IMU, four ESCs, and BLDC motors. Implement cascaded PID attitude stabilisation with RC input decoding.

📖 Introduction

Building your own flight controller is one of the deepest embedded-systems challenges: you need accurate sensor fusion at 500+ Hz, precise PWM generation for ESCs (1000–2000 µs pulses), cascaded PID loops for roll/pitch/yaw, and RC receiver decoding — all running concurrently with hard real-time deadlines.

This project uses a Raspberry Pi running a bare-metal Python loop (not Linux desktop) for deterministic timing, with pigpio for hardware-timed PWM. You'll learn why commercial flight controllers like Betaflight use STM32 microcontrollers — and understand every design decision they made.

🧩 Components Required

  • 🧠
    Raspberry Pi Zero 2W or Pi 4 — Pi Zero 2W is lighter; Pi 4 has more CPU headroom; both use pigpio for PWM; disable Wi-Fi during flight
  • 📐
    MPU-6050 IMU — 500 Hz sample rate for flight; enable DLPF to reduce vibration noise; mount with vibration-dampening foam
  • ⚡
    4× 30A BLHeli ESCs — BLHeli_32 supports DSHOT protocol; accepts 1000–2000 µs PWM signal; calibrate all ESCs identically
  • 🌀
    4× 2205 2300KV BLDC Motors — 5" props; CCW/CW mixed for yaw reaction torque cancellation; X-frame layout
  • 🔋
    3S LiPo 1500mAh 60C — 11.1V; 5V BEC for Raspberry Pi; LiPo alarm buzzer for low-voltage cutoff protection
  • 📡
    FlySky FS-i6 RC Transmitter + Receiver — 6-channel PPM/PWM output; decode via GPIO interrupts; arming switch on channel 5

⚙️ How It Works

  1. 1

    Attitude Estimation — Madgwick Filter

    Use the Madgwick AHRS algorithm (more accurate than complementary filter for drones). Feed 6-DOF IMU data at 500 Hz. Output: roll, pitch, yaw as quaternion. Convert to Euler angles for PID. Mount IMU aligned with drone body frame. Calibrate gyro bias on startup: average 500 readings at rest. Reject readings during motor vibrations using a digital low-pass filter (DLPF cutoff = 42 Hz on MPU-6050 register 0x1A).

  2. 2

    Cascaded PID — Roll/Pitch/Yaw

    Three independent PIDs, one per axis. Roll inner: error = desired_roll − measured_roll → output = roll_correction. Pitch inner: same for pitch. Yaw: rate control only (target = 0°/s for hover). Motor mixing (X-frame): M1 = throttle − roll + pitch + yaw; M2 = throttle + roll + pitch − yaw; M3 = throttle + roll − pitch + yaw; M4 = throttle − roll − pitch − yaw. Clamp all motor outputs to 1100–1900 µs range.

  3. 3

    ESC PWM via pigpio

    ESCs expect 50 Hz PWM with 1000 µs (minimum throttle) to 2000 µs (full throttle) pulse width. Use pi.set_servo_pulsewidth(pin, usec). Arm procedure: send 1000 µs on all 4 channels for 2 seconds at startup. Safety: arm/disarm switch on RC channel 5; if RC signal lost (>100 ms no pulse) → immediately reduce all motors to 1000 µs. Never run motors indoors without props removed during development.

  4. 4

    RC Input Decoding

    PPM receiver: single GPIO pin with all channels multiplexed. Decode using pigpio callback: measure time between rising edges. Channel separation pulse >3 ms. Individual channel pulse 1000–2000 µs. Alternatively use SBUS serial (100 kbps inverted UART). Map channels: ch1=roll, ch2=pitch, ch3=throttle, ch4=yaw, ch5=arming. Low-pass filter RC inputs to prevent stick jitter from feeding PID.

💻 Core Code Snippet

import pigpio, smbus2, time, math

pi = pigpio.pi()
bus = smbus2.SMBus(1)
MPU = 0x68
bus.write_byte_data(MPU, 0x6B, 0)   # wake MPU
bus.write_byte_data(MPU, 0x1A, 0x03) # DLPF 42Hz

MOTORS = [12, 16, 20, 21]  # GPIO pins for 4 ESCs
for p in MOTORS: pi.set_servo_pulsewidth(p, 1000)  # arm

Kp_r, Ki_r, Kd_r = 1.2, 0.01, 0.05
int_r = int_p = 0; prev_r = prev_p = 0

def read_mpu():
    d = bus.read_i2c_block_data(MPU, 0x3B, 14)
    def s(hi,lo): v=(d[hi]<<8|d[lo]); return v-65536 if v>32767 else v
    ax,ay,az = s(0,1)/16384.0,s(2,3)/16384.0,s(4,5)/16384.0
    gx,gy = s(8,9)/131.0,s(10,11)/131.0
    return ax,ay,az,gx,gy

roll=pitch=0; dt=0.002
while True:
    t0=time.time()
    ax,ay,az,gx,gy = read_mpu()
    a_roll = math.degrees(math.atan2(ay,az))
    a_pitch = math.degrees(math.atan2(-ax,az))
    roll = 0.98*(roll+gx*dt)+0.02*a_roll
    pitch = 0.98*(pitch+gy*dt)+0.02*a_pitch

    # Roll PID
    er = 0 - roll
    int_r += er*dt
    roll_out = Kp_r*er + Ki_r*int_r + Kd_r*(er-prev_r)/dt
    prev_r = er

    thr = 1300  # base throttle µs
    m =[thr-roll_out, thr+roll_out, thr+roll_out, thr-roll_out]
    for i,p in enumerate(MOTORS):
        pi.set_servo_pulsewidth(p, max(1100,min(1900,int(m[i]))))
    time.sleep(max(0,dt-(time.time()-t0)))
