📖 Introduction

Degrees of Freedom (DOF) in robotics refer to the number of independent joints. A 5-DOF arm (base rotation + shoulder + elbow + wrist pitch + gripper) can reach most positions in a hemisphere. Inverse kinematics (IK) computes what joint angles produce a desired end-effector (gripper) position — the reverse of forward kinematics (given angles → compute position). For a 5-DOF arm with known link lengths L1–L4, closed-form geometric IK gives exact solutions much faster than numerical iterative methods.

This project uses 16-channel PCA9685 I²C servo controller to drive all 5 MG996R servos from a Raspberry Pi. A tkinter UI on the 7" touchscreen lets users specify (x,y,z) Cartesian target coordinates and watch the arm move there smoothly.

⚙️ How It Works

  1. 1

    PCA9685 Servo Control

    I²C address 0x40. Set PWM frequency: pca.frequency = 50. Set servo angle: pulse width = 1000µs + angle×(1000/180)µs. PWM counts: pulse = int((pw_us / 20000) * 4096). pca.channels[ch].duty_cycle = pulse. All 5 channels update simultaneously with no jitter. Power servos from separate 5V 3A supply via PCA power pins (not Raspberry Pi).

  2. 2

    Forward Kinematics (FK)

    Link lengths: L1=100mm shoulder, L2=90mm upper arm, L3=80mm forearm, L4=60mm wrist. Given joint angles [θ1,θ2,θ3,θ4], end-effector position: r = L2*cos(θ2) + L3*cos(θ2+θ3) + L4*cos(θ2+θ3+θ4), z = L1 + L2*sin(θ2) + L3*sin(θ2+θ3) + L4*sin(θ2+θ3+θ4), x = r*cos(θ1), y = r*sin(θ1). Implements using numpy trigonometry.

  3. 3

    Inverse Kinematics (IK) Solver

    Given target (x, y, z): θ1 = atan2(y, x) (base rotation). Wrist position: r = sqrt(x²+y²). For 3-link planar IK in (r,z): D = (r²+z²-L2²-L3²)/(2*L2*L3); θ3 = atan2(±sqrt(1-D²), D) (elbow up/down solution). θ2 = atan2(z,r) - atan2(L3*sin(θ3), L2+L3*cos(θ3)). θ4 controls wrist pitch (angle relative to horizontal). Clamp all joint angles to mechanical limits before sending.

  4. 4

    Trajectory Planning + Pick-and-Place

    Interpolate between current and target joint angles in N steps: theta_interp = np.linspace(theta_current, theta_target, steps=50). Execute at 20ms/step for smooth 1-second motion. Pick-and-place sequence: (1) move to above object, (2) lower Z, (3) close gripper, (4) raise Z, (5) move to drop zone, (6) lower, (7) open gripper. Save named positions (e.g. "HOME", "PICK_A") to JSON for teachable repetitive tasks.

💻 Core Code Snippet

import math, numpy as np
from adafruit_pca9685 import PCA9685
import board, busio

i2c = busio.I2C(board.SCL, board.SDA)
pca = PCA9685(i2c)
pca.frequency = 50

# Link lengths (mm)
L1, L2, L3, L4 = 100, 90, 80, 60

def set_servo(channel, deg):
    deg = max(0, min(180, deg))
    pulse_us = 1000 + deg*(1000/180)
    duty = int((pulse_us/20000) * 0xFFFF)
    pca.channels[channel].duty_cycle = duty

def inverse_kinematics(x, y, z):
    theta1 = math.degrees(math.atan2(y, x))
    r = math.sqrt(x**2 + y**2)
    # Wrist at end of link3 (simplified, no link4 wrist bend)
    D = (r**2 + z**2 - L2**2 - L3**2) / (2*L2*L3)
    if abs(D) > 1: return None  # unreachable
    theta3 = math.degrees(math.atan2(math.sqrt(1-D**2), D))
    theta2 = math.degrees(math.atan2(z, r) -
                           math.atan2(L3*math.sin(math.radians(theta3)),
                                      L2 + L3*math.cos(math.radians(theta3))))
    return theta1, theta2, theta3, 90.0, 90.0

# Move to target position
target = (120, 50, 80)  # mm
angles = inverse_kinematics(*target)
if angles:
    for ch, angle in enumerate(angles):
        set_servo(ch, angle + 90)  # offset to servo centre
