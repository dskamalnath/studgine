📖 Introduction

A bicycle speedometer is a classic beginner-to-intermediate project that teaches interrupt-driven programming, real-world physics (circumference-to-speed math), odometer accumulation, and display multiplexing — all on a simple 16×2 LCD. The principle is identical to commercial cycling computers: a small magnet on one spoke passes a Hall effect sensor mounted on the fork once per wheel revolution, generating a clean digital pulse.

The Arduino measures the time between pulses to calculate rotation period, then uses wheel circumference to determine speed in km/h. An ISR-driven volatile counter accumulates total revolutions to compute distance. Three buttons cycle through display pages: (1) current speed, (2) trip distance and odometer, (3) max speed, average speed, and trip time — giving you a complete cycling computer for under $10.

⚙️ How It Works

  1. 1

    Configure the Hall Sensor on an Interrupt Pin

    Connect A3144 signal to Arduino D2 (INT0). Call attachInterrupt(digitalPinToInterrupt(2), onPulse, FALLING). In the ISR, record pulseTime = millis() and increment volatile unsigned long revCount and set a volatile bool newPulse = true flag for the main loop.

  2. 2

    Calculate Speed from Pulse Period

    In the main loop, when newPulse is true, compute: period = pulseTime - lastPulseTime. Speed in km/h = (CIRCUMFERENCE_MM / period) × 3600.0 / 1000000.0 where CIRCUMFERENCE_MM = π × wheel_diameter_mm (typical 700C road ≈ 2098 mm, 26″ MTB ≈ 2090 mm). Set speed to 0 if no pulse within 3 seconds (bike stopped).

  3. 3

    Odometer and Trip Distance

    Each revolution increment: tripDist_m += CIRCUMFERENCE_MM / 1000.0. Total odometer stored in EEPROM as a float. Read on boot: EEPROM.get(0, odometerKm). Write back every 100 m change to avoid excessive EEPROM wear (300,000 write endurance). Speed averaging: running sum over last 10 readings with circular buffer.

  4. 4

    Button Mode Cycling and Display

    Three modes via displayMode 0/1/2: Mode 0 — "SPD: xx.x km/h" + spin bar. Mode 1 — "DST: xx.xx km" + "ODO: xxxx.x km". Mode 2 — "MAX: xx km/h" + "AVG: xx.x T:xx:xx". Long-press mode-button for 2 s resets trip (distance + max speed + timer). Backlight button toggles LCD backlight to save battery.

💻 Core Code Snippet

#include <LiquidCrystal_I2C.h>
#include <EEPROM.h>
LiquidCrystal_I2C lcd(0x27, 16, 2);

const float CIRC_MM = 2098.0; // 700C road tyre
volatile unsigned long lastPulse = 0, period = 0;
volatile unsigned long revCount = 0;
volatile bool newPulse = false;
float maxSpeed = 0, tripDist = 0, odometerKm = 0;
unsigned long tripStart = 0;
int displayMode = 0;

void onPulse() {
  unsigned long now = millis();
  period = now - lastPulse;
  lastPulse = now;
  revCount++;
  tripDist += CIRC_MM / 1000000.0; // convert mm to km
  newPulse = true;
}

void setup() {
  attachInterrupt(digitalPinToInterrupt(2), onPulse, FALLING);
  lcd.init(); lcd.backlight();
  EEPROM.get(0, odometerKm);
  tripStart = millis();
}

void loop() {
  float speed = 0;
  if (millis() - lastPulse > 3000) {
    speed = 0; // stopped
  } else if (period > 0) {
    speed = (CIRC_MM / (float)period) * 3600.0 / 1000.0; // km/h
  }
  if (speed > maxSpeed) maxSpeed = speed;

  switch(displayMode) {
    case 0:
      lcd.setCursor(0,0); lcd.print("SPD: " + String(speed, 1) + " km/h    ");
      lcd.setCursor(0,1); lcd.print("REV: " + String(revCount) + "        ");
      break;
    case 1:
      lcd.setCursor(0,0); lcd.print("TRIP: " + String(tripDist, 2) + " km    ");
      lcd.setCursor(0,1); lcd.print("ODO: " + String(odometerKm + tripDist, 1));
      break;
    case 2:
      lcd.setCursor(0,0); lcd.print("MAX:" + String(maxSpeed,1) + " km/h    ");
      unsigned long t = (millis()-tripStart)/1000;
      lcd.setCursor(0,1); lcd.print("T:" + String(t/3600) + "h" + String((t%3600)/60) + "m");
      break;
  }
  delay(200);
}
