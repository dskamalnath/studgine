Offline Voice Assistant

Build a fully private, offline voice assistant on Raspberry Pi 4 — Vosk STT transcribes speech, an Ollama-hosted local LLM generates responses, and pyttsx3 speaks them aloud. Zero cloud dependency.

📖 Introduction

Commercial voice assistants (Alexa, Google Assistant, Siri) send your audio to cloud servers for processing. This project builds an equivalent that runs entirely on-device: Vosk small English model (~50MB) for offline speech recognition, Ollama running phi3.5 mini (2.8GB quantised) for natural language understanding and response generation, and pyttsx3 for local TTS.

Raspberry Pi 4 with 4–8GB RAM can run phi3.5 at ~2–5 tokens/second — slow but functional for a voice assistant responding to short questions. You'll learn audio streaming pipelines, wake word detection, and local LLM serving via Ollama's REST API.

🧩 Components Required

  • 🧠
    Raspberry Pi 4 (8GB RAM) — 8GB strongly recommended to fit phi3.5 (2.8GB) + OS overhead; Pi 5 8GB gives better LLM token rate
  • 🎤
    USB Microphone (RODE NT-USB Mini or similar) — USB mic avoids Pi's noisy 3.5mm ADC; ReSpeaker 2-Mic HAT excellent for directional sensitivity
  • 🔊
    USB Speaker or Bluetooth Speaker — pyttsx3 uses espeak-ng backend; USB speaker preferred; Bluetooth adds ~0.5s latency for A2DP connect
  • 💾
    32GB+ MicroSD (or USB SSD) — Ollama models stored on microSD or USB SSD; USB SSD is 10× faster for model loading (2s vs 20s cold start)

⚙️ How It Works

  1. 1

    Wake Word Detection

    Use Porcupine (free tier) or openWakeWord for always-on wake word detection: pip install pvporcupine openwakeword. openWakeWord runs at ~5% CPU on Pi 4. Configure wake word "hey pi" or custom. On detection: play a tone (GPIO buzzer or audio beep) and start recording. This avoids running Vosk STT continuously (too slow). Record until 1.5s of silence detected (using WebRTC VAD): pip install webrtcvad.

  2. 2

    Vosk Speech Recognition

    Download Vosk small model: vosk-model-small-en-us-0.15 (~50MB). Load: model = vosk.Model("vosk-model-small-en-us-0.15"); rec = vosk.KaldiRecognizer(model, 16000). Feed audio bytes (16kHz, mono, 16-bit PCM): rec.AcceptWaveform(audio_bytes). Get result: json.loads(rec.Result())["text"]. Vosk transcribes at 4–8× real-time on Pi 4, so a 3-second utterance takes ~0.5s to transcribe. Fallback: use OpenAI Whisper tiny model for better accuracy (slower: ~3s transcription for 3s audio).

  3. 3

    Local LLM via Ollama

    Install Ollama on Pi: curl -fsSL https://ollama.com/install.sh | sh. Pull model: ollama pull phi3.5 (2.8GB). Ollama runs as background service on port 11434. Query from Python: import requests; r = requests.post("http://localhost:11434/api/generate", json={"model":"phi3.5","prompt":query,"stream":False}). Add system prompt: "You are a concise voice assistant. Reply in 1-2 sentences." Shorter responses = faster TTS playback. Alternatively use llama3.2:1b for faster responses (1B parameters, ~1GB).

  4. 4

    Text-to-Speech + Built-in Commands

    pyttsx3 with espeak-ng: engine = pyttsx3.init(); engine.setProperty('rate', 175); engine.say(response); engine.runAndWait(). Set voice: engine.setProperty('voice', 'english_rp') (British accent). Add built-in commands that bypass LLM for speed: "what time is it" → datetime.now().strftime("%H:%M"). "turn on the light" → GPIO action. These respond in <0.5s. LLM handles everything else. Log all interactions to JSON file for future fine-tuning.

💻 Core Code Snippet

import vosk, pyaudio, json, requests, pyttsx3, time

model = vosk.Model("vosk-model-small-en-us-0.15")
rec = vosk.KaldiRecognizer(model, 16000)
engine = pyttsx3.init(); engine.setProperty('rate', 165)

OLLAMA_URL = "http://localhost:11434/api/generate"
SYSTEM = "You are a concise voice assistant. Reply in 1-2 sentences."

def transcribe(audio_data):
    rec.AcceptWaveform(audio_data)
    return json.loads(rec.Result()).get("text", "")

def ask_llm(query):
    prompt = f"{SYSTEM}\n\nUser: {query}\nAssistant:"
    r = requests.post(OLLAMA_URL, json={"model":"phi3.5","prompt":prompt,"stream":False})
    return r.json().get("response","").strip()

def speak(text):
    engine.say(text); engine.runAndWait()

# Main loop: (simplified - full version adds wake word + VAD)
p = pyaudio.PyAudio()
stream = p.open(format=pyaudio.paInt16, channels=1, rate=16000, input=True, frames_per_buffer=8000)
print("Listening... say something:")
data = stream.read(32000, exception_on_overflow=False)  # 2 seconds of audio
text = transcribe(data)
if text:
    print(f"You said: {text}")
    speak("Let me think...")
    reply = ask_llm(text)
    print(f"Assistant: {reply}")
    speak(reply)
