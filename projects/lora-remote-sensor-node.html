LoRa Remote Sensor Node

📖 Introduction

LoRa (Long Range) uses a proprietary chirp-spread-spectrum (CSS) modulation that trades data rate for remarkable link budget. A 100 mW transmitter on 433 MHz can reach 2–3 km in open terrain, handling concrete walls and foliage far better than Wi-Fi or Bluetooth.

This project builds a two-node point-to-point link: a sensor node (Transmitter) collects DHT22 temperature/humidity data and broadcasts a compact packet every 10 seconds. A base station (Receiver) decodes the packet, prints readings to Serial, and drives an LCD. You'll learn packet framing, RSSI/SNR diagnostics, and LoRa module SPI configuration — the same concepts used in commercial LoRaWAN deployments.

🧩 Components Required

  • 🎛️
    Arduino Nano / Uno ×2 — One TX node, one RX gateway
  • 📡
    SX1278 LoRa Module (Ra-02) ×2 — 433 MHz, 3.3 V, SPI interface + IPEX antenna
  • 🌡️
    DHT22 Temperature/Humidity Sensor — On the TX sensor node
  • 📟
    16×2 I2C LCD — On the RX gateway to display readings
  • ⚡
    3.3 V LDO (AMS1117-3.3) — LoRa module is 3.3 V; use level-shifted SPI or LDO
  • 🔋
    Li-Po 3.7 V Battery + TP4056 — Powers TX node in the field

⚙️ How It Works

  1. 1

    Wire SPI + 3.3 V Level Shift

    Connect RA-02 NSS→D10, SCK→D13, MOSI→D11, MISO→D12, DIO0→D2. The module operates at 3.3 V — use a logic-level shifter or a resistor divider on MOSI/SCK/NSS lines. MISO can connect directly since 3.3 V HIGH is read correctly by Arduino 5 V logic.

  2. 2

    Configure LoRa Parameters

    Both nodes must use identical settings: LoRa.setFrequency(433E6), spreading factor SF7 (fast, moderate range), bandwidth 125 kHz, coding rate 4/5. Increase to SF10 for longer range at lower data rate. Set TX power with LoRa.setTxPower(17).

  3. 3

    Build and Send a Packet

    On the TX node, read DHT22 every 10 s, format a CSV string "T:28.5,H:61.2,V:3.8" (adding battery voltage). Call LoRa.beginPacket(), print the string, then LoRa.endPacket(). The radio transmits and returns to idle — great for battery-powered remote nodes.

  4. 4

    Receive, Parse, and Display

    The RX gateway polls LoRa.parsePacket() in loop(). When a packet arrives, read the full string, split on commas, extract T/H/V values, print to LCD and Serial. Log LoRa.packetRssi() and LoRa.packetSnr() for link quality diagnostics.

💻 Core Code Snippet

// === TRANSMITTER NODE ===
#include <SPI.h>
#include <LoRa.h>
#include <DHT.h>

#define LORA_NSS 10
#define LORA_RST  9
#define LORA_DIO0 2
DHT dht(5, DHT22);
unsigned int pktCount = 0;

void setup() {
  dht.begin();
  LoRa.setPins(LORA_NSS, LORA_RST, LORA_DIO0);
  LoRa.begin(433E6);
  LoRa.setSpreadingFactor(7);
  LoRa.setTxPower(17);
}

void loop() {
  float t = dht.readTemperature();
  float h = dht.readHumidity();
  float batt = analogRead(A0) * (3.3 / 1023.0) * 2;  // voltage divider
  // Build packet
  String payload = "T:" + String(t,1) +
                   ",H:" + String(h,1) +
                   ",V:" + String(batt,2) +
                   ",N:" + String(pktCount++);
  LoRa.beginPacket();
  LoRa.print(payload);
  LoRa.endPacket();
  delay(10000);  // every 10 s
}

// === RECEIVER / GATEWAY ===
void loop() {
  int pktSize = LoRa.parsePacket();
  if (pktSize) {
    String incoming = "";
    while (LoRa.available()) incoming += (char)LoRa.read();
    int rssi = LoRa.packetRssi();
    float snr  = LoRa.packetSnr();
    // Parse CSV
    int ti = incoming.indexOf("T:") + 2;
    float temp = incoming.substring(ti, incoming.indexOf(',', ti)).toFloat();
    Serial.println("Temp=" + String(temp) + "  RSSI=" + rssi + "  SNR=" + snr);
    lcd.clear(); lcd.print("T:" + String(temp,1));
    lcd.setCursor(0,1); lcd.print("RSSI:" + String(rssi) + "dBm");
  }
}
