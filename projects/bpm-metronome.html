BPM Metronome

Adjust BPM with a rotary encoder, display the value on an LCD, and click a buzzer on every beat. A musical tool that teaches encoder reading, integer arithmetic, and precise timing.

📖 Introduction

A metronome counts out a steady musical beat, defined in Beats Per Minute (BPM). At 120 BPM the interval between beats is 500 ms; at 60 BPM it's 1000 ms (1 second). The relationship is simple: interval (ms) = 60,000 / BPM.

The rotary encoder is a key concept here — it's a digital input that generates two out-of-phase pulse trains (CLK and DT) whose relative order encodes direction of rotation. Reading it correctly with state tracking is a fundamental skill that applies to menus, volume knobs, and motor position feedback in virtually every embedded design.

⚙️ How It Works

  1. 1
    Read the encoder — Wire CLK → D2, DT → D3, SW (push) → D4. Enable INPUT_PULLUP on all three. On each loop iteration, compare current CLK state to the last known state. If CLK changed from HIGH to LOW (falling edge) AND DT ≠ CLK, direction is clockwise (BPM++); if DT == CLK, counter-clockwise (BPM--). Clamp: bpm = constrain(bpm, 30, 240);
  2. 2
    Calculate beat interval — Whenever BPM changes: unsigned long beatInterval = 60000UL / bpm;. Using unsigned long prevents integer overflow for slow tempos. Update LCD: lcd.setCursor(0,0); lcd.print("BPM: "); lcd.print(bpm); lcd.print(" "); The trailing spaces clear leftover digits when BPM decreases.
  3. 3
    Non-blocking beat tick — Avoid delay() so the encoder stays responsive. Instead: if (millis() - lastBeat >= beatInterval) { lastBeat = millis(); tone(BUZZ, 1000, 30); }. This fires a 30 ms, 1 kHz click glued to its own timed check, leaving the rest of loop free to read the encoder every iteration.
  4. 4
    Tap-tempo with the encoder button — When SW is pressed, record millis(). On the second press, compute bpm = 60000 / (now - lastTap); and clamp. This lets the user tap in the desired tempo live, a feature found in professional drum machines and DJ equipment.
