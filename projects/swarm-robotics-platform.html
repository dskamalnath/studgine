Swarm Robotics Platform

📖 Introduction

Swarm robotics studies how simple individual robots following local rules can produce complex collective behaviour — like ant colonies or bird flocking. This project builds the minimum viable swarm: 3 Arduino Nano robots, each with an nRF24L01 2.4GHz radio, two DC motors, and IR sensors. Each robot broadcasts its position estimate and receives peers' positions every 20ms.

Behaviours implemented: leader-follower (two robots maintain a fixed offset behind a manual-controlled leader), triangle formation (robots self-organise into an equilateral triangle), and flocking (Reynolds rules: separation, alignment, cohesion). Emergent collective behaviour from simple rules is the core insight.

🧩 Components Required

  • 🧠
    3× Arduino Nano (or Nano 33 BLE) — One per robot; Nano 33 BLE adds built-in IMU for heading estimation
  • 📡
    3× nRF24L01+ with PA+LNA — 2.4GHz; 250 kbps mode for reliability; PA+LNA version extends range to 100m; 3.3V logic
  • ⚙️
    3× L9110 Motor Driver + 2× N20 Motors — L9110 is tiny (DIP-8); fits on small robot chassis; 800mA per channel
  • 👁️
    3× TCRT5000 IR Sensors — Detect other robots for proximity-based separation rule; also useful for floor edge detection
  • 🔋
    3× 18650 Li-ion + TP4056 Charger — Per robot; 3.7V direct to motors via L9110; MT3608 boost to 5V for Arduino

⚙️ How It Works

  1. 1

    nRF24L01 Mesh Communication

    Each robot has a unique address (pipe). Robot 0 (leader) broadcasts its position struct every 20ms on pipe 0. Robots 1 and 2 listen on pipe 0 and also broadcast on their own pipes. Use RF24 library in RF24_PA_LOW power mode initially for lab testing. Channel 108 (above WiFi range). Packet: struct Msg { uint8_t id; float x, y, heading; } — 14 bytes at 250kbps takes ~450µs to transmit.

  2. 2

    Position Estimation

    Without GPS, use wheel odometry: encoder-counted wheel rotations → dead reckoning. x += cos(heading)*dist; y += sin(heading)*dist. Drift accumulates over time, but for short-range formation tasks (arena <2m) it's sufficient. Optional: use a fixed camera overhead (OpenCV + ArUco markers on each robot) to provide ground-truth position over Bluetooth to demonstrate the gap between odometry and ground truth.

  3. 3

    Leader-Follower Control Law

    Follower target = leader position + offset vector (distance d at angle α relative to leader heading). Compute error: angle_err = atan2(target.y - my.y, target.x - my.x) - my_heading. Turn PID on angle_err. Drive speed proportional to distance_err. Separation: if distance to any other robot < min_sep (15cm), override with repulsion: steer away. This prevents collisions in tight formations.

  4. 4

    Behaviour Selection

    Use a pushbutton (or Bluetooth command from phone) to cycle between: LEADER-FOLLOW, FORMATION, FLOCK, DISPERSE. Each mode changes the target computation but the underlying PID motor control is the same. FLOCKING uses Reynolds rules: separation force (repel if <10cm), alignment force (match average heading), cohesion force (move toward centroid of visible peers). Weighted sum of forces → target heading.

💻 Core Code Snippet

#include <RF24.h>
RF24 radio(9, 10); // CE, CSN

struct RobotState { uint8_t id; float x, y, heading; };
RobotState my_state, peers[3];
const byte pipes[3][6] = {"0Node","1Node","2Node"};
const uint8_t MY_ID = 0; // change per robot

void setup() {
  radio.begin(); radio.setDataRate(RF24_250KBPS);
  radio.openWritingPipe(pipes[MY_ID]);
  for(int i=0;i<3;i++) if(i!=MY_ID) radio.openReadingPipe(i+1, pipes[i]);
  radio.startListening();
}

void broadcastState() {
  radio.stopListening();
  radio.write(&my_state, sizeof(my_state));
  radio.startListening();
}

void receiveAll() {
  uint8_t pipe;
  while(radio.available(&pipe)) {
    RobotState msg;
    radio.read(&msg, sizeof(msg));
    if(msg.id < 3) peers[msg.id] = msg;
  }
}

float leaderFollowerTarget(uint8_t follow_id, float offset_d, float offset_a) {
  float tx = peers[follow_id].x + offset_d * cos(peers[follow_id].heading + offset_a);
  float ty = peers[follow_id].y + offset_d * sin(peers[follow_id].heading + offset_a);
  return atan2(ty - my_state.y, tx - my_state.x);
}
