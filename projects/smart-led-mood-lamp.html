Smart LED Mood Lamp

Build a colour-changing ambient lamp with 12 WS2812B NeoPixels and an Arduino Uno. Cycle through seven mood presets — Sunset, Ocean, Forest, Party, Relax, Focus & Night — with a single button press.

📖 Introduction

WS2812B LEDs integrate the LED chip and PWM driver into a single 5 mm package. Each pixel is individually addressable over one data wire at 800 kHz. The FastLED library manages timing automatically, so you can set any colour using HSV (hue, saturation, value) with a single function call.

A tactile button on hardware-interrupt pin D2 advances the mode counter. A debounce timer in the ISR prevents double-triggers. Modes store their palette as a small PROGMEM array, then a blend loop fades each LED from its current colour to the target over 30 frames, producing a polished 500 ms crossfade.

🤡 Components Required

  • 🧠
    Arduino Uno R3 — Any clone (~$4). Data output on digital pin D6.
  • 💡
    WS2812B LED Ring (12 pixels) — 5V addressable (~$2). Add 470 Ω resistor in series on data line.
  • 🔋
    5V 2A USB Power Adapter — 12 LEDs full-white draw ~720 mA. Power ring from adapter 5V, not Arduino pin.
  • 🔘
    Tactile Push Button + 10 kΩ pull-down — D2 hardware interrupt. Optional: 100 nF ceramic cap across button pins.
  • 🛧
    1000 µF / 6.3V Electrolytic Capacitor — Across 5V and GND close to the ring to absorb power-on current spike.
  • 🔧
    Breadboard, jumper wires, USB-A cable — Optional: frosted acrylic dome (~$3) as diffuser for soft lamp glow.

⚙ How It Works

  1. 1

    Interrupt Service Routine & Mode Counter

    An ISR attached to D2 (FALLING edge) increments currentMode. A if (millis() - lastPress > 200) guard inside the ISR provides software debounce. Mode number wraps with % NUM_MODES so it cycles back to Sunset after Night.

  2. 2

    Palette Crossfade

    Each of the 7 modes stores a 3-byte {hue, sat, val} tuple in PROGMEM. On mode change the loop uses FastLED’s blend() to interpolate every LED over 30 render frames (~500 ms at 60 fps), giving smooth colour transitions without blocking.

  3. 3

    Breathing Effect

    Relax and Focus modes modulate global brightness with a sine wave: setBrightness(128 + 127 * sin(millis() / 1500.0)). This creates a slow, organic 3-second inhale–exhale without changing hue or saturation.

  4. 4

    Party & Sparkle Mode

    Every 80 ms each pixel receives a new random8() hue at full saturation. Roughly 10 % of pixels are randomly set to CRGB::White for one frame to simulate sparkle. FastLED.delay(20) caps updates at 50 fps.

💻 Core Code Snippet

#include <FastLED.h>

#define LED_PIN   6
#define NUM_LEDS  12
#define BTN_PIN   2
#define NUM_MODES 7

CRGB leds[NUM_LEDS];
volatile uint8_t currentMode = 0;
volatile unsigned long lastPress = 0;

// {hue, saturation, value} per mode
const uint8_t pal[NUM_MODES][3] PROGMEM = {
  {5,   230, 200},  // Sunset
  {140, 200, 180},  // Ocean
  {96,  220, 160},  // Forest
  {0,   0,   255},  // Party (handled separately)
  {190, 180, 140},  // Relax
  {64,  200, 200},  // Focus
  {0,   0,    20},  // Night
};

void IRAM_ATTR onBtn() {
  if (millis() - lastPress > 200) {
    currentMode = (currentMode + 1) % NUM_MODES;
    lastPress = millis();
  }
}

void setup() {
  FastLED.addLeds<WS2812B, LED_PIN, GRB>(leds, NUM_LEDS);
  FastLED.setBrightness(200);
  pinMode(BTN_PIN, INPUT);
  attachInterrupt(digitalPinToInterrupt(BTN_PIN), onBtn, FALLING);
}

void loop() {
  uint8_t m = currentMode;
  if (m == 3) {                           // Party
    for (int i = 0; i < NUM_LEDS; i++)
      leds[i] = CHSV(random8(), 255, 255);
  } else {
    uint8_t bri = (m == 4 || m == 5)     // Relax / Focus breathing
      ? 128 + 127 * sin(millis() / 1500.0)
      : pgm_read_byte(&pal[m][2]);
    for (int i = 0; i < NUM_LEDS; i++)
      leds[i] = CHSV(pgm_read_byte(&pal[m][0]) + i * 10,
                     pgm_read_byte(&pal[m][1]), bri);
  }
  FastLED.show();
  FastLED.delay(20);
}
