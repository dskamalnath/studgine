PIR Intruder Alarm

Detect motion with a passive infrared sensor and trigger a buzzer alarm — learn interrupt-driven programming and time-delay deactivation.

📖 Introduction

A Passive Infrared (PIR) sensor detects the change in heat signature as a warm body moves through its field of view. Unlike active sensors that emit a signal, the PIR simply "listens" for IR radiation changes — making it cheap, power-efficient, and ideal for motion detection.

In this project you'll wire an HC-SR501 PIR module to an Arduino, sound a buzzer when motion is detected, and auto-disarm after a configurable timeout using millis()-based non-blocking timekeeping. You'll also explore sensitivity and time-delay trim-pot adjustments on the sensor itself.

⚙️ How It Works

  1. 1
    PIR output goes HIGH — When the HC-SR501 detects IR radiation change it pulls its OUT pin HIGH for the duration set by the onboard time-delay pot. Connect OUT to digital pin 2 on the Arduino, and set the pin as INPUT.
  2. 2
    Read and react — In loop() poll digitalRead(pirPin). When HIGH, record alarmStart = millis(), turn the LED on, and start the buzzer. Set a boolean alarmActive = true.
  3. 3
    Auto-deactivation — Each loop iteration, if alarmActive is true and millis() - alarmStart > ALARM_DURATION (e.g. 10 000 ms), turn off the buzzer and LED, reset the boolean. This keeps the alarm non-blocking so you can add other logic later.
  4. 4
    Tune the sensor — The HC-SR501 has two trim pots: one for sensitivity (detection range 3–7 m) and one for hold time (0.5 s–200 s). Optionally add a second jumper to set retriggering mode so each new motion resets the hold timer.
