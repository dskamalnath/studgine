Electronic Doorbell

Press a button to play a custom melody through a passive buzzer. Experiment with tone frequencies, note durations, and song arrays to personalise your doorbell chime.

📖 Introduction

A passive buzzer has no internal oscillator — unlike an active buzzer, it requires a varying frequency signal to produce sound. This makes it ideal for songs, since you control pitch by changing the frequency you supply. Arduino's tone(pin, frequency, duration) function does exactly this — it uses a hardware timer to generate a square wave at the requested frequency.

Musical notes map directly to frequencies: middle C (C4) = 261 Hz, D4 = 294 Hz, E4 = 330 Hz, and so on. By storing a melody as an array of note macros and a corresponding array of durations, the loop plays through each note sequentially, creating a recognisable tune.

⚙️ How It Works

  1. 1
    Define note frequencies — At the top of your sketch include or define the standard pitches: #define NOTE_C4 262, #define NOTE_D4 294, etc. (Arduino IDE ships with a pitches.h header you can include: #include <pitches.h>). Then declare your melody and duration arrays: int melody[] = {NOTE_C4, NOTE_E4, NOTE_G4, NOTE_C5}; int noteDur[] = {4, 4, 4, 2}; // 4=quarter, 2=half
  2. 2
    Wire buzzer and button — Buzzer – leg → GND, + leg → 100 Ω → D9. Button: one leg → 5 V, other leg → D2 and via 10 kΩ to GND. Configure D2 as INPUT.
  3. 3
    Play melody on button press — In loop(): if digitalRead(btnPin) == HIGH (button pressed) iterate through the melody array. For each note: calculate beat duration as int beatDur = 1000 / noteDur[i]; then call tone(BUZZER_PIN, melody[i], beatDur * 0.9); then delay(beatDur);. The 10% gap between notes avoids notes blurring together.
  4. 4
    Customise your chime — Change the melody[] and noteDur[] arrays to programme any song. Classic starters: the first 4 bars of Für Elise, Jingle Bells, or a 2-note "ding-dong". Add a second button on D3 to cycle through multiple melody modes stored in a 2D array.
