Wi-Fi Scrolling Notice Board

Update a scrolling message on a MAX7219 LED matrix from any browser on the same network — a great intro to captive portals and web servers on ESP8266.

📖 Introduction

The Wi-Fi Scrolling Notice Board turns a chain of MAX7219 LED matrix modules into a remotely-updatable display. Anyone connected to the same Wi-Fi network can type a new message in a browser form — no app, no Bluetooth, no physical access needed.

The ESP8266 runs a lightweight HTTP server. When a POST request arrives, the firmware updates a global string and restarts the scroll. This project teaches you web server routing, HTML form handling, and how to drive cascaded LED matrices over SPI — all directly transferable skills for embedded UI and signage work.

🧩 Components Required

  • 🎛️
    NodeMCU ESP8266 (v3) — Built-in Wi-Fi, 3.3 V logic
  • 📟
    MAX7219 LED Matrix Modules ×4 — 8×8 dot matrix, chainable via SPI
  • 🔌
    Breadboard + Jumper Wires — SPI: CLK → D5, MOSI → D7, CS → D8
  • 🔋
    5 V / 1 A USB Power Supply — Powers ESP8266 + matrix chain

⚙️ How It Works

  1. 1

    Chain MAX7219 Modules

    Connect Data-In of the first module to ESP8266 MOSI (D7), CLK to D5, and CS to D8. Chain following modules (DOUT → DIN). Set NUM_MAX in the MD_MAX72XX library to match your chain length.

  2. 2

    Start a Wi-Fi Access Point or Station

    Use WiFi.softAP("NoticeBoard","password") to create a dedicated hotspot, or use WiFi.begin(ssid, pass) to join your home network. The ESP8266WebServer listens on port 80.

  3. 3

    Serve the HTML Form

    The root handler (server.on("/", handleRoot)) returns an inline HTML page with a text input and Submit button. Styling is minimal — the whole page fits in a PROGMEM string literal to save RAM.

  4. 4

    Scroll the Updated Message

    On POST, extract the URL-decoded message string and store it globally. The scroll loop in loop() uses MD_Parola's displayText() with SCROLL_LEFT effect and restarts each cycle. Speed is configurable via a second form field.

💻 Core Code Snippet

#include <ESP8266WiFi.h>
#include <ESP8266WebServer.h>
#include <MD_Parola.h>
#include <MD_MAX72XX.h>
#include <SPI.h>

#define NUM_MAX  4
#define CLK_PIN  D5
#define DATA_PIN D7
#define CS_PIN   D8

MD_Parola display = MD_Parola(MD_MAX72XX::FC16_HW, CS_PIN, NUM_MAX);
ESP8266WebServer server(80);
String message = "HELLO WORLD";

void handleRoot() {
  String html = "<form method='POST' action='/update'>"
    "Message: <input name='msg' value='" + message + "'>"
    "<input type='submit' value='Update'></form>";
  server.send(200, "text/html", html);
}

void handleUpdate() {
  if (server.hasArg("msg")) {
    message = server.arg("msg");
    message.toUpperCase();
    display.displayText(message.c_str(), PA_CENTER, 60, 0, PA_SCROLL_LEFT, PA_SCROLL_LEFT);
  }
  server.sendHeader("Location", "/");
  server.send(303);
}

void setup() {
  display.begin(); display.setIntensity(5); display.displayClear();
  WiFi.softAP("NoticeBoard", "12345678");
  server.on("/", handleRoot);
  server.on("/update", HTTP_POST, handleUpdate);
  server.begin();
  display.displayText(message.c_str(), PA_CENTER, 60, 0, PA_SCROLL_LEFT, PA_SCROLL_LEFT);
}

void loop() {
  server.handleClient();
  if (display.displayAnimate()) {
    display.displayReset();  // restart scroll
  }
}
