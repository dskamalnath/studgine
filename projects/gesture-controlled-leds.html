Gesture-Controlled LEDs

Swipe left, right, up, or down to change LED colours and animations — introduces the APDS-9960 I2C gesture sensor and event-driven programming.

Home › Projects › Gesture-Controlled LEDs
👋

Swipe gesture → APDS-9960 I2C event → LED colour change

▶ Watch on YouTube

📖 Introduction

The APDS-9960 is a tiny I2C sensor that detects four directional gestures (LEFT, RIGHT, UP, DOWN) by measuring IR light reflected from a moving hand across its photodiode array. It also provides ambient light, colour (RGBC), and proximity readings — all in one compact module.

In this project, each swipe direction cycles through a colour on an RGB LED or triggers a different light animation on a strip. It's your first step into event-driven, gesture-based user interfaces — the same concept used in smart displays and touchless kiosks.

⚙️ How It Works

  1. 1
    Wire I2C — APDS-9960 SDA → A4, SCL → A5 (Arduino Uno). Power at 3.3 V. Include #include <SparkFun_APDS9960.h>, initialise with apds.init() and apds.enableGestureSensor(true) in setup().
  2. 2
    Read gestures — In loop(): if apds.isGestureAvailable() is true, call int gesture = apds.readGesture(). Returns DIR_LEFT, DIR_RIGHT, DIR_UP, or DIR_DOWN constants from the library.
  3. 3
    Change LED colour — Use a switch(gesture) block. Each case sets different PWM values on R (pin 9), G (pin 10), B (pin 11). For example: LEFT→red (255,0,0), RIGHT→green, UP→blue, DOWN→yellow. Store the current colour in a variable so it persists between gestures.
  4. 4
    Add animations — Extend with colour-cycling: each LEFT/RIGHT swipe steps through a predefined colour array. UP triggers a fast blink animation, DOWN triggers a slow pulse using an abs(sin()) brightness fade in a loop.
