Wi-Fi Oscilloscope

Stream live ADC samples from an ESP32 over WebSocket to an HTML5 canvas in any browser — up to ~15 kHz analog bandwidth for audio and slow-signal debugging.

📖 Introduction

The ESP32's SAR ADC can sample at up to ~1 Msps in theory, though Wi-Fi overhead limits practical streaming to around 10–50 kSps. Still, that's enough to observe audio signals (up to ~15 kHz), PWM ripple, sensor outputs, and low-frequency AC signals — far better than just a Serial Plotter.

The ESP32 runs a WebSocket server (AsyncWebServer + AsyncWebSocket). The firmware collects 256-sample ADC bursts on pin 34 (input-only, best ADC linearity) and serialises them as JSON or binary blobs over the WebSocket. A tiny HTML/JavaScript client served from SPIFFS renders the data on a canvas element, mimicking a real oscilloscope display complete with grid, voltage scale, and frequency estimate.

🧩 Components Required

  • 🎛️
    ESP32 DevKit (30-pin) — Dual-core, built-in Wi-Fi, 12-bit ADC on GPIO34
  • 🛡️
    100 kΩ + 100 kΩ Voltage Divider — Shifts 0–5 V signals into 0–3.3 V ADC safe range
  • 🔌
    0.1 µF Capacitor (input decoupling) — Reduces high-frequency noise at ADC input
  • 🧱
    Breadboard + Probe Clips — For connecting signal source under test
  • 💾
    MicroUSB Cable + PC/Phone Browser — No extra hardware for display — browser is the UI

⚙️ How It Works

  1. 1

    Serve HTML from SPIFFS

    Upload the oscilloscope HTML/JS page to ESP32 SPIFFS (Flash File System) using the Arduino IDE SPIFFS uploader. The root GET handler serves this file so any browser connecting to the ESP32's IP gets the scope UI immediately — no app installation needed.

  2. 2

    Collect ADC Burst with Timer ISR

    Use ESP32's hardware timer to trigger ADC readings at a precise rate (e.g., 10 kHz = 100 µs interval). Fill a double-buffer: while buffer A is being read, the ISR fills buffer B. This minimises sample jitter — crucial for correct waveform timing.

  3. 3

    Send Binary Frame over WebSocket

    Once a 256-sample buffer is ready, pack as a binary message: 2 bytes per sample (little-endian uint16) → 512 bytes total. Call ws.binaryAll(buffer, 512). Binary frames are ~6× faster than JSON strings and easier for JavaScript to decode with Int16Array.

  4. 4

    Render Waveform on Canvas

    In the browser JS, ws.onmessage receives an ArrayBuffer. Create a Uint16Array view, scale each sample to canvas Y-coordinates (y = canvasH - (sample / 4095) * canvasH), then draw a polyline using ctx.lineTo(). Compute frequency by counting zero-crossings; display Vpp from min/max samples.

💻 Core Code Snippet

// ESP32 Firmware — WebSocket Oscilloscope
#include <WiFi.h>
#include <ESPAsyncWebServer.h>
#include <SPIFFS.h>

AsyncWebServer  server(80);
AsyncWebSocket  ws("/ws");

#define ADC_PIN   34
#define BUF_SIZE  256
uint16_t adcBuf[BUF_SIZE];
hw_timer_t* timer = NULL;
volatile int sampleIdx = 0;
volatile bool bufReady = false;

void IRAM_ATTR onTimer() {
  if (!bufReady) {
    adcBuf[sampleIdx++] = analogRead(ADC_PIN);
    if (sampleIdx >= BUF_SIZE) {
      sampleIdx = 0; bufReady = true;
    }
  }
}

void setup() {
  SPIFFS.begin(true);
  WiFi.softAP("ESP32-Scope","12345678");
  ws.onEvent([](AsyncWebSocket* s, AsyncWebSocketClient* c,
                AwsEventType t, void* a, uint8_t* d, size_t l){});
  server.addHandler(&ws);
  server.serveStatic("/", SPIFFS, "/").setDefaultFile("index.html");
  server.begin();
  // 10 kHz timer
  timer = timerBegin(0, 80, true);  // 80 prescaler = 1 µs tick
  timerAttachInterrupt(timer, &onTimer, true);
  timerAlarmWrite(timer, 100, true);  // 100 µs = 10 kHz
  timerAlarmEnable(timer);
}

void loop() {
  if (bufReady) {
    ws.binaryAll((uint8_t*)adcBuf, BUF_SIZE * 2);
    bufReady = false;
  }
  ws.cleanupClients();
}
