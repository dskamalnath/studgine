Adaptive Traffic Light (CV)

A Pi Camera detects vehicle queue density per lane using YOLOv8 object detection and dynamically allocates green-light duration proportional to queue length — a hardware proof-of-concept of a smart-city adaptive traffic management system controlled by computer vision.

📖 Introduction

Traditional traffic lights run on fixed-time cycles, keeping queues waiting even when the opposing lane is empty. Adaptive signal control systems (used in cities like Sydney and London) analyse real-time queue density and allocate green phases proportionally. This project implements the concept on a tabletop scale model: miniature toy cars represent vehicles, a Pi Camera captures each lane's ROI (region of interest), YOLOv8n detects and counts vehicles, and an Arduino Mega controls 8 physical LEDs forming a 4-way intersection traffic light array based on the Pi's allocation decisions.

🧩 Components Required

  • 📷
    Raspberry Pi Camera Module v2 — Overhead mount above intersection model; 640×480; 30fps; wide-angle lens for all 4 lanes in frame
  • 🚗
    Tabletop Road Model + Toy Cars — 600×600mm printed road layout; Lego or diecast 1:64 cars; contrasting colours for YOLO detection
  • 🚦
    Arduino Mega + 8-12 RGB LEDs — Physical 4-way traffic light array; red/amber/green per direction; Pi sends commands via UART serial
  • 🧠
    Raspberry Pi 4 (4 GB) — YOLOv8n TFLite at ~10fps; queue density per ROI; adaptive allocation algorithm; sends commands to Arduino

⚙️ How It Works

  1. 1

    Multi-Lane ROI Vehicle Detection

    Define 4 lane ROIs by pixel regions in the camera frame: ROIS = {'N':(0,0,160,240), 'S':(160,0,320,240), 'E':(320,0,480,240), 'W':(480,0,640,240)}. For each ROI, crop frame, run YOLOv8n: results = model.predict(crop, classes=[2,5,7]) (car, bus, truck). Count detections per lane: queue[lane] = len(results[0].boxes). Runs at ~10fps on Pi 4. Alternative: use background subtraction (MOG2) for faster detection without YOLO.

  2. 2

    Proportional Green Time Allocation

    Adaptive algorithm: compute total queue: total = sum(queue.values()). Allocate green time per lane proportional to queue: green_time[lane] = MAX_CYCLE * queue[lane] / max(total, 1). Clamp: min 5s, max 60s per phase. Sort lanes by queue depth (serve longest queue first). Emergency mode: if any lane queues >20 vehicles, double its allocation. Compare to baseline fixed-60s cycle: adaptive reduces average wait time by ~35% in simulations.

  3. 3

    Arduino Traffic Light Controller

    Pi sends traffic plan via UART: serial.write(f"A:{gA},B:{gB},C:{gC},D:{gD}\n".encode()). Arduino parses and executes: set active lane to GREEN, others RED, wait green_time. Amber transition: 3s. Full sequence: RGRG cycle with proportional durations. LED pins: each direction has R+G+Y on 3 GPIO pins. Cycle report sent back: Serial.print("CYCLE_DONE"). Pi logs each cycle duration + queue counts to CSV for statistics.

  4. 4

    Performance Dashboard

    Overlaid CV display: draw ROI rectangles, vehicle bounding boxes, queue count per lane, active green direction. Flask web dashboard: real-time queue bar chart + cumulative wait time comparison (adaptive vs. fixed baseline). Log 100 cycles and compute: average queue length, max wait time, throughput (vehicles cleared per minute). Demonstrate improvement with varying traffic densities — creates compelling data for project presentation.

💻 Core Code Snippet

from ultralytics import YOLO
from picamera2 import Picamera2
import serial, time, cv2, numpy as np

model = YOLO('yolov8n.pt')  # or TFLite version
cam = Picamera2(); cam.start()
arduino = serial.Serial('/dev/ttyUSB0', 9600, timeout=1)

ROIS = {
    'N': (0,   0,   320, 240),
    'S': (320, 0,   640, 240),
    'E': (0,   240, 320, 480),
    'W': (320, 240, 640, 480),
}
MIN_GREEN, MAX_GREEN, CYCLE = 5, 60, 80

def count_vehicles(frame, roi):
    x1,y1,x2,y2 = roi
    crop = frame[y1:y2, x1:x2]
    results = model.predict(crop, classes=[2,5,7], verbose=False)
    return len(results[0].boxes)

def adaptive_schedule(queues):
    total = max(sum(queues.values()), 1)
    schedule = {lane: max(MIN_GREEN, min(MAX_GREEN,
        int(CYCLE * q / total))) for lane, q in queues.items()}
    # Serve longest queue first
    return dict(sorted(schedule.items(), key=lambda x: x[1], reverse=True))

while True:
    frame = cam.capture_array()
    queues = {lane: count_vehicles(frame, roi)
              for lane, roi in ROIS.items()}
    sched = adaptive_schedule(queues)
    cmd = ','.join(f'{l}:{t}' for l,t in sched.items()) + '\n'
    arduino.write(cmd.encode())
    print(f"Queue: {queues} → Schedule: {sched}")
    time.sleep(1)
