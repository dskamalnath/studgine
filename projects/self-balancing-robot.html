Self-Balancing Robot

📖 Introduction

The self-balancing (inverted pendulum) robot is the classic advanced project that ties together IMU sensor fusion, PID control theory, real-time loop scheduling, and motor drive electronics. Understanding this project means understanding most of what goes into drone flight controllers, bipedal robots, and gyroscopic camera gimbals.

The core challenge: the robot falls over in milliseconds if the control loop is too slow. You'll implement a complementary filter (or Kalman filter) to fuse accelerometer + gyroscope data for a low-noise tilt angle, then tune a cascaded PID: outer loop for position/speed, inner loop for angle.

🧩 Components Required

  • 🧠
    Raspberry Pi 4 (or Pi 3B+) — Runs Python control loop at 100 Hz via pigpio or RPi.GPIO; requires low-latency OS (disable unnecessary services)
  • 📐
    MPU-6050 IMU (I2C) — 3-axis gyro + 3-axis accelerometer; 1 kHz output; ±2000°/s gyro; I2C address 0x68
  • ⚙️
    2× N20 DC Gear Motors (+ encoders) — 100–300 RPM, 6V; Hall-effect encoders optional but recommended for speed feedback in outer loop
  • 🔌
    L298N or TB6612FNG H-Bridge — TB6612FNG preferred (3.4A peak, lower heat, logic-level PWM); controls forward/reverse + speed
  • 🔋
    2S LiPo Battery (7.4V, 1000mAh) — Powers motors directly; 5V UBEC regulator for Raspberry Pi; LiPo suits high-current motor spikes
  • 🖨️
    3D-Printed / MDF Chassis — Low centre-of-mass body with IMU mounted at robot centroid; wheel axle at exact bottom

⚙️ How It Works

  1. 1

    Sensor Fusion — Complementary Filter

    Raw accelerometer angle: accel_angle = atan2(ay, az) * 180/π. Raw gyro rate: gyro_rate = gx (°/s). Complementary filter: angle = 0.98 * (angle + gyro_rate * dt) + 0.02 * accel_angle. The 0.98/0.02 weighting trusts the gyro short-term and accelerometer long-term, eliminating both gyro drift and accelerometer noise. Update at 100 Hz (dt = 10 ms).

  2. 2

    Cascaded PID Control

    Inner loop (angle PID): setpoint = balance_angle (typically ±2° from vertical). Error = setpoint − measured_angle. PID output → motor PWM. Kp ≈ 30, Ki ≈ 0.5, Kd ≈ 0.8 (tune empirically). Run at 100 Hz. Outer loop (speed PID): uses encoder tick rate to compute actual speed; adjusts balance_angle setpoint to drive position correction. Run at 20 Hz. The cascaded structure separates fast balance stabilisation from slow positional drift correction.

  3. 3

    Motor PWM via pigpio

    Use pigpio for hardware PWM (not software PWM — causes jitter). pi.set_PWM_frequency(pin, 1000). PWM duty cycle 0–255. Direction via H-bridge IN1/IN2 pins. Clamp PID output: pwm = max(-255, min(255, pid_output)). For L298N: enable pin gets PWM, IN1/IN2 set direction. Dead-band: skip PWM < 20 to prevent motor buzzing at near-zero command.

  4. 4

    PID Tuning Procedure

    Step 1: set Ki=Kd=0, increase Kp until robot starts oscillating rapidly. Step 2: add Kd to dampen oscillations. Step 3: add Ki slowly to correct slow lean drift. Balance point offset: robot may need a non-zero setpoint angle if centre of mass isn't perfectly centred — find empirically while holding robot. Use serial logging to plot angle and PWM during tuning.

💻 Core Code Snippet

import smbus2, time, pigpio, math

pi = pigpio.pi()
bus = smbus2.SMBus(1)
MPU = 0x68; bus.write_byte_data(MPU, 0x6B, 0)  # wake MPU6050

Kp, Ki, Kd = 30.0, 0.5, 0.8
integral = 0; prev_error = 0; angle = 0
SETPOINT = 0.0  # degrees from vertical

def read_mpu():
    raw = bus.read_i2c_block_data(MPU, 0x3B, 14)
    ax = (raw[0]<<8|raw[1])/16384.0
    ay = (raw[2]<<8|raw[3])/16384.0
    az = (raw[4]<<8|raw[5])/16384.0
    gx = (raw[8]<<8|raw[9])/131.0
    return ax, ay, az, gx

def set_motors(pwm):
    speed = abs(int(pwm)); fwd = pwm > 0
    pi.write(17, 1 if fwd else 0); pi.write(18, 0 if fwd else 1)
    pi.write(22, 0 if fwd else 1); pi.write(23, 1 if fwd else 0)
    pi.set_PWM_dutycycle(12, speed); pi.set_PWM_dutycycle(13, speed)

dt = 0.01
while True:
    t0 = time.time()
    ax, ay, az, gx = read_mpu()
    accel_angle = math.degrees(math.atan2(ay, az))
    angle = 0.98*(angle + gx*dt) + 0.02*accel_angle
    error = SETPOINT - angle
    integral += error * dt
    derivative = (error - prev_error) / dt
    pwm = Kp*error + Ki*integral + Kd*derivative
    pwm = max(-255, min(255, pwm))
    set_motors(pwm); prev_error = error
    time.sleep(max(0, dt - (time.time()-t0)))
