📖 Introduction

Commercial programmable timer relays used in farms, greenhouses, factories, and homes can cost $30–$100 each. This project builds a 4-channel equivalent with a richer programming interface — a 16×2 LCD and 4 navigation buttons allowing you to set independent daily ON/OFF times for each relay channel. Schedules are stored in EEPROM so they survive power cuts.

The project covers one of the most practically applicable embedded patterns: the hierarchical menu system. Real industrial controllers (PLCs, VFDs) use exactly this button-and-LCD navigation approach. You'll also learn safe relay switching — preventing simultaneous energisation of multiple channels to manage inrush current, and debouncing the programming buttons for reliable menu navigation.

🧩 Components Required

  • ⚡
    4-Channel 5V Relay Module — Opto-isolated; rated for 10A/250VAC; one per controlled load
  • 🧠
    Arduino Uno / Mega — Uno for 4 ch; Arduino Mega if expanding to 8+ channels (more digital IO)
  • ⏰
    DS3231 RTC Module — Precision ±2 ppm RTC; battery-backed; I2C interface for accurate daily scheduling
  • 🖥️
    16×2 I2C LCD — Menu display; only 2 wires (SDA/SCL) plus power; PCF8574 I2C backpack
  • 🔘
    4× Push Buttons — UP / DOWN / SELECT / BACK navigation for 4-level menu hierarchy
  • 💾
    EEPROM (built-in Arduino) — Arduino has 1 KB EEPROM; stores 4×(ON_hour, ON_min, OFF_hour, OFF_min, enabled) = 20 bytes

⚙️ How It Works

  1. 1

    DS3231 RTC — Accurate Timekeeping

    Use the RTClib library. On first boot (or RTC battery dead), set RTC time to compile time: rtc.adjust(DateTime(F(__DATE__), F(__TIME__))). Main loop reads DateTime now = rtc.now() every second. Compare now.hour() and now.minute() against each channel's stored ON and OFF times. Midnight wrap: handle the case where OFF time < ON time (e.g., ON=22:00, OFF=02:00) by checking if hour >= onH || hour < offH.

  2. 2

    Hierarchical Menu System

    Four-level menu tree: Level 0 = status view (all 4 channels, current time). Level 1 = select channel (CH1–CH4). Level 2 = select parameter (ON time / OFF time / Enable). Level 3 = edit value (UP/DOWN increment hour, SELECT moves to minutes, SELECT again confirms). Use a menuLevel variable and a switch-case to render the appropriate LCD content. Key debounce: 200 ms debounce timer prevents multiple triggers per press. BACK button always goes up one level.

  3. 3

    EEPROM Schedule Persistence

    Define a struct: struct Schedule { uint8_t onH, onM, offH, offM; bool enabled; }. Store 4 schedules at EEPROM addresses 0, 10, 20, 30. Write with EEPROM.put(addr, sched[i]) only when the user confirms an edit (reduces EEPROM write cycles — EEPROM has only 100,000 write cycles). On boot, read all 4 schedules with EEPROM.get(addr, sched[i]) and validate (check if read values are in valid time range 0–23:0–59; default to 00:00–00:00 disabled if corrupt).

  4. 4

    Safe Relay Switching Logic

    Relay module inputs are active-LOW (energise relay by pulling pin LOW). Use a setRelay(int ch, bool on) function that adds a 50 ms inter-channel delay when turning on multiple relays simultaneously (limits peak inrush current to power supply). Add manual override: holding SELECT for 3 seconds on a channel in the status view toggles it ON/OFF regardless of schedule (manual override flag, cleared at next schedule change). LED indicators: onboard relay LED sufficient, or add external 5 mm LEDs on relay secondary side for visual status at a glance.

💻 Core Code Snippet

#include <RTClib.h>
#include <EEPROM.h>
#include <LiquidCrystal_I2C.h>

RTC_DS3231 rtc;
LiquidCrystal_I2C lcd(0x27, 16, 2);

const int RELAY[4] = {4, 5, 6, 7};

struct Schedule { uint8_t onH, onM, offH, offM; bool enabled; };
Schedule sched[4];

void loadSchedules() { for(int i=0;i<4;i++) EEPROM.get(i*10, sched[i]); }
void saveSchedule(int i) { EEPROM.put(i*10, sched[i]); }

bool isActive(int ch) {
  if(!sched[ch].enabled) return false;
  DateTime now = rtc.now();
  int nowM = now.hour()*60 + now.minute();
  int onM = sched[ch].onH*60 + sched[ch].onM;
  int offM = sched[ch].offH*60 + sched[ch].offM;
  if(onM < offM) return nowM >= onM && nowM < offM;
  else return nowM >= onM || nowM < offM; // overnight wrap
}

void setup() {
  rtc.begin(); loadSchedules();
  lcd.begin(); lcd.backlight();
  for(int i=0;i<4;i++) { pinMode(RELAY[i], OUTPUT); digitalWrite(RELAY[i], HIGH); }
}

void loop() {
  for(int i=0;i<4;i++) {
    digitalWrite(RELAY[i], isActive(i) ? LOW : HIGH); // active LOW relay
    delay(50); // stagger switching
  }
  // ... menu handling and LCD update
  delay(1000);
}
