📖 Introduction

Overhead water tanks are common in areas with unreliable municipal supply — knowing the fill level before a morning rush prevents the embarrassing discovery of an empty tank. This project mounts an HC-SR04 ultrasonic distance sensor upside-down inside the tank lid, measuring the air gap from the sensor to the water surface. The ESP8266 converts distance to fill percentage using the known tank height, then pushes the value to a Blynk IoT dashboard accessible from anywhere.

The Blynk 2.0 platform provides a polished gauge widget, history graph, and event notifications. For SMS/messaging alerts, the Telegram Bot API requires only a simple HTTPS POST — no special IoT middleware needed. This project covers median-filtered ultrasonic sensing, Blynk virtual pins, HTTP GET/POST for Telegram, and fail-safe design for water measurement in humid environments.

  • 🌐

⚙️ How It Works

  1. 1

    Ultrasonic Distance → Fill Percentage

    Mount HC-SR04 on the inside tank lid, sensor face pointing downward toward water surface. Mounting height from bottom = TANK_HEIGHT (e.g. 150 cm for a standard 1000 L tank). Measure distance to water surface: distance_cm = duration_us / 58.2. Calculate water level: water_cm = TANK_HEIGHT - distance_cm - SENSOR_OFFSET. Fill percentage: fillPct = constrain(water_cm / TANK_HEIGHT * 100, 0, 100). Take 5 median-filtered readings to reject false echoes from tank ripples. Discard readings <2 cm (sensor dead zone) or >TANK_HEIGHT+5 (floating debris).

  2. 2

    Blynk Virtual Pin Update

    Use Blynk 2.0 (cloud.blynk.io). Create a Gauge widget on Virtual Pin V1 (range 0–100). In firmware: Blynk.virtualWrite(V1, fillPct). Set the update interval with a BlynkTimer: timer.setInterval(30000L, sendData). In the Blynk app, configure the Gauge widget min=0, max=100, label "%". Add a SuperChart widget on V1 to graph the level history over 24 hours, allowing you to see daily consumption patterns. Also write to V2 for absolute water height in cm.

  3. 3

    Telegram Low-Level Alert

    Create a Telegram bot via @BotFather (get TOKEN) and Find your chat ID with @userinfobot. When fillPct drops below 20% for 3 consecutive readings (confirms it's not a sensor glitch), send: WiFiClientSecure client; client.connect("api.telegram.org",443); String msg = "Tank level CRITICAL: " + String(fillPct) + "%"; String url = "/bot" + token + "/sendMessage?chat_id=" + chatId + "&text=" + msg; Implement a 4-hour alert rate limit so you're not spammed every 30 seconds while the tank stays low. Reset alert flag once fill goes above 30%.

  4. 4

    OTA Updates & Wi-Fi Reconnect

    Add ArduinoOTA so you can update the firmware without physically accessing the watertight enclosure. Include automatic Wi-Fi reconnect with if(WiFi.status() != WL_CONNECTED) WiFi.reconnect() in the main loop — the ESP8266 will reconnect within 10 seconds of a router restart. Set Blynk heartbeat to 60 seconds. Add a watchdog timer (ESP.wdtFeed()) to auto-reboot if the main loop stalls during an HTTPS call timeout.

💻 Core Code Snippet

#define BLYNK_TEMPLATE_ID "TMPLxxxxxx"
#define BLYNK_AUTH_TOKEN "your_token"
#include <ESP8266WiFi.h>
#include <BlynkSimpleEsp8266.h>
#include <WiFiClientSecure.h>

const int TRIG=D6, ECHO=D7;
const float TANK_H = 150.0; // cm
const int LOW_THRESHOLD = 20; // percent

BlynkTimer timer;

float readFillPct() {
  long readings[5];
  for(int i=0;i<5;i++){
    digitalWrite(TRIG,LOW); delayMicroseconds(2);
    digitalWrite(TRIG,HIGH); delayMicroseconds(10); digitalWrite(TRIG,LOW);
    readings[i] = pulseIn(ECHO, HIGH, 25000);
    delay(60);
  }
  // median of 5
  for(int i=0;i<4;i++) for(int j=i+1;j<5;j++) if(readings[j]<readings[i]){long t=readings[i];readings[i]=readings[j];readings[j]=t;}
  float dist = readings[2] / 58.2;
  float wh = TANK_H - dist - 5.0; // 5cm sensor offset
  return constrain(wh/TANK_H*100.0, 0, 100);
}

void sendData() {
  float pct = readFillPct();
  Blynk.virtualWrite(V1, pct);
  if(pct < LOW_THRESHOLD) sendTelegramAlert(pct);
}

void setup() {
  pinMode(TRIG,OUTPUT); pinMode(ECHO,INPUT);
  Blynk.begin(BLYNK_AUTH_TOKEN, "WiFi_SSID", "WiFi_Pass");
  timer.setInterval(30000L, sendData);
}

void loop() { Blynk.run(); timer.run(); }
