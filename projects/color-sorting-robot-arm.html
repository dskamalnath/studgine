Color Sorting Robot Arm

Use a TCS3200 RGB frequency sensor and three servo motors to build a robot arm that identifies object colors on a conveyor belt and drops them into matching bins — a fun introduction to embedded machine vision and inverse kinematics.

📖 Introduction

Industrial color sorting is everywhere — from candy production lines that reject wrong-colored pieces to recycling plants sorting plastics by polymer type. This project captures that same concept at hobby scale using the TCS3200 color-to-frequency sensor chip and three SG90 servo motors.

The TCS3200 contains an 8×8 array of photodiodes: 16 with red filters, 16 green, 16 blue, and 16 clear. By driving the S2/S3 select pins, you cycle through R/G/B channels and measure the output frequency — higher frequency means more of that colour in the object. The sorted output frequencies are compared to pre-calibrated reference values to classify objects as red, green, blue, or yellow. A base servo rotates the arm to the correct bin angle, a shoulder servo lowers it, and a gripper servo grips/releases. All sequences interlock so the arm never collides with the belt.

🧩 Components Required

  • 🎨
    TCS3200 Color Sensor Module — RGB frequency output; S0–S3 for photodiode filter select + frequency scale
  • 🤖
    3× SG90 Servo Motors — Base rotation (0–180°) + shoulder lift + gripper open/close
  • 🔧
    Arduino Uno / Nano — Controls sensor polling, servo PWM, and state machine logic
  • 🏗️
    3D-Printed / Acrylic Arm Frame — STL files available on Thingiverse; acrylic alternative with M3 hardware
  • 🔋
    5V 2A External Power Supply — Servos can draw 600 mA each; do NOT power from Arduino 5V pin
  • 📺
    16×2 I2C LCD — Displays detected color label + per-bin sort counter

⚙️ How It Works

  1. 1

    TCS3200 Color Reading

    Drive S0=HIGH, S1=HIGH to set frequency output scale to 100%. Set S2/S3 to select filter: S2=LOW S3=LOW → Red; S2=HIGH S3=HIGH → Green; S2=LOW S3=HIGH → Blue. Use pulseIn(OUT_PIN, LOW) to measure the period of the output square wave in microseconds — shorter period = higher frequency = more of that color. Take 10 readings per channel and average to suppress flicker noise. Calibrate white and black card reference values during setup.

  2. 2

    Color Classification Algorithm

    Normalise raw R/G/B frequencies against calibrated white/black bounds to produce 0–255 analog values per channel. Apply nearest-neighbour classification against stored prototypes: Red (R↑ G↓ B↓), Green (R↓ G↑ B↓), Blue (R↓ G↓ B↑), Yellow (R↑ G↑ B↓). Calculate Euclidean distance in RGB space — closest match wins. For unknown/noisy readings, classify as "SKIP" and allow belt to carry object past arm.

  3. 3

    Servo Arm State Machine

    Finite state machine with 6 states: WAIT → SENSE → CLASSIFY → PICK → DROP → HOME. WAIT: arm parked at home position, gripper open. SENSE: wait for object to arrive at sensor position (proximity IR trigger). PICK: lower shoulder to pick angle, close gripper, raise. DROP: rotate base to target bin angle, lower, open gripper. HOME: return to home position. Each state waits for servo to settle (100–200 ms delay after each write) before advancing.

  4. 4

    White-Balance Calibration Routine

    At startup, run calibration: place white card under sensor, press button — Arduino records minFreq (brightest reading) for each channel. Place black card, press button — records maxFreq. These bounds map to 255 and 0 respectively for normalisation. Store calibration in EEPROM (EEPROM.put(addr, calStruct)) so it survives power cycles. A "recalibrate" button in the serial menu lets you redo this any time without reflashing.

💻 Core Code Snippet

#include <Servo.h>
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

Servo baseServo, shoulderServo, gripperServo;
LiquidCrystal_I2C lcd(0x27, 16, 2);

#define S0 4 #define S1 5 #define S2 6 #define S3 7 #define OUT 8

int readColor(bool s2, bool s3) {
  digitalWrite(S2, s2); digitalWrite(S3, s3);
  delay(10);
  long sum = 0;
  for(int i=0;i<10;i++) sum += pulseIn(OUT, LOW);
  return sum / 10;
}

String classifyColor() {
  int r = readColor(LOW,LOW), g = readColor(HIGH,HIGH), b = readColor(LOW,HIGH);
  // lower freq = stronger signal; normalize against calibration
  if(r < g && r < b) return "RED";
  if(g < r && g < b) return "GREEN";
  if(b < r && b < g) return "BLUE";
  if(r < b && g < b) return "YELLOW";
  return "UNKNOWN";
}

int binAngles[] = {30, 90, 135, 170}; // R,G,B,Y
void dropToBin(int angle) {
  shoulderServo.write(60);    // lower
  delay(400);
  baseServo.write(angle);     // rotate
  delay(500);
  gripperServo.write(80);     // open
  delay(300);
  shoulderServo.write(0);     // raise
  delay(400);
  baseServo.write(90);        // home
}

void loop() {
  String col = classifyColor();
  lcd.print(col);
  int ang = (col=="RED")?30:(col=="GREEN")?90:(col=="BLUE")?135:170;
  gripperServo.write(170);   // grip
  delay(500);
  dropToBin(ang);
}
