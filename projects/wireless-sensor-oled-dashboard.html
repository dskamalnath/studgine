Wireless Sensor OLED Dashboard

Multiple ESP-NOW nodes transmit sensor readings wirelessly to a central ESP32 that aggregates and displays all data on a large OLED panel. No Wi-Fi router required — direct ESP-NOW peer-to-peer links at up to 200 m.

📖 Introduction

ESP-NOW is Espressif's proprietary connectionless wireless protocol that operates at the MAC layer alongside, but independently of, Wi-Fi. Unlike MQTT over Wi-Fi, ESP-NOW requires no router, no IP stack, and no internet connection — nodes discover each other directly using MAC addresses. Latency is under 1 ms and the protocol is extremely power-efficient, making it ideal for battery-powered sensor nodes.

This project puts 3–6 ESP32 transmitter nodes throughout a building or field, each reading local sensors (temperature, humidity, soil moisture, gas, etc.) and sending compact data packets to a single ESP32 receiver every 30 seconds. The receiver aggregates all readings and presents a clear scrolling dashboard on a 1.3″ or larger OLED. The entire system is infrastructure-free.

🧩 Components Required

  • 📡
    3–6× ESP32 Dev Boards — 1 receiver + 2–5 transmitter nodes; use identical MCU for simplified code reuse
  • 🌡️
    DHT22 / BME280 per node — Temperature + humidity; BME280 adds pressure; one sensor per transmitter node
  • 🖥️
    1.3″ SH1106 OLED (Receiver) — 128×64; bigger screen for the receiver dashboard; shows all node data simultaneously
  • 🔋
    18650 LiPo per Node — ESP32 deep sleep between transmissions: 5 µA sleep + 240 mA active ~150 ms = months battery life

⚙️ How It Works

  1. 1

    Transmitter Node Setup (ESP-NOW Sender)

    Each node has a hard-coded NODE_ID (1, 2, 3…). On wake from deep sleep: initialise Wi-Fi in STA mode (required for ESP-NOW), call esp_now_init(), register the receiver's MAC address as a peer with esp_now_add_peer(), read sensor, pack into a 12-byte struct {uint8_t nodeId, float temp, float hum, uint32_t uptime}, send with esp_now_send(receiverMAC, data, sizeof(data)), confirm send callback, then enter deep sleep for 30 s.

  2. 2

    Receiver: OnDataRecv Callback

    The receiver always-on ESP32 registers esp_now_register_recv_cb(OnDataRecv). In the callback, cast the received bytes back to the struct, store in a nodeData[6] array indexed by nodeId. Update a "last-seen" timestamp. The OLED refresh loop (every 2 s) renders all nodes. Mark stale nodes (no update > 120 s) with a ⚠ symbol so you know if a node has died or is out of range.

  3. 3

    OLED Multi-Node Dashboard Layout

    Use u8g2 library for crisp font rendering. Layout: one row per node (3 rows fit on 128×64 at font size 8). Each row: "N1: 27.4°C 61% ✓". Cycle through pages if more than 6 nodes (auto-scroll every 5 s). Add a top status bar showing total packet count and last update time. Highlight the most recently updated node with inverted text (white bg, black text).

  4. 4

    Range Extension & Broadcast Mode

    For broadcast (no peer registration): use the broadcast MAC FF:FF:FF:FF:FF:FF. This loses delivery confirmation but simplifies node code. To extend range beyond 100 m: add an external 2.4 GHz PCB antenna to the ESP32 (remove the onboard chip antenna) or use ESP32 modules with IPEX connector. For maximum range, elevate antennas and ensure line of sight. Achieved >300 m LOS at 1 Mbps in tests.

💻 Core Code Snippet

// === TRANSMITTER NODE (node_id = 1) ===
#include <esp_now.h>
#include <WiFi.h>

uint8_t receiverMAC[] = {0xAA,0xBB,0xCC,0xDD,0xEE,0xFF};
struct SensorPacket { uint8_t nodeId; float temp; float hum; };
SensorPacket pkt;

void setup() {
  WiFi.mode(WIFI_STA); esp_now_init();
  esp_now_peer_info_t peerInfo; memcpy(peerInfo.peer_addr, receiverMAC, 6);
  peerInfo.channel=0; peerInfo.encrypt=false;
  esp_now_add_peer(&peerInfo);
  pkt.nodeId=1; pkt.temp=readTemp(); pkt.hum=readHum();
  esp_now_send(receiverMAC,(uint8_t*)&pkt,sizeof(pkt));
  esp_deep_sleep(30000000); // 30s deep sleep
}

// === RECEIVER ===
#include <esp_now.h>

struct SensorPacket { uint8_t nodeId; float temp; float hum; };
SensorPacket nodes[6];

void OnDataRecv(const uint8_t *mac, const uint8_t *data, int len) {
  SensorPacket *p = (SensorPacket*)data;
  if(p->nodeId >= 1 && p->nodeId <= 6) nodes[p->nodeId - 1] = *p;
  updateOLED();
}

void setup() {
  WiFi.mode(WIFI_STA); esp_now_init();
  esp_now_register_recv_cb(OnDataRecv);
}
