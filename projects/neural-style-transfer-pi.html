Neural Style Transfer on Pi

Transform your Pi Camera live feed into Monet, Van Gogh, or Picasso in real-time using TFLite Magenta arbitrary style transfer. The two-model pipeline (style_predict 256×256 + style_transform 384×384) runs entirely on a Raspberry Pi 4 in INT8-quantised format at 2–3 fps. No GPU, no cloud — pure ARM Cortex-A72 inference.

📖 Introduction

Neural Style Transfer (NST) was first published in Gatys et al. (2015) — it could apply any painting's style to any photo, but required thousands of optimisation iterations (30+ minutes). Google Magenta's fast arbitrary style transfer (2017) solved this: two small feed-forward networks replace optimisation. The "predict" network encodes the style image into a 100-dimensional style bottleneck. The "transform" network takes the content image + style bottleneck and generates the stylised output in a single forward pass — 400ms total on Pi 4 with INT8 quantisation. Mix multiple styles by interpolating bottlenecks.

🧩 Components Required

  • 📷
    Raspberry Pi Camera v2 or HQ — Pi Camera → picamera2 → 384×384 crop for content input; 30fps capture but only 2–3fps processed (every 10th frame); preview at full speed
  • 🖥️
    Raspberry Pi 4 (4GB or 8GB) — 4×Cortex-A72 @ 1.8GHz; INT8 quantised TFLite runs in ~420ms per frame; 4GB sufficient for both models loaded simultaneously
  • 📺
    HDMI Monitor (or VNC Viewer) — Display live before/after side-by-side in Pygame fullscreen; HDMI framebuffer direct; or stream to Chromebook/laptop via VNC
  • 🎨
    Style Image Collection (JPG files) — 20+ painting images at 256×256; a button cycles through styles at runtime; pre-compute style bottlenecks at startup for instant style switching

⚙️ How It Works

  1. 1

    Download and Load TFLite Models

    Two models from Google Magenta: style_predict_quantized_256.tflite (1.3MB) and style_transform_quantized_384.tflite (4.8MB). Download: wget https://tfhub.dev/google/lite-model/magenta/arbitrary-image-stylization-v1-256/int8/predict/1?lite-format=tflite -O predict.tflite; wget ... -O transform.tflite. Load: import tflite_runtime.interpreter as tflite; pred_interp = tflite.Interpreter("predict.tflite"); pred_interp.allocate_tensors(). Get input/output tensor details to know exact shapes and types.

  2. 2

    Pre-compute Style Bottlenecks

    At startup, run each style image through the predict model once. Input: (1, 256, 256, 3) float32. style_img = cv2.resize(cv2.imread("starry_night.jpg"), (256,256)) / 255.0; style_input = style_img[np.newaxis].astype(np.float32); pred_interp.set_tensor(pred_input_idx, style_input); pred_interp.invoke(); bottleneck = pred_interp.get_tensor(pred_output_idx).copy(). Shape: (1, 1, 1, 100). Cache all style bottlenecks in a dict — instant style switching without re-running predict model.

  3. 3

    Real-time Style Transform

    Capture from Pi Camera: from picamera2 import Picamera2; cam = Picamera2(); cam.start(). Per frame: frame = cam.capture_array(); content = cv2.resize(frame, (384,384)) / 255.0; content_in = content[np.newaxis].astype(np.float32). Set both inputs to transform model: xfm_interp.set_tensor(content_idx, content_in); xfm_interp.set_tensor(style_idx, current_bottleneck); xfm_interp.invoke(); styled = xfm_interp.get_tensor(output_idx)[0]. Convert back: out_img = (styled * 255).clip(0,255).astype(np.uint8). Display via Pygame surface.

  4. 4

    Style Blending + UI Controls

    Mix styles by interpolating bottlenecks: blended = alpha * bneck_monet + (1-alpha) * bneck_picasso. Use alpha slider on touchscreen or GPIO buttons. Strength control: scale bottleneck intensity. Side-by-side display: combined = np.hstack([original_resized, styled_display]); surface = pygame.surfarray.make_surface(combined.transpose(1,0,2)). GPIO button mapping: up/down = cycle styles; left/right = adjust strength. Save to file: cv2.imwrite(f"styled_{timestamp}.jpg", out_img). FPS counter: use time.perf_counter() before/after invoke().

💻 Core Code Snippet

import cv2, numpy as np, time
import tflite_runtime.interpreter as tflite
from picamera2 import Picamera2

# Load models
pred = tflite.Interpreter("predict_quantized_256.tflite"); pred.allocate_tensors()
xfm  = tflite.Interpreter("transform_quantized_384.tflite"); xfm.allocate_tensors()
p_in  = pred.get_input_details()[0]['index']
p_out = pred.get_output_details()[0]['index']
c_in  = xfm.get_input_details()[0]['index']   # content
s_in  = xfm.get_input_details()[1]['index']   # style bottleneck
x_out = xfm.get_output_details()[0]['index']

def get_bottleneck(style_path):
    img = cv2.resize(cv2.imread(style_path), (256,256)) / 255.0
    pred.set_tensor(p_in, img[np.newaxis].astype(np.float32))
    pred.invoke()
    return pred.get_tensor(p_out).copy()

style_bn = get_bottleneck("starry_night.jpg")  # pre-compute

cam = Picamera2(); cam.start()
while True:
    t0 = time.perf_counter()
    frame = cam.capture_array()
    content = cv2.resize(frame[:,:,:3], (384,384))[np.newaxis] / 255.0
    xfm.set_tensor(c_in, content.astype(np.float32))
    xfm.set_tensor(s_in, style_bn)
    xfm.invoke()
    out = (xfm.get_tensor(x_out)[0] * 255).clip(0,255).astype(np.uint8)
    fps = 1 / (time.perf_counter() - t0)
    print(f"FPS: {fps:.1f}")
    cv2.imshow("NST", cv2.cvtColor(out, cv2.COLOR_RGB2BGR))
