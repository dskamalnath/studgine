LED Cube 3×3×3

Hand-solder 27 LEDs into a 3D cube and animate patterns via multiplexed layer scanning. A beautiful soldering and programming challenge that builds lasting circuit skills.

Home › Projects › LED Cube 3×3×3

Layer multiplexing → 3D pattern animations in a 3×3×3 voxel space

▶ Watch on YouTube

📖 Introduction

A 3×3×3 LED cube is one of the most visually impressive beginner builds — 27 LEDs forming a transparent 3D lattice that can display animations like rain, propellers, spirals, and layer scans. It's also a fantastic soldering exercise and teaches you the layer-multiplexing technique used in larger LED matrices and display panels.

The cube needs only 12 Arduino pins despite having 27 LEDs: 9 column wires (3×3 grid, sharing cathodes per column) and 3 layer wires (one per horizontal plane, sharing anodes per layer). By rapidly cycling through layers at >50 Hz and enabling the right column pins for each, the human eye perceives all LEDs as simultaneously lit — that's the multiplexing principle.

🧩 Components Needed

  • 🧠
    Arduino UnoController
  • 💡
    27× 5 mm LEDsAny colour — amber or blue look great
  • 🔌
    9× 100 Ω resistorsOne per column to limit current across 3 series LEDs
  • 🔌
    3× NPN transistors (2N2222)Layer enable switches (each layer draws up to 270 mA)
  • 🪛
    Soldering iron, solder, fluxFor building the cube frame
  • 📋
    Stripboard / PCB baseMounting the finished cube

⚙️ How It Works

  1. 1
    Build the cube structure — Bend all LED cathode (-) leads flat. Solder 9 LEDs into three 3×3 flat grids, connecting all cathodes in rows within each grid. Then stack and solder the three grids onto vertical anode (+) columns to form the cube. Each of the 9 vertical pillars shares one resistor-protected column wire. Each of the 3 horizontal layers shares one layer-enable wire to a transistor base.
  2. 2
    Wire layers via transistors — Layer anode planes connect to collector of 2N2222. Emitters → GND. Arduino D10, D11, D12 → 1 kΩ → transistor bases. Column wires go directly to Arduino D2–D10 via 100 Ω. Driving a layer is: pull that transistor base HIGH, then set column pins LOW for the LEDs you want ON (active-low, cathodes pulled to GND through resistors).
  3. 3
    Layer multiplexing in code — Store cube state as a 3×9 byte array: byte cube[3][9];. In a fast ISR or tight loop, cycle through each layer: disable all layers, set 9 column pins according to cube[layer][col], enable current layer transistor, delayMicroseconds(500);, then next layer. Refresh rate = 1/(3 × 500 µs) = ~667 Hz — well above the 50 Hz flicker threshold.
  4. 4
    Create animations — Programme in the main loop using non-blocking millis() timers. Classic effects: layer sweep (enable one layer at a time, scan through), corner to corner blink, spiral outward, "rain" (random column, move top-to-bottom), propeller (rotate 3-LED line in XZ plane). Each effect simply updates the cube[][] array and the ISR renders it.
