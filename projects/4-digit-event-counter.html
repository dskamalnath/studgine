4-Digit Event Counter

Count button presses or object crossings on a 4-digit 7-segment display — learn multiplexed display driving and integer arithmetic.

Each button press increments the count on the display

▶ Watch on YouTube

📖 Introduction

A 4-digit 7-segment display is one of the most rewarding components to drive from scratch. Each digit contains seven LED segments (a–g) that combine to form numbers 0–9. With four digits, you can show any value from 0000 to 9999 — perfect for a people counter, bottle counter, laps tracker, or production line tally.

The display is multiplexed: only one digit is actually lit at any moment, but cycling through all four faster than ~60 Hz tricks the eye into seeing them all on simultaneously. This technique applies to virtually every multi-digit display you'll ever use — LCD, LED, or OLED.

⚙️ How It Works

  1. 1
    Segment encoding — Create a byte array of size 10 where each element encodes which of the 7 segments to light for digits 0–9. Each bit maps to a segment pin (e.g. 0b01101101 = "5"). Connect segment pins a–g to Arduino digital pins 2–8 via 220 Ω resistors.
  2. 2
    Multiplex the 4 digits — Connect the 4 common-cathode pins to pins 10–13. In a displayNumber(int n) function, split n into thousands/hundreds/tens/units using integer division and modulo. Enable one digit at a time, write its segments, wait ~5 ms, then move to the next.
  3. 3
    Debounce the button — Read the count-up button on pin A0. Detect a LOW-to-HIGH edge, check that at least 50 ms have passed since the last press, then increment the counter variable. Check for 9999 overflow and reset to 0.
  4. 4
    Reset button — Wire a second button to A1. On press, set the counter to 0. Call displayNumber(counter) continuously in the main loop — the rapid multiplexing makes all four digits appear solid and bright.
