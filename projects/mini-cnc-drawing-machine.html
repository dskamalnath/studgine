📖 Introduction

A mini CNC (Computer Numerical Control) drawing machine — also known as an XY plotter — is one of the most satisfying intermediate builds. The core principle is simple: two stepper motors drive an X and Y carriage independently; a servo lifts and lowers the pen. By controlling all three axes in coordinated moves, you can draw anything a CAD program can describe.

The firmware is GRBL, an open-source CNC controller that runs entirely on an Arduino Uno and interprets industry-standard G-code over serial. The hardware cost is low (scavenged DVD-drive rails are popular for the linear slides), and the software toolchain is well-established: Inkscape with the G-code extension converts vector art to G-code, and Universal G-Code Sender (UGS) streams it to the Arduino. The result is a fully functional pen plotter for under $35.

🧩 Components Required

  • 🎛️
    Arduino Uno + CNC Shield V3 — GRBL firmware upload target; accepts 4× A4988 drivers
  • ⚙️
    2× NEMA 17 (or DVD stepper) + A4988 Drivers — X and Y axis movement, 1/16 microstepping for smoothness
  • 🔧
    SG90 Micro Servo — Pen lift/lower via M3/M5 spindle commands mapped in GRBL
  • 🏗️
    Linear Rail (DVD drive slides or MDF/acrylic) — Smooth XY travel; belt-drive via GT2 belt + 20T pulley or threaded rod
  • ⚡
    12 V 2 A Power Supply — Regulated 12 V for stepper motors; Arduino powered via USB from PC
  • 💾
    PC with Universal G-Code Sender + Inkscape — G-code generation and streaming via COM port at 115200 baud

⚙️ How It Works

  1. 1

    Flash GRBL Firmware to Arduino

    Download grbl-master from GitHub, add to Arduino IDE as a library, then upload "grblUpload" sketch. No custom code needed! GRBL configures all step/direction pins automatically. After upload, open UGS at 115200 baud, type $$ to view settings, and set $100/$101 steps/mm based on your pulley/belt geometry (GT2 belt + 20T pulley → 80 steps/mm at 1/16 micro).

  2. 2

    Map Pen Up/Down to Servo via Spindle

    GRBL's spindle enable pin (D12) drives the SG90 servo signal wire. In GRBL settings, enable spindle with $30=1000. In your G-code, M3 S90 (pen down) sets D12 PWM to ≈ 1.5 ms pulse, and M5 (pen up) sets 1 ms. Calibrate servo angles by adjusting S value so the pen just touches paper without pressing hard.

  3. 3

    Generate G-code from Vector Art

    In Inkscape, draw or import an SVG. Use the "Gcodetools" or "J Tech Photonics" extension to export G-code. Alternatively, use the web-based SVG-to-G-code converter at jscut.org. Key parameters: feed rate (F800, mm/min), pen-up height, and coordinate origin. GRBL's work coordinate system lets you set origin with G92 X0 Y0 before each job.

  4. 4

    Stream and Monitor via UGS

    Open the .gcode file in Universal G-Code Sender. The visualizer shows the toolpath in 3D before sending. Click "Send" to stream line-by-line. UGS buffers 16 lines ahead (RX buffer technique) for smooth motion. Watch for "ALARM:1" (soft limit hit) — set machine limits with $130/$131 to your actual travel in mm. Cancel mid-job with the Feed Hold button.

💻 Core Code Snippet

; Example G-code — draw a 60×60mm square on A4 paper
G21          ; Units in millimetres
G90          ; Absolute coordinates
G92 X0 Y0    ; Set origin at current position

M5           ; Pen UP (servo retract)
G0 X0 Y0     ; Rapid move to start corner (safe height)
M3 S90       ; Pen DOWN (servo deploy)

G1 X60 Y0  F800   ; Draw right edge  → (60, 0)
G1 X60 Y60        ; Draw top edge    → (60, 60)
G1 X0  Y60        ; Draw left edge   → (0,  60)
G1 X0  Y0         ; Draw bottom edge → back to origin

M5           ; Pen UP
G0 X0 Y0     ; Return home
M2           ; End program

; GRBL key settings ($$ command):
; $100 = 80.000 (X steps/mm, GT2+20T+1/16 micro)
; $101 = 80.000 (Y steps/mm same)
; $110 = 800.000 (X max feed rate mm/min)
; $130 = 200.000 (X max travel mm)
; $131 = 200.000 (Y max travel mm)
