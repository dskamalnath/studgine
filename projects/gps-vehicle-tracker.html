What You'll Build

A real-time GPS vehicle tracker using an ESP32 and a NEO-6M GPS module. The ESP32 parses NMEA sentences from the GPS receiver over hardware serial, extracts latitude, longitude, speed, and satellite count, then pushes the data to a Firebase Realtime Database every 5 seconds over Wi-Fi. A companion web page fetches the latest coordinates from Firebase and drops a marker on an embedded Google Maps iframe — updating the vehicle's position automatically without any page refresh.

How It Works

1

Parse NMEA Sentences

NEO-6M outputs NMEA 0183 data at 9600 baud. TinyGPS++ parses $GPRMC and $GPGGA sentences to extract lat/lng, speed, and fix quality.

2

Connect to Firebase

Using Firebase_ESP_Client, the ESP32 authenticates with an API key and writes a JSON payload: {"lat": 3.14, "lng": 101.68, "spd": 42.5} to the /vehicle/latest node.

3

Build the Map Page

A hosted HTML page uses the Firebase JS SDK to listen on the /vehicle/latest node via onValue(). When data changes, it calls map.setCenter() and updates the marker in under 1 second.

4

Geofence Alerts

Define a circular geofence with a centre lat/lng and radius in metres. Use the TinyGPS++ distanceBetween() method. When breached, trigger a push notification via Firebase Cloud Messaging.
