Satellite Attitude Control Sim

A complete Attitude Determination and Control System (ADCS) demonstration using a Raspberry Pi with LSM9DS1 magnetometer/gyroscope, running a full quaternion-based PD controller that commands three reaction wheel motors to maintain a target orientation — the exact same algorithm used in real CubeSat missions.

📖 Introduction

CubeSat attitude control is one of the most mathematically demanding aspects of small satellite engineering. Euler angles suffer from gimbal lock — quaternions provide stable 3D orientation representation without singularities. This project implements a hardware ADCS testbed: LSM9DS1 9-DOF IMU measures orientation; a Mahony/Madgwick filter fuses accel+gyro+mag to estimate quaternion attitude; a quaternion error PD controller computes required torques; three BLDC motor reaction wheels (one per axis) execute those torques. The physical platform can rotate freely on a gimbal mount.

🧩 Components Required

  • 🧲
    LSM9DS1 9-DOF IMU — Accel+Gyro+Magnetometer; I²C/SPI; 100Hz; Mahony filter for quaternion estimation; SparkFun or Adafruit breakout
  • ⚙️
    3× BLDC Motors + ESCs (Reaction Wheels) — 2204 2300KV motors; 3cm aluminium flywheel discs; 20A SimonK ESCs; one per X/Y/Z axis; bidirectional via BLHeli
  • 🧠
    Raspberry Pi 4 — Runs Mahony filter + quaternion PD controller at 200Hz; pwm via pigpio for ESC control; matplotlib 3D visualiser
  • 🔩
    3-Axis Air Bearing / Gimbal Mount — 3D-printed orthogonal frames on bearings; allows free rotation in all 3 axes; replicate zero-gravity orientation freedom

⚙️ How It Works

  1. 1

    Mahony Attitude Estimator

    Fuse accelerometer (gravity reference), gyroscope (rate integration), and magnetometer (yaw reference) to estimate quaternion orientation. Mahony complementary filter: vErr = cross(aEst, aNorm) + cross(mEst, mNorm); integralFB += Ki*vErr*dt; gyr += Kp*vErr + integralFB; q += 0.5*q⊗[0,gyr]*dt; q = q/|q|. Kp=2.0, Ki=0.005. Output: quaternion [q0,q1,q2,q3] representing current body orientation relative to Earth frame.

  2. 2

    Quaternion Error + PD Controller

    Target quaternion q_target (e.g., nadir-pointing). Error quaternion: q_err = q_target ⊗ q_current* (conjugate). Extract error axis-angle: theta = 2*arccos(q_err[0]); axis = q_err[1:]/sin(theta/2). PD control torques: τ = -Kp*(theta*axis) - Kd*(omega). Where omega is angular rate from gyro. Convert torques to reaction wheel speed commands: Ω_new = Ω_old + τ/(J_wheel)*dt. J_wheel: wheel moment of inertia (~5×10⁻⁵ kg·m²).

  3. 3

    ESC Reaction Wheel Control

    pigpio PWM for bidirectional ESC control: neutral = 1500μs pulse, full forward = 2000μs, full reverse = 1000μs. BLHeli ESCs in bidirectional mode. Map torque command to ESC pulse: pulse = 1500 + int(torque * 500 / MAX_TORQUE). Update at 50Hz (ESC refresh rate). Wheel speed feedback: optional Hall-effect sensor RPM counter. Saturation limits: max ±6000 RPM per wheel; desaturation algorithm fires magnetorquers when wheels near saturation (or use torque rods in simplified version).

  4. 4

    3D Visualisation + Telemetry

    matplotlib 3D body frame animation: draw CubeSat cube oriented by current quaternion. Real-time plots: quaternion components vs time, angular rate, wheel speeds, pointing error (degrees). Data logged at 200Hz to NumPy binary file for post-session analysis. Disturbance injection: simulate solar pressure torque = 0.001 N·m step at t=10s; observe controller response and settling time. Calculate settling time and steady-state pointing error for project report.

💻 Core Code Snippet

import numpy as np

def quat_multiply(q1, q2):
    """Hamilton quaternion product q1 ⊗ q2."""
    w1,x1,y1,z1 = q1; w2,x2,y2,z2 = q2
    return np.array([
        w1*w2-x1*x2-y1*y2-z1*z2,
        w1*x2+x1*w2+y1*z2-z1*y2,
        w1*y2-x1*z2+y1*w2+z1*x2,
        w1*z2+x1*y2-y1*x2+z1*w2])

def quat_error(q_target, q_current):
    return quat_multiply(q_target, np.array([q_current[0],-q_current[1],-q_current[2],-q_current[3]]))

Kp_att, Kd_att = 0.5, 0.1
J_wheel = 5e-5  # kg·m²

q_target = np.array([1.,0.,0.,0.])  # identity (nadir)
q_current = np.array([0.98,0.1,0.05,0.0]);  # initial perturbation
q_current /= np.linalg.norm(q_current)
omega = np.zeros(3)       # body angular rate rad/s
wheel_omega = np.zeros(3) # reaction wheel speeds rad/s
dt = 0.005  # 200 Hz

for step in range(2000):
    q_err = quat_error(q_target, q_current)
    angle  = 2*np.arccos(np.clip(q_err[0],-1,1))
    if abs(angle) > 1e-6:
        axis = q_err[1:] / (np.sin(angle/2)+1e-9)
    else:
        axis = np.zeros(3)
    torque = -Kp_att*(angle*axis) - Kd_att*omega
    wheel_omega += (torque / J_wheel) * dt
    omega_dot = -torque  # reaction to body
    omega += omega_dot * dt
    q_dot = 0.5*quat_multiply(q_current,[0,*omega])
    q_current += q_dot*dt; q_current/=np.linalg.norm(q_current)
