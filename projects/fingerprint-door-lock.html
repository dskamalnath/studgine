Fingerprint Door Lock

Enrol fingerprints into an R307 optical sensor — a matching scan unlocks a servo-driven deadbolt, teaching biometric UART communication and template management.

📖 Introduction

The Fingerprint Door Lock replaces a keypad or RFID card with your own biology. The R307 optical fingerprint sensor captures a 256×288 pixel image, extracts 64 characteristic templates, and compares them against up to 162 stored prints in flash — all via UART. The Arduino simply sends search commands and interprets the match score.

A match rotates a servo to 90° (unlocked) for 5 seconds, then reverts. Three failed attempts trigger a buzzer lockout. You'll learn UART biometric protocol, template enrolment workflow, servo timing, and security lockout logic — directly applicable to access control systems.

🧩 Components Required

  • 🎛️
    Arduino Uno / Nano — UART communication host
  • 👆
    R307 / R305 Optical Fingerprint Sensor — UART, 3.3 V logic, 300 dpi
  • ⚙️
    SG90 Servo Motor — Deadbolt rotation (0°–90°)
  • 🔊
    Piezo Buzzer — Access granted / denied tones
  • 🖥️
    16×2 I2C LCD — Status messages
  • 🔴
    Red + Green LEDs — Visual match / deny feedback
  • 🔌
    Breadboard + Jumper Wires — Level shifter if using 5→3.3 V

⚙️ How It Works

  1. 1

    Enrolment Mode

    Hold the ENROL button and place your finger twice. The Adafruit Fingerprint library's enrollStart(id), image2Tz(1), image2Tz(2), createModel(), and storeModel(id) sequence stores the fused template in sensor flash. Repeat for up to 20 users, incrementing the ID each time.

  2. 2

    Recognition Loop

    In normal operation, call getImage() → image2Tz() → fingerFastSearch(). Fast search scans all stored templates in <1 s and returns the best match ID and confidence score. Require confidence ≥ 50 to accept (0–255 scale).

  3. 3

    Unlock Servo & LED Feedback

    On match: rotate servo to 90°, light green LED, play 2-tone success melody, display "Welcome User N". Start a 5-second timer then re-lock (servo to 0°). On failure: red LED, error tone, and increment fail counter.

  4. 4

    Lockout & Logging

    After 3 consecutive failures, engage a 30-second lockout (servo stays locked, buzzer alarm). Log access events (user ID, timestamp from DS3231, result) to Serial for audit trail. Optional: upload events to EEPROM circular buffer.

💻 Core Code Snippet

#include <Adafruit_Fingerprint.h>
#include <Servo.h>
#include <SoftwareSerial.h>

SoftwareSerial mySerial(2, 3);
Adafruit_Fingerprint finger(&mySerial);
Servo lock;
int failCount = 0;

void loop() {
  int result = getFingerprintIDez();

  if (result >= 0) {
    // Valid match — unlock
    failCount = 0;
    lock.write(90);        // unlocked position
    digitalWrite(GREEN_LED, HIGH);
    tone(BUZZER, 1000, 200); delay(300); tone(BUZZER, 1500, 200);
    Serial.print("Access granted, User "); Serial.println(result);
    delay(5000);
    lock.write(0);         // re-lock
    digitalWrite(GREEN_LED, LOW);
  } else if (result == -2) {
    // No match
    failCount++;
    digitalWrite(RED_LED, HIGH);
    tone(BUZZER, 400, 500);
    if (failCount >= 3) {
      tone(BUZZER, 300, 5000); // alarm
      delay(30000);            // lockout period
      failCount = 0;
    }
    delay(1000);
    digitalWrite(RED_LED, LOW);
  }
}

int getFingerprintIDez() {
  if (finger.getImage() != FINGERPRINT_OK) return -1;
  if (finger.image2Tz() != FINGERPRINT_OK) return -1;
  if (finger.fingerFastSearch() != FINGERPRINT_OK) return -2;
  if (finger.confidence < 50) return -2;
  return finger.fingerID;
}
