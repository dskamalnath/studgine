Gait Abnormality Detector

An MPU-6050 IMU embedded in each shoe insole streams 200Hz gait kinematics over BLE to a Raspberry Pi that classifies limping, asymmetry, and fall risk in real time using a trained decision-tree model — a compelling medical wearable technology demonstration.

📖 Introduction

Gait analysis is used clinically to detect neurological disorders, orthopaedic injuries, and fall risk in elderly patients. Traditional gait labs cost thousands of pounds. This project creates an instrumented shoe insole using two MPU-6050 6-DOF IMUs (one per foot) inside 3D-printed or foam insole carriers. Data streams over nRF24L01 radio (or Bluetooth) to a central Raspberry Pi. Feature extraction computes step time symmetry, cadence, stride length, and heel-strike impact. A Random Forest trained on a labelled gait dataset classifies: normal, limping, shuffle (Parkinson's indicator), and high-fall-risk.

🧩 Components Required

  • 🦶
    2× MPU-6050 IMU (Insole Nodes) — 6-DOF accel+gyro; I²C; 200Hz data rate; each embedded in Arduino Pro Mini + nRF24L01 radio in insole
  • 📡
    2× nRF24L01 + Arduino Pro Mini — 2.4GHz; 250kbps; one per insole node; <2g total weight per insole; LiPo + TP4056 charger
  • 🧠
    Raspberry Pi 4 Receiver Hub — nRF24L01 receiver; scikit-learn Random Forest (300 trees); real-time feature computation + classification
  • 🖥️
    Gait Display Dashboard — matplotlib real-time step time plots; symmetry bar; classification label display; session CSV export

⚙️ How It Works

  1. 1

    Insole Node IMU Streaming

    Arduino Pro Mini reads MPU-6050 at 200Hz: Wire.beginTransmission(0x68); accel_x = readWord(0x3B); gyro_y = readWord(0x45);. Package as 12-byte struct [timestamp, ax, ay, az, gx, gy, gz]. Transmit via nRF24L01: radio.write(&data, sizeof(data)). Battery: 150mAh LiPo → ~6hr runtime. Insole weight: <15g per foot using printed TPU carrier. Heel-strike detection: vertical_accel_peak > 1.8g.

  2. 2

    Gait Feature Extraction

    Receive both insole streams; sync by timestamp. Detect step events (heel-strike peaks). Compute per-step features: step_time = time between peaks; step_length approximated from gyro integration; left/right step time symmetry = |tL - tR| / ((tL+tR)/2) * 100%. Stride: two consecutive same-side steps. Cadence: steps/minute. Vertical impact: peak accel at heel-strike. Collect sliding window of 20 steps → feature vector for classification.

  3. 3

    Random Forest Classification

    Train on labelled gait dataset (or simulate: normal person, person simulating limping, shuffling, etc.). Feature vector: [mean_step_time, symmetry%, cadence, mean_impact, stride_variance, stance_ratio]. Random Forest with 300 trees, max_depth=8. Test accuracy: ~91% on 4-class problem. Export: joblib.dump(model, 'gait_model.pkl'). Pi inference: model.predict([features])[0] — very fast (<1ms). Output: NORMAL / LIMPING / SHUFFLING / HIGH_FALL_RISK.

  4. 4

    Fall Risk Score + Alerting

    Fall risk score: weighted combination of symmetry%, stride variance, and mean_impact. Threshold >70% triggers haptic buzzer on wrist band (Arduino Nano BLE + vibration motor). Flask dashboard on Pi shows: step-time sequence bar chart, colour-coded symmetry plot, classification history over session. Export session to PDF report with summary statistics (useful for physiotherapy appointments).

💻 Core Code Snippet

# Raspberry Pi receiver + classifier
import struct, numpy as np
from RF24 import RF24, RF24_PA_LOW
import joblib

radio = RF24(22, 0)  # CE=GPIO22, CSN=SPI0
radio.begin(); radio.openReadingPipe(1, 0xF0F0F0F0E1)
radio.setPALevel(RF24_PA_LOW); radio.startListening()

model = joblib.load('gait_rf_model.pkl')
scaler = joblib.load('gait_scaler.pkl')
step_times = {'L':[], 'R':[]}
prev_peak = {'L':0,'R':0}

def extract_features(steps_L, steps_R):
    if len(steps_L)<5 or len(steps_R)<5: return None
    tL, tR = np.mean(steps_L[-10:]), np.mean(steps_R[-10:])
    symm = abs(tL-tR)/((tL+tR)/2)*100
    cadence = 60/(np.mean(steps_L[-5:]+steps_R[-5:]))
    variance = np.std(steps_L[-10:]+steps_R[-10:])
    return [tL, tR, symm, cadence, variance]

while True:
    if radio.available():
        payload = radio.read(12)
        foot, ts, az = struct.unpack('Bff', payload[:9])
        side = 'L' if foot==0 else 'R'
        if az > 1.8 and ts - prev_peak[side] > 0.3:
            step_times[side].append(ts - prev_peak[side])
            prev_peak[side] = ts
        feats = extract_features(step_times['L'], step_times['R'])
        if feats:
            x = scaler.transform([feats])
            pred = model.predict(x)[0]
            print(f"Gait: {pred} (symmetry={feats[2]:.1f}%)")
