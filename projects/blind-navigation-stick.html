Blind Navigation Stick

A smart cane attachment with HC-SR04 ultrasonic sensors that detects obstacles at knee, waist, and overhead heights. Provides escalating haptic vibration and audio beeps that increase in frequency as the user approaches an object.

📖 Introduction

Over 43 million people worldwide are blind, and hundreds of millions more have significant visual impairment. Standard white canes detect ground-level obstacles well but miss suspended objects like car wing mirrors, tree branches, or shop signs at head height. This ultrasonic stick attachment addresses that gap with three sensors at different angle profiles, providing full-height obstacle detection in a 3-metre forward arc.

This project is deeply impactful as an engineering design exercise — students must balance battery life, response time, feedback legibility (distinguishing vibration patterns), waterproofing, weight, ergonomics, and cost. An excellent capstone or final-year project topic.

🧩 Components Required

  • 📡
    3× HC-SR04 Ultrasonic Sensors — Angled at ground, waist, head heights; 2 cm–4 m range, ±3 mm accuracy
  • 📳
    3× Small Haptic Vibration Motors (ERM) — One per sensor; mounted at hand grip position for tactile distance feedback
  • 🔊
    Passive Buzzer / Mini Speaker — Distance-varying beep pattern; optional pre-recorded voice with DFPlayer Mini
  • 🔧
    Arduino Nano — Compact form factor; fits inside stick mounting bracket
  • 🔋
    18650 Li-Ion Cell + TP4056 — 3000 mAh gives ~12 h runtime; USB-C charging port on stick body
  • 🎙️
    DFPlayer Mini (Optional) — Plays MP3 voice prompts: "Obstacle at 50 cm"; stores audio on microSD card

⚙️ How It Works

  1. 1

    Multi-Sensor Distance Measurement

    Read three HC-SR04 sensors sequentially (trigger sensors one at a time to avoid crosstalk). Each reading: 10 µs TRIG pulse → measure ECHO pulse duration → distance = duration × 0.0343 / 2. Allow 60 ms between sensor reads to prevent ultrasonic interference. Takes 5 readings per sensor per cycle and uses the median value to reject spike noise from reflective surfaces.

  2. 2

    Tiered Proximity Feedback

    Three distance zones produce different feedback intensities: FAR (100–300 cm): single slow beep every 1 s, vibration motor 1 (low intensity PWM 80). NEAR (40–100 cm): rapid beep every 300 ms, all three motors at medium intensity. DANGER (<40 cm): continuous tone, all motors at maximum (255 PWM). Different motors for different sensors let the user feel which height zone has an obstacle — crucial for informed navigation decisions.

  3. 3

    Non-Blocking Haptic & Audio Patterns

    Use millis()-based non-blocking timers for all feedback patterns — never delay(). This ensures sensor readings continue even while producing feedback. Implement pattern struct: {onTime, offTime, repeats}. Example: three-pulse emergency = {50ms, 50ms, 3×}. This keeps the feedback loop running at 20 Hz update rate regardless of audio/vibration patterns.

  4. 4

    Water Detection Mode

    Optional: add a water/puddle sensor (two copper tape strips on stick base). On water detection (conductivity closes circuit): trigger special fast vibration pattern + rising tone alert — a pattern distinctly different from obstacle alerts. This warns the user to step differently (raise foot higher) rather than stop, distinguishing wet surface from solid obstacle. Store pattern preferences in EEPROM for custom user configurations.

💻 Core Code Snippet

#define TRIG1 2 #define ECHO1 3  // low (ground)
#define TRIG2 4 #define ECHO2 5  // mid (waist)
#define TRIG3 6 #define ECHO3 7  // high (head)
#define VIB1 9 #define VIB2 10 #define VIB3 11
#define BUZZER 8

long readDistance(int trig, int echo) {
  digitalWrite(trig, LOW); delayMicroseconds(2);
  digitalWrite(trig, HIGH); delayMicroseconds(10);
  digitalWrite(trig, LOW);
  return pulseIn(echo, HIGH, 23200) * 0.0343 / 2;
}

void alertProximity(int vibPin, long dist) {
  int intensity = 0;
  if(dist < 40)        intensity = 255;
  else if(dist < 100)  intensity = 150;
  else if(dist < 200)  intensity = 60;
  analogWrite(vibPin, intensity);
  if(intensity > 150) tone(BUZZER, 1000 + (200-dist)*5);
  else if(intensity > 0) tone(BUZZER, 600);
  else noTone(BUZZER);
}

void loop() {
  long d1 = readDistance(TRIG1, ECHO1);
  delay(30);
  long d2 = readDistance(TRIG2, ECHO2);
  delay(30);
  long d3 = readDistance(TRIG3, ECHO3);
  alertProximity(VIB1, d1);
  alertProximity(VIB2, d2);
  alertProximity(VIB3, d3);
  delay(10);
}
