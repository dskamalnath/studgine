LiDAR Obstacle Mapper

Mount an RPLIDAR A1 on a Raspberry Pi 4 differential-drive robot and run ROS Noetic Hector SLAM to build a live 2D occupancy grid map of your environment in real time — a genuine entry point into autonomous robotics navigation.

📖 Introduction

SLAM (Simultaneous Localisation and Mapping) lets a robot build a map of an unknown environment while tracking its own position within that map. The RPLIDAR A1 performs 360° laser ranging at 10 Hz, producing 360 distance measurements per rotation. ROS (Robot Operating System) provides the software framework: drivers publish sensor_msgs/LaserScan messages, Hector SLAM subscribes and incrementally updates a 2D occupancy grid map (grey = unknown, white = free, black = occupied).

The occupancy map can then drive autonomous navigation (ROS Navigation Stack with move_base) for obstacle avoidance and goal-directed movement — a complete mobile robotics stack.

🧩 Components Required

  • 🔴
    RPLIDAR A1 (360° 2D LiDAR) — 6m range; 10Hz; 360 points/scan; USB serial; SLAMTEC SDK + ROS driver package
  • 🤖
    Raspberry Pi 4 (4 GB) Robot Base — Ubuntu 20.04 Server + ROS Noetic; differential-drive robot base; 2× encoder motors; L298N driver
  • 🧭
    IMU MPU-6050 (optional for Hector) — Hector SLAM works without odometry; IMU improves map consistency during fast rotations
  • 💻
    RViz Remote Visualisation PC — Same ROS network; RViz displays occupancy grid, robot pose, laser scan overlay in real time over WiFi

⚙️ How It Works

  1. 1

    ROS Setup + RPLIDAR Driver

    Install ROS Noetic on Ubuntu 20.04. Install driver: sudo apt install ros-noetic-rplidar-ros. Launch: roslaunch rplidar_ros rplidar.launch. This publishes /scan topic as sensor_msgs/LaserScan with 360 ranges. Verify with: rostopic echo /scan | head -30. Check scan frame: set frame_id=laser. Install hector SLAM: sudo apt install ros-noetic-hector-slam.

  2. 2

    Hector SLAM Configuration

    Edit hector_slam.launch: set odom_frame = laser (scan-only SLAM, no wheel odometry required). Key parameters: map_resolution=0.05 (5cm/cell), map_size=1024 (51.2m map), map_update_distance_thresh=0.4. Launch: roslaunch hector_slam_launch tutorial.launch. The algorithm correlates each incoming scan to the current map to estimate rigid-body transform (Gauss-Newton scan-matching).

  3. 3

    RViz Visualisation + Map Save

    On remote PC: export ROS_MASTER_URI=http://raspberrypi.local:11311. Launch RViz: add Map display (topic /map), LaserScan display (topic /scan), TF display. Colour map: grey=unknown, white=free, black=occupied. Drive robot around room to build map. Save: rosrun map_server map_saver -f my_room → creates my_room.pgm + my_room.yaml.

  4. 4

    Autonomous Navigation with Saved Map

    Load saved map: rosrun map_server map_server my_room.yaml. Launch AMCL (Adaptive Monte Carlo Localisation): estimates robot pose within known map via particle filter. Launch move_base: global planner (A* on occupancy grid) + local planner (TEB) computes collision-free paths to /move_base_simple/goal poses clicked in RViz. Close the full autonomous navigation loop from LiDAR scan → SLAM map → AMCL → path planning → motor commands.

💻 Core Code Snippet

#!/usr/bin/env python3
# ROS node: navigate to a GPS-style goal
import rospy
from geometry_msgs.msg import PoseStamped
from nav_msgs.msg import OccupancyGrid
import numpy as np

rospy.init_node('goal_sender')
pub = rospy.Publisher('/move_base_simple/goal', PoseStamped, queue_size=1)
rospy.sleep(1.0)

def send_goal(x, y, yaw_deg=0):
    goal = PoseStamped()
    goal.header.frame_id = "map"
    goal.header.stamp = rospy.Time.now()
    goal.pose.position.x = x
    goal.pose.position.y = y
    # Simple yaw to quaternion
    import math
    goal.pose.orientation.z = math.sin(math.radians(yaw_deg)/2)
    goal.pose.orientation.w = math.cos(math.radians(yaw_deg)/2)
    pub.publish(goal)
    rospy.loginfo(f"Sent goal: ({x:.2f}, {y:.2f})")

# Waypoint tour
waypoints = [(1.0, 0.5, 0), (2.0, 1.0, 90), (0.0, 0.0, 180)]
for wp in waypoints:
    send_goal(*wp)
    rospy.sleep(8.0)  # allow time to reach each waypoint
