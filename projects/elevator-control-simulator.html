Elevator Control Simulator

Simulate a 3-floor elevator with floor-call buttons, direction indicator LEDs, and a stepper motor–driven platform that services requests using the SCAN (elevator) algorithm.

📖 Introduction

The elevator (SCAN) algorithm is a classic operating-systems scheduling topic that maps directly to a physical hardware build. The key insight: instead of servicing floor calls in first-come-first-served order (which causes starvation), the elevator sweeps in one direction servicing all pending floors, then reverses — just like a real lift. This minimises total stepper motor travel distance.

This project uses a 28BYJ-48 unipolar stepper motor with ULN2003 driver to physically move a small platform representing the elevator cabin between three floors defined by limit switches or optical encoders. Six push buttons (up/down per floor) trigger floor calls, while a 16×2 LCD shows current floor, direction, and pending queue. It's an excellent data-structures + embedded-systems crossover project.

🧩 Components Required

  • ⚙️
    28BYJ-48 Stepper + ULN2003 Driver — 64 steps/rev × 64:1 gearbox = 4096 steps/rev; 5 V; perfect for slow, precise platform movement
  • 🧠
    Arduino Uno — Runs SCAN scheduler, reads buttons, drives stepper + LEDs + LCD
  • 🔴
    6× Push Buttons — 2 per floor (up + down call); cabin has 3 destination buttons; total 9 buttons including lobby
  • 📏
    3× IR Slot Sensors (H206) — Optical end-stop per floor; reads the platform vane position for absolute floor detection
  • 🖥️
    16×2 I2C LCD — Displays current floor, direction (▲▼), pending call queue
  • 💡
    6× LEDs (Green × 3, Red × 3) — Per-floor indicator: green = cabin here, red = pending call

⚙️ How It Works

  1. 1

    Floor Homing & Step Counting

    On power-up, the motor drives downward until the floor-1 IR sensor triggers — this is the home (reference) position. Absolute floor positions are stored as step counts from home: floor 1 = 0, floor 2 = 1365 steps, floor 3 = 2730 steps (adjust for your physical build). A running currentStep counter tracks position. IR sensors provide correction every time the platform actually reaches a floor, preventing accumulated step errors from gear slip.

  2. 2

    SCAN Scheduling Algorithm

    Maintain a sorted queue of pending floor requests. SCAN rule: while moving UP, only service floors ≥ currentFloor. When no more requests above, reverse direction and service floors ≤ currentFloor. Implementation: int queue[3]; bool pending[4]. When a button is pressed, set pending[floor] = true. In the scheduler: scan upward through pending[] for the next floor in direction, or reverse if none found. Queue display: render on LCD as e.g. "Queue: 2,3" with arrow direction.

  3. 3

    Stepper Motor Movement

    Use the Stepper library: Stepper motor(STEPS, 8, 10, 9, 11). Set speed to 8 RPM (slow enough for realistic simulation). To move between floors, calculate stepsNeeded = (targetFloor - currentFloor) * STEPS_PER_FLOOR. Call motor.step(stepsNeeded) (blocks until complete; use non-blocking AccelStepper for a more realistic gradual acceleration profile). At arrival: turn off stepper coils (digitalWrite all coil pins LOW) to prevent overheating when stationary.

  4. 4

    Door Open Simulation & Safety Delays

    At each stop: flash the floor LED for 3 seconds (simulates door open/close time). During this window, accept new button presses to add to queue but do not move. If a destination on the same floor as the next queued stop is pressed while doors are "open", service it immediately (door-hold). After 3 seconds, close doors (stop LED flash) and immediately compute the next SCAN target. Buzzer: single beep on floor arrival, double beep on direction change.

💻 Core Code Snippet

#include <Stepper.h>
#include <LiquidCrystal_I2C.h>

const int STEPS_PER_FLOOR = 1365;
Stepper motor(4096, 8, 10, 9, 11);
LiquidCrystal_I2C lcd(0x27, 16, 2);

int currentFloor = 1, direction = 1; // 1=UP, -1=DOWN
bool pending[4] = {false,false,false,false}; // index 1..3

const int BTN[4] = {0,2,3,4}; // pins for floors 1,2,3

void setup() {
  motor.setSpeed(8); lcd.begin();
  for(int i=1;i<=3;i++) pinMode(BTN[i], INPUT_PULLUP);
  lcd.print("F1 IDLE"); 
}

int nextFloor() {
  // SCAN: look in current direction first
  if(direction == 1) {
    for(int f=currentFloor+1;f<=3;f++) if(pending[f]) return f;
    direction=-1;
    for(int f=currentFloor-1;f>=1;f--) if(pending[f]) return f;
  } else {
    for(int f=currentFloor-1;f>=1;f--) if(pending[f]) return f;
    direction=1;
    for(int f=currentFloor+1;f<=3;f++) if(pending[f]) return f;
  }
  return 0; // no pending
}

void loop() {
  for(int i=1;i<=3;i++) if(digitalRead(BTN[i])==LOW) { pending[i]=true; delay(50); }
  int target = nextFloor();
  if(target) {
    int steps = (target - currentFloor) * STEPS_PER_FLOOR;
    motor.step(steps);
    currentFloor = target; pending[target]=false;
    lcd.clear(); lcd.print("Floor "+String(target));
    delay(3000); // door open
  }
}
