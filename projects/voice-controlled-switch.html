What You'll Build

An entirely offline voice-controlled relay system. You record ~50 samples each of short commands ("lights on", "lights off", "fan on", "fan off") using Edge Impulse Studio's browser-based data collector. A DSP block computes MFCC features from 1-second audio windows; a small neural network (Dense, 20 neurons) is trained and quantised to an INT8 TFLite model that fits in the Nano BLE's 256 KB SRAM. The exported Arduino library is included in the sketch; inference runs every 200 ms with no Wi-Fi or cloud dependency.

How It Works

1

Collect Training Data

In Edge Impulse Studio, create a project, connect the Nano BLE, and record 50+ 1-second samples for each keyword ("lights on", "lights off", "fan on", "fan off", "unknown", "noise") using the built-in PDM microphone.

2

Train TFLite Model

Set an MFCC DSP block (13 coefficients, 0.02 s frame, 0.02 s stride) then an INT8-quantised Dense NN. Train for 100 epochs; target >93 % accuracy. Export as an Arduino library ZIP.

3

Run Inference

In the Arduino sketch, call ei_microphone_inference_start(), fill the buffer, then run_classifier(). The highest-probability label above CONFIDENCE_THRESHOLD = 0.85 triggers a relay action.

4

Drive Relay

Map each confirmed label to a relay pin action: toggle state on "on/off" commands. A confirmed-count debounce (require 2 consecutive positive detections) prevents false triggers from noise.
