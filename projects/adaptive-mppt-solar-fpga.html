Adaptive MPPT Solar Controller (FPGA)

Implement Maximum Power Point Tracking entirely in VHDL on a Lattice iCE40UP5k FPGA. The Perturb-and-Observe algorithm runs at 1MHz, reading solar panel voltage and current from an INA219 over I²C. It adjusts PWM duty cycle of a synchronous buck converter to keep the panel at its peak power point — using fully open-source tools: Yosys, nextpnr, icepack, and iceprog.

📖 Introduction

Maximum Power Point Tracking is the algorithm inside every solar charge controller. A solar panel's P-V (Power-Voltage) curve has a unique maximum power point that moves with irradiance and temperature. MPPT tracks this peak by continuously perturbing the operating voltage and observing if power increases or decreases — adjusting duty cycle accordingly. Implementing this in VHDL on an FPGA is a classic digital design exercise: you learn synchronous state machines, fixed-point arithmetic, SPI/I²C bus masters, and PWM generation — all in hardware with deterministic 1MHz timing unachievable on a microcontroller with interrupt latency.

  • ☀️

  • 🔧
    Sync Buck Converter (LM2596 or TPS40057) — External PWM input; MOSFET switch; inductor 47μH; 100μF output cap; FPGA GPIO drives gate via logic-level MOSFET buffer (3.3V→5V level shift)

⚙️ How It Works

  1. 1

    FPGA PWM Generation Module

    VHDL entity with configurable duty cycle register. 12-bit PWM at 244kHz switching frequency (48MHz clock / 196 counter): process(clk) begin if rising_edge(clk) then if counter < duty then pwm_out <= '1'; else pwm_out <= '0'; end if; counter <= counter + 1; end if; end process;. Duty cycle register updated by MPPT controller every 1ms (1kHz MPPT loop, 244kHz PWM — independent clock domains). Minimum duty step: 1/4096 = 0.024% for fine-grained tracking.

  2. 2

    I²C Master for INA219

    Implement I²C master state machine in VHDL: states IDLE → START → ADDR → ACK_ADDR → DATA_MSB → ACK_DATA → DATA_LSB → STOP. Clock: 400kHz I²C from 48MHz system clock (divide by 120). INA219 register 0x01 = shunt voltage (current), register 0x02 = bus voltage. Fixed-point conversion: voltage = bus_reg * 4mV; current = shunt_reg * 10μV / 0.1Ω. Power computed in FPGA: power = voltage * current (16×16 multiplier).

  3. 3

    Perturb-and-Observe Algorithm in VHDL

    Synchronous state machine: PERTURB → MEASURE → OBSERVE → ADJUST. PERTURB: duty_cycle += step (+/-). MEASURE: wait 1ms for transient to settle; read INA219 power. OBSERVE: compare new_power vs prev_power. ADJUST: if new_power > prev_power: perturb in same direction; else: reverse direction. VHDL: if new_power > prev_power then direction <= direction; else direction <= not direction; end if; prev_power <= new_power; duty_cycle <= duty_cycle + (step when direction='1' else -step);. Typical step size: 32/4096 = 0.78% duty cycle.

  4. 4

    Open-Source Toolchain Build

    Synthesis: yosys -p "synth_ice40 -top mppt_top -json mppt.json" mppt_top.vhd i2c_master.vhd pwm_gen.vhd. Place-and-route: nextpnr-ice40 --up5k --package sg48 --json mppt.json --asc mppt.asc --pcf pins.pcf. Bitstream pack: icepack mppt.asc mppt.bin. Program FPGA: iceprog mppt.bin. Pin constraint file (up5k pins): map I²C SDA/SCL, PWM_OUT, LED indicators to physical package pins. Simulation: GHDL + GTKWave to verify MPPT FSM logic before hardware.

💻 Core Code Snippet

-- VHDL: Perturb-and-Observe MPPT Controller (simplified)
library ieee; use ieee.std_logic_1164.all; use ieee.numeric_std.all;

entity mppt_controller is
  port (
    clk      : in  std_logic;
    rst_n    : in  std_logic;
    power    : in  unsigned(23 downto 0);   -- from INA219 (mW)
    duty_out : out unsigned(11 downto 0)    -- 12-bit PWM duty
  );
end entity;

architecture rtl of mppt_controller is
  constant STEP      : unsigned(11 downto 0) := to_unsigned(32, 12);
  signal duty        : unsigned(11 downto 0) := to_unsigned(2048, 12);
  signal prev_power  : unsigned(23 downto 0) := (others => '0');
  signal direction   : std_logic := '1';
  signal timer       : unsigned(15 downto 0) := (others => '0');
begin
  duty_out <= duty;

  process(clk, rst_n) begin
    if rst_n = '0' then duty <= to_unsigned(2048,12); prev_power <= (others=>'0');
    elsif rising_edge(clk) then
      timer <= timer + 1;
      if timer = 0 then  -- 1kHz update (48000 cycles at 48MHz)
        if power > prev_power then
          -- P increased: continue same direction
          if direction = '1' then duty <= duty + STEP; else duty <= duty - STEP; end if;
        else
          -- P decreased: reverse direction
          direction <= not direction;
          if direction = '0' then duty <= duty + STEP; else duty <= duty - STEP; end if;
        end if;
        prev_power <= power;
      end if;
    end if;
  end process;
end architecture;
