What You'll Build

A Wi-Fi security camera using the $5 ESP32-CAM module. The built-in OV2640 camera is initialised via the esp_camera library with SVGA resolution (800×600). An AsyncWebServer serves an MJPEG stream endpoint at /stream — any browser on the same network can view the live feed. A simple frame-differencing algorithm in the ESP32 detects motion, saves a JPEG snapshot to the onboard microSD, and sends an email alert with the image attached using SMTP over TLS (ESP_Mail_Client library).

How It Works

1

Camera Initialisation

Configure camera_config_t with XCLK, PCLK, VSYNC, HSYNC, D0–D7 pin mappings. Set PIXFORMAT_JPEG and FRAMESIZE_SVGA. Use esp_camera_init() — allow 500 ms warmup for auto-exposure.

2

MJPEG Stream Server

In the /stream handler, write HTTP multipart headers then loop: capture frame with esp_camera_fb_get(), write JPEG boundary, write frame buffer, call esp_camera_fb_return(). Achieves 10–15 fps on good Wi-Fi.

3

Motion Detection

Downsample every frame to 20×15 grayscale. Compute pixel-difference sum vs previous frame. If sum exceeds threshold, flag motion, save JPEG to SD with timestamp filename, and arm the email sender.

4

Email Alert with Snapshot

Use ESP_Mail_Client to authenticate with Gmail SMTP (port 465, SSL). Attach the saved JPEG and send "Motion detected at [timestamp]" subject with a 60-second cooldown between alerts.
