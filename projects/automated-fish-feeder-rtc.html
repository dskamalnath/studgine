Automated Fish Feeder (RTC)

Schedule up to 4 precision feedings per day using a DS3231 RTC, a servo-driven auger mechanism, and EEPROM-stored feeding times — keeps your aquarium fish fed while you're away.

📖 Introduction

Unlike simple interval timers that drift over days, the DS3231 is a temperature-compensated real-time clock accurate to ±2 ppm — roughly ±1 minute per year. This is essential for aquarium feeding where fish need consistent, predictable meal times for health and digestion.

The auger mechanism is the key upgrade over beginner drum-sweep feeders. A servo rotates a helical auger inside a PVC tube loaded with pellet food. The auger pushes a consistent, measured amount of food to the dispensing outlet each rotation — far more reliable than a drum with a slot, which can jam or overdose. Feeding schedules and portion sizes are stored in EEPROM so they survive power cuts.

🧩 Components Required

  • 🎛️
    Arduino Uno / Mega — Main controller for RTC, servo, LCD, and buttons
  • ⏰
    DS3231 RTC Module — I2C, ±2 ppm accuracy, coin cell backup
  • 🔩
    MG996R High-Torque Servo — Rotates auger — needs high torque for dry pellets
  • 📟
    20×4 LCD with I2C — Shows time, schedule, next meal countdown
  • 🔘
    Tactile Buttons ×4 — Set hour, minute, confirm, manual feed override
  • 🧱
    PVC Pipe + Auger (3D printed) — Food container and helical dispensing auger

⚙️ How It Works

  1. 1

    Read RTC Time

    Use the RTClib library to get a DateTime object every second. Display hour:minute:second on the LCD first row. On first boot, check if DS3231 has lost power — if so, prompt the user to set the current time using the 4-button interface.

  2. 2

    Store Schedule in EEPROM

    Allow up to 4 feeding times. Each slot takes 2 bytes (hour + minute) stored at EEPROM addresses 0–7. An 8th byte stores a bitmask of enabled slots. Use the button UI to browse and edit slots; call EEPROM.update() only when values change to preserve EEPROM write cycles.

  3. 3

    Trigger the Auger

    Every second, compare current H:M with each enabled schedule slot. On a match, activate the servo for portionRevolutions full rotations (typically 2–4). Use servo.write(180) for one direction then servo.write(0) to stop — or use continuous rotation servo mode for constant-turn control.

  4. 4

    Manual Feed and Countdown

    A dedicated "Feed Now" button triggers an immediate single portion for testing. The LCD second row counts down to the next scheduled feeding. Log each feeding event to EEPROM with a running counter so you can verify consistency on the Serial Monitor.

💻 Core Code Snippet

#include <Wire.h>
#include <RTClib.h>
#include <Servo.h>
#include <EEPROM.h>
#include <LiquidCrystal_I2C.h>

RTC_DS3231 rtc;
Servo auger;
LiquidCrystal_I2C lcd(0x27, 20, 4);

#define AUGER_PIN   9
#define NUM_SLOTS   4
#define EEPROM_BASE 0

struct FeedSlot { uint8_t hour; uint8_t minute; bool enabled; };
FeedSlot schedule[NUM_SLOTS];

void loadSchedule() {
  for (int i = 0; i < NUM_SLOTS; i++) {
    schedule[i].hour    = EEPROM.read(EEPROM_BASE + i * 3);
    schedule[i].minute  = EEPROM.read(EEPROM_BASE + i * 3 + 1);
    schedule[i].enabled = EEPROM.read(EEPROM_BASE + i * 3 + 2);
  }
}

void dispense(int revolutions) {
  auger.attach(AUGER_PIN);
  for (int r = 0; r < revolutions; r++) {
    auger.write(180); delay(600);
    auger.write(90);  delay(100);  // brief pause between revolutions
  }
  auger.detach();  // prevent jitter when idle
}

void setup() {
  rtc.begin();
  if (rtc.lostPower()) rtc.adjust(DateTime(F(__DATE__), F(__TIME__)));
  loadSchedule();
  lcd.init(); lcd.backlight();
  auger.attach(AUGER_PIN);
}

void loop() {
  DateTime now = rtc.now();
  lcd.setCursor(0,0);
  lcd.print(now.timestamp(DateTime::TIMESTAMP_FULL));
  for (int i = 0; i < NUM_SLOTS; i++) {
    if (schedule[i].enabled &&
        now.hour()   == schedule[i].hour &&
        now.minute() == schedule[i].minute &&
        now.second() == 0) {
      dispense(3);  // 3 auger revolutions = one portion
      lcd.setCursor(0,1); lcd.print("FED " + String(i+1) + " @ " +
        String(schedule[i].hour) + ":" + String(schedule[i].minute));
    }
  }
  delay(900);
}
