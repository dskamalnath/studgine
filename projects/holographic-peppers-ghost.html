Holographic Pepper's Ghost Display

A Raspberry Pi renders four rotated copies of a 3D wireframe animation on a black background, displayed on an HDMI screen laying flat. A hand-cut or 3D-printed 45° Mylar reflector pyramid placed on top of the screen reflects each quadrant inward — creating the illusion of a glowing 3D hologram floating in mid-air, from any viewing angle.

📖 Introduction

Pepper's Ghost is a 19th-century illusion technique used in Victorian theatres to make ghosts appear on stage. A 45° semi-transparent reflector causes light from a hidden display to appear to float in free space. This project modernises it: a Raspberry Pi renders a 3D wireframe object (rotating DNA helix, Earth globe, or custom 3D model) using PyOpenGL or Pygame/Pyglet, splits it into four rotated quadrants on screen (front/back/left/right views), and displays on a 1080p HDMI tablet or monitor laid flat. A 20cm Mylar pyramid placed over the screen creates a convincing free-floating hologram visible from all sides.

🧩 Components Required

  • 🖥️
    10" HDMI LCD Screen (IPS, placed flat) — 1920×1080; high brightness (>300 nits); black background essential for illusion quality; flat bezel-less preferred
  • 💎
    Mylar / PET Reflector Pyramid — 20×20cm base; 45° walls; laser-cut 0.5mm Mylar film or 3D-printed frame + polished PET sheet; 5cm apex aperture
  • 🧠
    Raspberry Pi 4 (4 GB) — PyOpenGL + GLFW for 3D rendering; OpenGL ES 3.0 on VideoCore VI GPU; 60fps at 1080p; hardware texture units
  • 🌑
    Dark Room / Enclosure — Surround pyramid with black card enclosure; room ambient light < 10 lux for maximum hologram visibility; optional RGB accent lighting

⚙️ How It Works

  1. 1

    3D Wireframe Rendering with PyOpenGL

    Render 3D object (DNA double helix, rotating sphere, wireframe cube) with PyOpenGL + GLFW on Pi GPU. The key is all content on pure black (RGB 0,0,0) background — anything black becomes invisible in the Mylar reflection while coloured wireframes appear to glow. Use additive blending: glBlendFunc(GL_SRC_ALPHA, GL_ONE) for glowing neon wire effect. Geometry: DNA helix = parametric helix points connected with GL_LINE_STRIP: x=r*cos(t), y=b*t/(2π), z=r*sin(t).

  2. 2

    Quad-Split Screen Layout

    The 1080×1080 display area is divided into 4 quadrants. Each quadrant gets one rotated view: top quadrant = front view, bottom = back view (180° rotated), left = left side view (90° CCW), right = right side view (90° CW). Render to FBO (framebuffer object), then blit to each quadrant with appropriate rotation transform. For simple 2D implementation: use Pygame to blit 4 rotated copies of the rendered image: pygame.transform.rotate(frame, angle) at offsets (W//4, 0), (W//4, H//2), (0, H//4), (W//2, H//4).

  3. 3

    Pyramid Physics: Why It Works

    When you look at a 45° mirror, you see a virtual image of whatever is below/beside it. The Mylar panel reflects the light from the screen quadrant toward you at 90°. Your eye perceives the image as floating at the pyramid's apex centre. Four panels create four views (front, back, left, right) simultaneously — creating the 360° holographic illusion. Key: Mylar semi-transparency means you see the virtual image floating inside the transparent pyramid structure itself, not reflected from a surface.

  4. 4

    Content Creation + Animation Loop

    Animation loop: update rotation angle each frame, re-render 3D scene. For variety: DNA double helix (biology demo), wireframe Earth globe (geography), 3D snowflake (seasonal), custom ASCII name in 3D neon, anatomical heart model (OBJ file loader). Add audio: play ambient drone sound or heartbeat synced to animation. Touchscreen interactivity: touch to change displayed object. Suitable for science fair, museum exhibit, or product launch demo — extremely high visual impact for low cost.

💻 Core Code Snippet

import pygame, math, sys
pygame.init()
W, H = 800, 800  # square display
screen = pygame.display.set_mode((W, H), pygame.FULLSCREEN)
pygame.display.set_caption("Hologram")

def draw_helix(surface, cx, cy, angle, color, scale=1.0):
    """Draw a rotating DNA double helix wireframe."""
    pts1, pts2 = [], []
    for i in range(60):
        t = i / 60 * 4 * math.pi
        # Rotate around Y axis by 'angle'
        x = 0.3 * math.cos(t + angle)
        y = (i - 30) / 30.0  # -1 to 1
        z = 0.3 * math.sin(t + angle)
        # Simple perspective projection
        fov = 3.0; d = fov / (fov + z)
        sx = int(cx + x * d * 80 * scale)
        sy = int(cy + y * d * 100 * scale)
        pts1.append((sx, sy))
        # Complementary strand (offset by π)
        x2 = 0.3 * math.cos(t + angle + math.pi)
        z2 = 0.3 * math.sin(t + angle + math.pi)
        d2 = fov / (fov + z2)
        pts2.append((int(cx + x2*d2*80*scale), int(cy + y*d2*100*scale)))
    for i in range(len(pts1)-1):
        pygame.draw.line(surface, color, pts1[i], pts1[i+1], 2)
        pygame.draw.line(surface, (0,200,200), pts2[i], pts2[i+1], 2)
        if i % 8 == 0:
            pygame.draw.line(surface, (100,255,100), pts1[i], pts2[i], 1)

angle = 0
while True:
    for e in pygame.event.get():
        if e.type == pygame.QUIT: sys.exit()
    screen.fill((0, 0, 0))
    cx, cy = W//2, H//2
    views = [(cx, cy//2, 0), (cx, cy*3//2, math.pi), (cx//2, cy, math.pi/2), (cx*3//2, cy, -math.pi/2)]
    for vx, vy, offset in views:
        draw_helix(screen, vx, vy, angle + offset, (0, 255, 200), scale=0.5)
    pygame.display.flip()
    angle += 0.02; pygame.time.Clock().tick(60)
