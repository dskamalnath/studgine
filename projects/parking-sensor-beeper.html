Parking Sensor Beeper

An HC-SR04 sensor beeps faster as an object gets closer, just like a car parking assistant. Learn distance-to-frequency mapping and proportional audio feedback design.

HC-SR04 echo → distance (cm) → beep interval

▶ Watch on YouTube

📖 Introduction

The HC-SR04 ultrasonic distance sensor works by emitting a 40 kHz ultrasonic burst from its TRIG pin and measuring how long it takes the echo to return on the ECHO pin. Since sound travels at ~343 m/s (at 20 °C), the formula is: distance (cm) = (echoTime_µs / 2) / 29.1.

This project maps that continuous distance measurement to a beep interval — close objects produce rapid continuous beeps (a solid tone below 10 cm), far objects produce slow single chirps. It's exactly how parking sensors in cars work, making this a commercially realistic embedded systems exercise.

⚙️ How It Works

  1. 1
    Wire HC-SR04 — VCC → 5 V, GND → GND, TRIG → D9, ECHO → D10. The sensor operates at 5 V; its ECHO pin outputs 5 V logic which is fine for standard Uno but use a voltage divider to 3.3 V when using 3.3 V boards.
  2. 2
    Measure distance — Trigger a 10 µs pulse: digitalWrite(TRIG, HIGH); delayMicroseconds(10); digitalWrite(TRIG, LOW);. Then: long duration = pulseIn(ECHO, HIGH); long dist = duration / 58; (the constant 58 combines ÷2 for round-trip and ÷29.1 for cm/µs – rounded to integer-friendly value). Read every 60 ms minimum to allow echo to clear.
  3. 3
    Map distance to beep interval — int beepInterval = map(dist, 5, 80, 50, 1000); Clamp with constrain(). Also scale tone frequency: int freq = map(dist, 5, 80, 2400, 800); — as you approach, the pitch rises and the beep rate increases simultaneously, creating a clear urgency gradient.
  4. 4
    Non-blocking beep output — Use millis(): if elapsed ≥ beepInterval, play tone(BUZZ, freq, 40); and reset timer. If distance < 8 cm, switch to continuous tone: tone(BUZZ, 2500);. If distance > 100 cm, noTone(BUZZ); — no alerts for objects out of parking range.
