Fire-Fighting Robot

Flame sensors guide a wheeled robot autonomously toward a fire source where a small fan extinguishes it — combining motor control, sensor fusion, and priority logic.

📖 Introduction

The Fire-Fighting Robot autonomously searches for flame sources and deploys a fan to extinguish them. Three IR flame sensors (left, centre, right) detect 760–1100 nm infrared light emitted by fire. The robot's L298N motor driver steers toward whichever sensor reports the highest IR intensity — a simple but effective heat-seeking algorithm.

When the centre sensor detects close proximity (<20 cm), the robot stops and activates a DC fan motor to blow out the flame. You'll learn differential drive kinematics, proportional steering, sensor-priority arbitration, and real-time actuator switching.

  • 🎛️
    Arduino Uno — Main controller
  • 🔥
    IR Flame Sensors ×3 — Left, centre, right (analog output)
  • ⚙️
    L298N Dual H-Bridge Motor Driver — Controls 2 DC motors
  • 🚗
    DC Gear Motors ×2 + Wheels — Differential drive chassis
  • 💨
    5 V DC Fan (or pump) — Fire extinguisher mechanism
  • 🔋
    7.4 V Li-Po Battery (2S) — Robot power supply
  • 🤖
    2WD Robot Chassis Kit — Acrylic or aluminium frame

⚙️ How It Works

  1. 1

    Flame Sensor Reading

    Analog flame sensors return ~1023 in darkness and drop toward 0 when detecting fire. Read all three sensors (A0, A1, A2) and detect fire if any reading < threshold (e.g., 300). The sensor with the lowest reading points toward the flame source.

  2. 2

    Steering Algorithm

    If Left < Right and Left < Centre: turn left. If Right < Left and Right < Centre: turn right. If Centre has lowest value: go forward. If all three sensors above threshold: spin slowly to search. This is a simple priority arbitration pattern.

  3. 3

    Extinguish Phase

    When the centre sensor drops below EXTINGUISH_THRESHOLD (≈100) the robot stops all motors and fires the fan relay for 3 seconds. After extinguishing, back up 10 cm and resume searching to check if fire is out.

  4. 4

    Motor Control with L298N

    Set direction via IN1/IN2 (left motor) and IN3/IN4 (right motor). Speed via PWM on ENA/ENB. Use analogWrite(ENA, 180) for forward, lower values for turns. Short the motor enable jumpers if fixed full speed is preferred.

💻 Core Code Snippet

#define FL  A0  // Left flame sensor
#define FC  A1  // Centre flame sensor
#define FR  A2  // Right flame sensor
#define FAN_PIN  8
#define THRESHOLD 300
#define EXTINGUISH 100

void loop() {
  int l = analogRead(FL);
  int c = analogRead(FC);
  int r = analogRead(FR);

  if (c < EXTINGUISH) {
    stopMotors();
    digitalWrite(FAN_PIN, HIGH);
    delay(3000);
    digitalWrite(FAN_PIN, LOW);
    reverse(500); // back up
    return;
  }

  if (l < THRESHOLD || c < THRESHOLD || r < THRESHOLD) {
    if(l < r && l < c)      turnLeft(150);
    else if(r < l && r < c)  turnRight(150);
    else                     moveForward(180);
  } else {
    spinSearch(100); // rotate to find fire
  }
}

void moveForward(int spd) {
  analogWrite(ENA,spd); analogWrite(ENB,spd);
  digitalWrite(IN1,HIGH); digitalWrite(IN2,LOW);
  digitalWrite(IN3,HIGH); digitalWrite(IN4,LOW);
}
