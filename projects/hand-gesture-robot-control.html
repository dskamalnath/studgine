Hand Gesture Robot Control

Control a wheeled robot wirelessly using hand gestures — MediaPipe classifies wrist/finger gestures on Raspberry Pi, sends commands via Bluetooth HC-05 to an Arduino-based robot chassis.

📖 Introduction

This project merges computer vision with wireless robotics: a Raspberry Pi running MediaPipe Hands translates intuitive gestures (open hand = stop, fist = forward, pointing left/right = turn) into serial commands sent via Bluetooth to an Arduino-controlled robot chassis. No joystick or phone app required — just your hand.

The split-architecture design (Pi for CV, Arduino for motor control) is an important embedded pattern: offload computationally expensive tasks to a more capable processor while keeping safety-critical real-time motor control on dedicated microcontrollers.

🧩 Components Required

  • 🧠
    Raspberry Pi 4 (Controller) — Runs MediaPipe + gesture classifier; connects to HC-05 Bluetooth master via UART (/dev/ttyS0)
  • 🤖
    Arduino Uno + L298N (Robot) — Receives single-byte commands ('F','B','L','R','S') via HC-05 slave Bluetooth; drives 2 DC motors
  • 📡
    2× HC-05 Bluetooth Modules — One master on Pi (AT mode: set as master, pair with slave), one slave on Arduino; UART 9600 baud
  • 📷
    USB Webcam or Pi Camera — 30 FPS sufficient; position facing user at waist height for unobstructed hand view
  • 🔋
    2× 18650 Battery Pack (Robot) — 7.4V for motors; separate 5V powerbank for Raspberry Pi; keeps power systems isolated

⚙️ How It Works

  1. 1

    Gesture Classification with MediaPipe

    MediaPipe Hands provides 21 3D landmarks. Engineer geometric features from landmarks: finger extension (tip-to-MCP distances), wrist tilt (y-gradient of landmarks). Define gesture rules: Fist: all 5 fingertips below MCP joints → command 'F' (forward). Open hand: all tips above MCP → 'S' (stop). Point left: only index finger extended, wrist tilted left → 'L'. Point right: index extended, tilted right → 'R'. Thumbs up: thumb extended upward → 'B' (back). Add smoothing: command must be stable for 5 consecutive frames before sending.

  2. 2

    Bluetooth Serial Communication

    On Raspberry Pi: configure HC-05 as master via AT commands (AT+ROLE=1, AT+CMODE=0, AT+BIND=slave_address). In Python: ser = serial.Serial('/dev/ttyS0', 9600). Send command: ser.write(b'F'). On Arduino: char cmd = Serial.read(). Map to motor actions: 'F'→both forward, 'B'→both reverse, 'L'→left motor stop, 'R'→right motor stop, 'S'→all stop. Add watchdog: if no command received for 500ms, auto-stop (safety for Bluetooth dropout).

  3. 3

    Speed Control via Finger Count

    Extend the gesture vocabulary: map finger count (1–4 fingers extended while hand tilts forward) to speed levels 25%, 50%, 75%, 100% PWM. Send speed as a second byte after command byte. Arduino reads two bytes: command + speed. Motor PWM: analogWrite(ENA, speed); analogWrite(ENB, speed). Gradual speed ramp: interpolate between current and target speed over 0.1s to prevent motor current spikes.

  4. 4

    Live Overlay + Debugging

    Draw MediaPipe landmark skeleton on frame with mp.solutions.drawing_utils.draw_landmarks(). Add text overlays: current gesture name, confidence, command sent, Bluetooth status. Record a video of gesture sequences for demo purposes: VideoWriter to mp4. Display on HDMI monitor or stream via Flask + MJPEG for remote viewing on phone browser during robot control.

💻 Core Code Snippets

# Pi side: gesture_controller.py
import cv2, mediapipe as mp, serial, time

ser = serial.Serial('/dev/ttyS0', 9600, timeout=1)
mp_hands = mp.solutions.hands
hands = mp_hands.Hands(max_num_hands=1, min_detection_confidence=0.75)
cap = cv2.VideoCapture(0)
prev_cmd = b'S'; stable = 0; STABLE_THRESH = 5

def get_gesture(lm):
    tips = [8, 12, 16, 20]; mcps = [5, 9, 13, 17]
    extended = sum(lm[t].y < lm[m].y for t,m in zip(tips,mcps))
    tilt = lm[0].x - lm[9].x
    if extended == 0: return b'F'
    if extended == 4: return b'S'
    if extended == 1:
        return b'L' if tilt > 0.1 else (b'R' if tilt < -0.1 else b'F')
    return b'S'

while True:
    ret, frame = cap.read()
    results = hands.process(cv2.cvtColor(frame, cv2.COLOR_BGR2RGB))
    cmd = b'S'
    if results.multi_hand_landmarks:
        lm = results.multi_hand_landmarks[0].landmark
        cmd = get_gesture(lm)
    if cmd == prev_cmd: stable += 1
    else: stable = 0; prev_cmd = cmd
    if stable >= STABLE_THRESH:
        ser.write(cmd); stable = 0
