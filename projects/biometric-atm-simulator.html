Biometric ATM Simulator

Combine fingerprint authentication (AS608), RFID card verification (MFRC522), and PIN entry on a Raspberry Pi 7" touchscreen to simulate a fully secure ATM transaction flow with live account balance display and transaction logging.

📖 Introduction

Modern ATMs use multi-factor authentication: something you have (card), something you know (PIN), and increasingly something you are (biometric). This project replicates all three layers on a Raspberry Pi. The MFRC522 RFID reader verifies a Mifare card UID; the AS608 optical fingerprint module performs 1:N matching against a stored database; and a touchscreen GPIO keypad accepts a 4-digit PIN. SQLite stores accounts with hashed PINs and balance history.

The project demonstrates state-machine design, multi-modal authentication sequencing, and secure data storage — skills directly applicable to embedded fintech systems.

🧩 Components Required

  • 👆
    AS608 Optical Fingerprint Module — UART at 57600 baud; 127 fingerprint capacity; 1:N matching in <0.5s; match threshold <0.1% FAR
  • 💳
    MFRC522 RFID Reader — SPI 3.3V; reads 13.56 MHz Mifare Classic cards; 4-byte or 7-byte UID extracted per card
  • 🖥️
    Raspberry Pi 4 + 7" DSI Touchscreen — tkinter GUI ATM interface; touchscreen keypad for PIN entry; 800×480 display
  • 🗄️
    SQLite Database — Stores accounts (card_uid, fp_id, pin_hash, balance); transaction log table with timestamps

⚙️ How It Works

  1. 1

    Card Authentication (RFID)

    Continuously poll MFRC522 via SPI using MFRC522 library. When a card approaches, read UID: reader.read_id_no_block(). Compare UID against SQLite accounts table. If found, store session card_uid and advance to fingerprint stage. Timeout after 10s with "CARD NOT RECOGNISED" message and reset state machine.

  2. 2

    Fingerprint Verification

    Use pyfingerprint library on UART. Command: f.readImage(); f.convertImage(0x01); result = f.searchTemplate(). Returns (position, accuracy_score). If position matches account's fp_id and score > 50, advance to PIN. If 3 failed attempts, lock account for 60s and log event. Enrolment mode: two readings combined into template stored at AS608 internal flash position.

  3. 3

    PIN Verification + Transaction

    Touchscreen shows 4×3 numeric keypad. Compute SHA-256 of entered PIN: hashlib.sha256(pin.encode()).hexdigest(). Compare against stored hash in SQLite (never store plain PIN). On match: show balance, offer Withdraw/Deposit/Check. Deduct withdrawal amount with UPDATE accounts SET balance=balance-amount WHERE card_uid=?. Insert row into transactions table with timestamp.

  4. 4

    Audit Trail + Security

    All events logged: card scans (success/fail), fingerprint results, PIN attempts, transactions — each with UTC timestamp and session_id. Failed auth attempts increment lockout counter persisted in SQLite. Receipt printed to console or thermal printer (ESC/POS via UART). Session token ephemeral — cleared from memory on logout or 30s inactivity timeout.

💻 Core Code Snippet

import sqlite3, hashlib, serial, spidev, time
from pyfingerprint.pyfingerprint import PyFingerprint

# State machine
STATE = 'IDLE'
session = {}

def check_card(uid):
    db = sqlite3.connect('atm.db')
    r = db.execute("SELECT * FROM accounts WHERE card_uid=?", (uid,)).fetchone()
    db.close()
    return r

def verify_pin(card_uid, pin):
    pin_hash = hashlib.sha256(pin.encode()).hexdigest()
    db = sqlite3.connect('atm.db')
    r = db.execute(
        "SELECT balance FROM accounts WHERE card_uid=? AND pin_hash=?",
        (card_uid, pin_hash)).fetchone()
    db.close()
    return r[0] if r else None

def withdraw(card_uid, amount):
    db = sqlite3.connect('atm.db')
    bal = db.execute("SELECT balance FROM accounts WHERE card_uid=?",
                     (card_uid,)).fetchone()[0]
    if bal >= amount:
        db.execute("UPDATE accounts SET balance=balance-? WHERE card_uid=?",
                   (amount, card_uid))
        db.execute("INSERT INTO transactions VALUES(?,?,?,datetime('now'))",
                   (card_uid, 'WITHDRAW', amount))
        db.commit()
        return True
    db.close()
    return False

# Main loop: poll RFID → fingerprint → PIN
f = PyFingerprint('/dev/ttyS0', 57600, 0xFFFFFFFF, 0x00000000)
while True:
    # RFID poll handled via MFRC522 interrupt callback
    time.sleep(0.1)
