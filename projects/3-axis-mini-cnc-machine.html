📖 Introduction

Computer Numerical Control (CNC) machines are the workhorses of modern manufacturing. This mini CNC project teaches the full stack: mechanical design (linear rails + belt drive + lead screws), stepper motor control (microstepping with TMC2208 Spread Cycle silence), GRBL firmware, G-code generation in CAM software, and PC streaming via Universal-G-Code-Sender. The 300×200×60mm work envelope is suitable for PCB milling (0.2mm traces), wood carving, and thin aluminium engraving.

🧩 Components Required

  • 🔧
    Arduino Mega 2560 + CNC V3 Shield — GRBL 1.1h firmware; 3× axis sockets; spindle/coolant/probe pins; limit switch inputs
  • ⚡
    3× TMC2208 Stepper Drivers — 2A RMS; up to 1/256 microstepping; UART mode for silent operation; no resonance ringing
  • 🔩
    3× NEMA-17 Steppers (1.8°, 2A) — X/Y: belt drive GT2 20T + 8mm pre-loaded V-slot linear rails; Z: M5 lead screw 0.8mm pitch
  • 🌀
    300W DC Spindle Motor + ER11 Collet — 12V 300W; 10,000 RPM; PWM speed control via CNC shield spindle output; ER11 for ⌀0.8–7mm end mills
  • 📏
    Mechanical Frame + Rails — 2040 V-slot aluminium extrusion; polycarbonate bed; 3× endstop microswitches for homing

⚙️ How It Works

  1. 1

    GRBL Firmware Configuration

    Flash GRBL 1.1h to Mega. Configure settings in config.h: $100=80, $101=80, $102=400 (steps/mm for X, Y, Z with GT2 belt + 20T pulley + 1/16 microstepping). Set $110, $111, $112 (max feed rate mm/min). Enable homing: $22=1. Set invert mask for axis directions: $3=7. Validate by jogging each axis manually from UGS (Universal G-Code Sender).

  2. 2

    CAM Toolpath Generation

    Design PCB in KiCad → export Gerber → import into FlatCAM. Configure: isolation routing, 0.2mm V-bit, 0.1mm depth of cut, 600mm/min feed rate, G-code export. For 3D carving: FreeCAD Path workbench or Fusion 360 CAM. Verify G-code in NCViewer before running. Key G-codes: G0 (rapid move), G1 (feed move), G2/G3 (arc), M3/M5 (spindle on/off).

  3. 3

    Workpiece Probing + Z-Zero

    Use GRBL probing: connect alligator clip to PCB copper and bit to probe pin. Command: G38.2 Z-5 F50 — Z axis descends until contact, GRBL records exact Z zero. Compensates for PCB warping with auto-levelling mesh using 16-point probe grid (via OpenPnP or ChiliPeppr). This dramatically improves depth consistency for 0.2mm trace isolation milling.

  4. 4

    Streaming G-Code + Monitoring

    Universal G-Code Sender (UGS Platform) streams G-code line-by-line to GRBL via USB serial. Monitor real-time position, feed override (10–200%), spindle speed override, and emergency stop. For unattended operation, use grbl-stream.py: opens serial, waits for ok response before each line. Implement soft limits ($20=1) to prevent carriage crashes outside work volume.

💻 Core Code Snippet

import serial, time

def grbl_stream(port, gcode_file):
    s = serial.Serial(port, 115200, timeout=1)
    time.sleep(2)  # GRBL init
    s.write(b"\r\n\r\n")
    time.sleep(2); s.flushInput()
    
    with open(gcode_file, 'r') as f:
        lines = f.readlines()
    
    for i, line in enumerate(lines):
        line = line.strip()
        if not line or line.startswith(';'):
            continue
        # Remove inline comments
        line = line[:line.find(';')] if ';' in line else line
        print(f"[{i:04d}] Sending: {line}")
        s.write((line + '\n').encode())
        # Wait for GRBL 'ok'
        while True:
            resp = s.readline().decode().strip()
            if 'ok' in resp: break
            if 'error' in resp:
                print(f"GRBL Error: {resp}")
                s.close(); return
    
    print("Job complete!")
    s.close()
