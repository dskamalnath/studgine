Automated Pet Feeder

A servo opens a food gate on a timed schedule stored in Arduino EEPROM. A rewarding practical build that teaches servo angles, delay scheduling, and persistent storage.

📖 Introduction

This project combines three important concepts: servo motor control (generating 50 Hz PWM pulses to position a rotary actuator), millis()-based scheduling (firing events at set intervals without blocking), and EEPROM storage (retaining the feeding schedule across power cycles so the feeder keeps working even after unplugging).

The physical design uses a servo horn attached to a rotating disc or sliding gate that blocks a chute opening. When the servo rotates to the open position, gravity feeds a portion of kibble into the bowl below. A defined rotation window (e.g. 2 seconds) controls the portion size.

⚙️ How It Works

  1. 1
    Set up servo and schedule — Wire servo signal → D9 (with 1 kΩ in series to protect PWM pin). Servo myServo; myServo.attach(9);. Define feeding times as minutes-since-midnight in an array: int schedule[] = {8*60, 13*60, 19*60};. Write these to EEPROM on first boot or when the user adjusts them via buttons.
  2. 2
    Software clock — Since we don't have a real-time clock module, keep a variable unsigned long elapsedSec = millis() / 1000; and convert to hours/minutes. Or integrate an optional DS3231 RTC for ±2 ppm accuracy. At startup, load schedule integers from EEPROM: int t = EEPROM.read(i * 2) | (EEPROM.read(i*2+1) << 8);
  3. 3
    Trigger the feed cycle — Each loop: compare current minutes to each schedule entry within a ±1-minute window to avoid missing the exact second. When matched and not already fed this minute: myServo.write(90); delay(2000); myServo.write(0);. Set a flag to prevent re-triggering within the same minute.
  4. 4
    Display and manual override — Show the next feeding time on the LCD. Button A triggers an immediate manual feed. Button B cycles through schedule slots on the LCD to let the user verify or adjust them. Write any change back to EEPROM instantly so it survives power loss.
