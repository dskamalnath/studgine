Speed Interrupt Counter

An IR break-beam sensor on a spinning disk triggers hardware interrupts to measure RPM in real time. A hands-on intro to attachInterrupt() and elapsed-time rotation maths.

📖 Introduction

Hardware interrupts let Arduino react to an external signal instantly — unlike polling the pin in loop() which might miss rapid pulses. When the IR receiver detects a break in the beam (the hole in the spinning disk passes), it triggers an interrupt service routine (ISR) in microseconds, incrementing a pulse counter.

By counting pulses over a known time window (typically 1 second), you can calculate RPM: RPM = (pulses / slots) × 60. With a 6-slot encoder disk and 30 pulses/second, that's 300 RPM. This technique is used in automotive ABS sensors, industrial motor drives, and 3D printer extruder encoders.

⚙️ How It Works

  1. 1
    Attach interrupt on sensor output — Wire sensor OUT → D2 (interrupt 0 on Uno). In setup: attachInterrupt(digitalPinToInterrupt(2), countPulse, FALLING);. The ISR is minimal — just increment a counter: void countPulse() { volatile unsigned long pulseCount++; }. The volatile keyword tells the compiler this variable can change outside the normal execution flow.
  2. 2
    Calculate RPM every second — In loop, check millis() - lastCalc >= 1000. When true: noInterrupts(); unsigned long count = pulseCount; pulseCount = 0; interrupts(); — this atomically captures and resets the counter. Then: float rpm = (count / SLOTS) * 60.0; where SLOTS is the number of holes/slots in the encoder disk.
  3. 3
    Display on LCD — lcd.setCursor(0,0); lcd.print("RPM: "); lcd.print((int)rpm); lcd.print(" ");. Display RPS on row 2: lcd.setCursor(0,1); lcd.print(rpm/60.0, 1); lcd.print(" rev/s ");. The trailing spaces overwrite leftover characters as RPM decreases.
  4. 4
    Convert to surface speed — If you know the wheel circumference: float speedMps = (rpm / 60.0) * circumferenceMetres;. Convert to km/h: speedKmh = speedMps * 3.6;. This is exactly what bicycle computers and go-kart speedometers compute using a single magnet and Hall-effect sensor — effectively the same system.
