CubeSat Model with Sensor Suite

A 1U CubeSat (10×10×10cm) model using Raspberry Pi Zero 2W as the on-board computer. LSM9DS1 provides attitude estimation, INA219 monitors simulated solar panel power, BME280 logs thermal telemetry, and an RTL-SDR simulates radio downlink. A Flask ground station web dashboard displays all telemetry live over Wi-Fi — a complete educational spacecraft model.

📖 Introduction

CubeSats are standardised 10cm cube satellites — over 2000 have been launched. This project builds a functional educational model at 1:1 scale that simulates every subsystem: On-Board Computer (Pi Zero 2W), Attitude Determination (LSM9DS1 Mahony filter), Electrical Power System (INA219 solar monitor), Thermal Control (BME280), and Communications (RTL-SDR beacon simulation). The ground station Flask dashboard simulates mission control, receiving telemetry "downlinked" over local Wi-Fi in the same AX.25 packet format used by real amateur CubeSats. Students understand spacecraft systems engineering through a tangible, working model.

🧩 Components Required

  • 🔲
    Raspberry Pi Zero 2W — Quad-core ARM Cortex-A53; 512MB RAM; fits 1U CubeSat frame (65mm board); Wi-Fi onboard for telemetry; runs Raspberry Pi OS Lite; consumes ~1W
  • 📡
    LSM9DS1 9-DOF IMU (ADCS) — 3-axis accel+gyro+mag; SPI or I²C; Mahony filter attitude estimation; reports roll/pitch/yaw; simulates ADCS subsystem star tracker output
  • ☀️
    INA219 + Small Solar Cells (EPS) — 4× 0.5W 5V solar cells on CubeSat faces; INA219 I²C power monitor; measures panel voltage+current; simulates Electrical Power Subsystem telemetry
  • 🌡️
    BME280 (Temperature/Pressure) + RTL-SDR — BME280 monitors board temperature; RTL-SDR USB dongle: pySX127x library simulates LoRa/FSK beacon; FM transmit test mode for RF characterisation demo

⚙️ How It Works

  1. 1

    ADCS: Attitude Determination

    LSM9DS1 via SPI: read accelerometer (roll/pitch from gravity), gyroscope (rate integration), magnetometer (yaw). Mahony complementary filter: ax,ay,az = imu.read_accel(); gx,gy,gz = imu.read_gyro(); mx,my,mz = imu.read_mag(); error = cross(normalised_accel, gravity_ref); integrator += Ki*error; rate_corrected = gyro + Kp*error + integrator; quaternion = integrate(rate_corrected, dt). Convert to Euler: roll = atan2(2*(q0*q1+q2*q3), 1-2*(q1²+q2²)); pitch = asin(2*(q0*q2-q3*q1)). Simulates star tracker attitude output for ADCS loop.

  2. 2

    EPS: Power Monitoring

    Four solar cell panels: each cell generates ~0.5W at full sun. Total simulated bus power: 2W budget for Pi Zero + sensors. INA219 setup: from ina219 import INA219; ina = INA219(0.1, busnum=1); ina.configure(); V = ina.voltage(); I = ina.current(); P = ina.power(). Simulate orbital sun/eclipse: 90min orbit, ~60min sun / ~30min eclipse. In software: in_eclipse = (orbit_pos % 90) > 60; panel_power = P if not in_eclipse else 0. Battery state of charge: charge/discharge model based on current draw vs panel power. Alert on low battery (<20% SoC).

  3. 3

    Telemetry Beacon Simulation

    Build AX.25-format beacon packet (same format as real amateur CubeSats): import struct; callsign = b"STUDGN"; beacon = struct.pack("!6s1s4f4f", callsign, b"\x00", roll, pitch, yaw, panel_power, temperature, battery_soc, orbit_pos). Transmit: publish to local MQTT broker every 10s. Ground station (separate Pi or PC) subscribes. Alternatively, use RTL-SDR in transmit mode (if using HackRF/RTl-SDR-TX) or Si4432 RF module to broadcast actual 433MHz beacon receivable by any RTL-SDR ground station within 100m.

  4. 4

    Flask Ground Station Dashboard

    Flask-SocketIO server receives MQTT telemetry: @socketio.on('get_telem'); def get_telem(): emit('telem', {'roll': roll, 'pitch': pitch, 'power': panel_power, 'temp': temp, 'soc': battery_soc}). Web dashboard with Chart.js live graphs: orbit track (polar plot), attitude cube 3D visualisation (Three.js), power budget bar chart, temperature trend. Mission Timeline: pass schedule, next communication window countdown. Archive: log all telemetry to SQLite; download CSV for analysis. Deploy as a systemd service: auto-starts on Pi boot.

import time, math, json
from flask import Flask, render_template
from flask_socketio import SocketIO, emit
import smbus2

app = Flask(__name__); socketio = SocketIO(app)

# INA219 simplified I²C read (bus voltage register)
def read_ina219_voltage(bus, addr=0x40):
    bus.write_word_data(addr, 0x00, 0x399F)  # Config: 32V, 2A
    time.sleep(0.01)
    raw = bus.read_word_data(addr, 0x02)
    raw = ((raw & 0xFF) << 8) | ((raw >> 8) & 0xFF)
    return (raw >> 3) * 0.004  # 4mV per LSB

def mahony_update(ax, ay, az, gx, gy, gz, q, dt, Kp=2.0, Ki=0.005):
    """Simplified Mahony filter step."""
    q0,q1,q2,q3 = q
    recip_norm = 1.0/math.sqrt(ax**2+ay**2+az**2)
    ax,ay,az = ax*recip_norm, ay*recip_norm, az*recip_norm
    v0 = 2*(q1*q3 - q0*q2); v1 = 2*(q0*q1 + q2*q3); v2 = q0**2 - q1**2 - q2**2 + q3**2
    ex = ay*v2 - az*v1; ey = az*v0 - ax*v2; ez = ax*v1 - ay*v0
    gx += Kp*ex; gy += Kp*ey; gz += Kp*ez
    q0 += 0.5*dt*(-q1*gx - q2*gy - q3*gz); q1 += 0.5*dt*(q0*gx + q2*gz - q3*gy)
    q2 += 0.5*dt*(q0*gy - q1*gz + q3*gx); q3 += 0.5*dt*(q0*gz + q1*gy - q2*gx)
    n = 1.0/math.sqrt(q0**2+q1**2+q2**2+q3**2)
    return [q0*n, q1*n, q2*n, q3*n]

@socketio.on('request_telemetry')
def send_telem(): emit('telemetry', {'status':'nominal', 'roll': 0, 'pitch': 0})

if __name__ == '__main__': socketio.run(app, host='0.0.0.0', port=5000)
