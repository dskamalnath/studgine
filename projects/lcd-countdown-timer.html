📖 Introduction

A countdown timer is a step up from a stopwatch — you need to store a user-configurable target time, implement an increment/decrement button interaction, and count down while displaying a formatted MM:SS string on a 16×2 LCD. It's a practical lesson in state machine design: your sketch cycles through SET → RUNNING → ALARM → RESET states, and each button press triggers a transition.

The I2C LCD backpack reduces the 16-pin parallel LCD to just 2 wires (SDA + SCL), saving breadboard space and freeing digital pins for buttons.

⚙️ How It Works

  1. 1
    I2C LCD wiring — LCD SDA → A4, SCL → A5, VCC → 5 V, GND → GND. Buttons on D4 (Set+), D5 (Start/Stop), D6 (Reset). Buzzer on D8. Use LiquidCrystal_I2C lcd(0x27, 16, 2); — scan for your I2C address with the I2C Scanner sketch if 0x27 doesn't work.
  2. 2
    SET state — Each press of the Set button adds 30 seconds to int setSeconds (cap at 600 = 10 min, wrap to 0). Display on row 1: formatted MM:SS. Row 2 shows "PRESS START". Hold Set for 2 s to subtract 30 s (long-press detection with millis()).
  3. 3
    RUNNING state — Record startMs = millis();. Each loop: long elapsed = (millis() - startMs) / 1000;. int remaining = setSeconds - elapsed;. Format: int m = remaining/60; int s = remaining%60;. When remaining ≤ 0, switch to ALARM state.
  4. 4
    ALARM and RESET — In ALARM: buzz with tone(8, 2000) and flash LCD backlight on/off every 300 ms. lcd.setBacklight(1/0) via the I2C library. Reset button calls noTone(), clears display, and returns to SET state with 0 seconds.
