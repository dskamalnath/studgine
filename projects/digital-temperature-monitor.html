Digital Temperature Monitor

Build a real-time temperature and humidity display using a DHT22 sensor and a 16×2 I²C LCD with an Arduino Uno. Readings update every 2 seconds. A threshold alert triggers a buzzer or LED when temperature exceeds a set limit.

📖 Introduction

The DHT22 (also labeled AM2302) is a calibrated digital sensor that measures temperature (−40 to +80 °C, ±0.5 °C accuracy) and relative humidity (0–100 %, ±2 % accuracy). It communicates over a single-wire protocol at 5 V — no ADC needed. The DHT.h library abstracts the timing, giving you readings with one function call.

A 16×2 LCD with an I²C backpack (PCF8574 expander) reduces the wiring from 6 pins to just SDA and SCL. The LiquidCrystal_I2C library handles the I²C protocol, so the display is a drop-in replacement for parallel wiring with fewer than half the connections.

🤡 Components Required

  • 🧠
    Arduino Uno R3 — DHT22 on D4, I²C on A4 (SDA) and A5 (SCL), alert LED on D8.
  • 🌡
    DHT22 Sensor Module — Temperature & humidity, ±0.5 °C. VCC to 5V, DATA to D4, 10 kΩ pull-up on data pin.
  • 📺
    16×2 LCD with I²C Backpack (PCF8574) — Default I²C address 0x27 or 0x3F. SDA → A4, SCL → A5. Adjustable contrast pot on backpack.
  • 🔘
    10 kΩ resistor — DHT22 data-pin pull-up to VCC. Essential for reliable 1-wire timing.
  • 🔊
    Passive Buzzer or Red LED + 220 Ω — Alert indicator on D8. Trigger above threshold (default 30 °C).
  • 🔧
    Breadboard, jumper wires, USB-A cable — Full Dupont jumper set; keep DHT22 data line under 20 cm for noise-free readings.

⚙ How It Works

  1. 1

    DHT22 Read Cycle

    Every 2 000 ms the sketch calls dht.readTemperature() and dht.readHumidity(). The library sends a start pulse, then reads 40 bits of data (16 humidity + 16 temperature + 8 checksum). If the checksum fails isnan() returns true and the LCD shows “Sensor Err” instead of a bogus value.

  2. 2

    I²C LCD Update

    lcd.setCursor(0, 0) and lcd.print() write the temperature to row 0 and humidity to row 1. The display is cleared only when values change to prevent visible flicker. Calling lcd.clear() every loop causes a noticeable blink — cursor repositioning avoids this.

  3. 3

    Threshold Alert

    If temp > THRESHOLD (default 30 °C), pin D8 goes HIGH. Connected to a passive buzzer, this produces a 1 kHz tone via tone(8, 1000), or an LED simply lights. The threshold constant is defined at the top of the sketch for easy adjustment.

  4. 4

    Serial Monitor Logging

    Every reading is also printed to Serial at 9600 baud in CSV format: millis,temp,humidity. This lets you log data to a spreadsheet via the Arduino IDE serial plotter or a Python script with pyserial for simple data-logging experiments.

💻 Core Code Snippet

#include <DHT.h>
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

#define DHTPIN    4
#define DHTTYPE   DHT22
#define ALERT_PIN 8
#define THRESHOLD 30.0f

DHT dht(DHTPIN, DHTTYPE);
LiquidCrystal_I2C lcd(0x27, 16, 2);

float lastTemp = -999, lastHum = -999;

void setup() {
  Serial.begin(9600);
  dht.begin();
  lcd.init();
  lcd.backlight();
  pinMode(ALERT_PIN, OUTPUT);

  lcd.setCursor(0, 0); lcd.print("Studgine Monitor");
  lcd.setCursor(0, 1); lcd.print("Initialising...");
  delay(1500);
  lcd.clear();
}

void loop() {
  float t = dht.readTemperature();
  float h = dht.readHumidity();

  if (isnan(t) || isnan(h)) {
    lcd.setCursor(0, 0); lcd.print("Sensor Error!   ");
    lcd.setCursor(0, 1); lcd.print("Check wiring    ");
    delay(2000);
    return;
  }

  if (t != lastTemp || h != lastHum) {
    lcd.setCursor(0, 0);
    lcd.print("Temp: ");
    lcd.print(t, 1);
    lcd.print((char)223);          // degree symbol
    lcd.print("C   ");

    lcd.setCursor(0, 1);
    lcd.print("Hum:  ");
    lcd.print(h, 1);
    lcd.print("%    ");

    lastTemp = t; lastHum = h;
  }

  digitalWrite(ALERT_PIN, t > THRESHOLD ? HIGH : LOW);
  Serial.print(millis()); Serial.print(',');
  Serial.print(t);        Serial.print(',');
  Serial.println(h);

  delay(2000);
}
