IoT Flood Sensor Alert

Monitor a sump pit, basement, or outdoor channel with float switches and an HC-SR04 ultrasonic sensor. ESP8266 fires Telegram/IFTTT alerts on rising water and logs levels to ThingSpeak.

📖 Introduction

Flooding causes billions of dollars of damage annually — and the frustrating part is that most of it happens slowly enough that an early warning could make all the difference. An IoT flood monitor provides exactly that: a low-cost, always-on sentinel that pushes a phone alert the moment water starts rising.

This project uses three approaches in concert: float switches (simple, reliable NO/NC contacts) at multiple heights for coarse threshold alerts; an HC-SR04 ultrasonic sensor for continuous distance measurement (± 3 mm resolution); and an ESP8266 for connectivity. Water-level data is logged to ThingSpeak every 5 minutes for trend analysis, and Telegram notifications fire on threshold crossings with an escalating message tone (warning > alert > critical).

🧩 Components Required

  • 📡
    ESP8266 NodeMCU — Wi-Fi client for Telegram + ThingSpeak cloud logging
  • 📏
    HC-SR04 Ultrasonic Sensor — Measures water surface distance; mount above channel/pit
  • 🔵
    3× Float Switches (NO type) — Low / medium / high level thresholds on vertical cable
  • 🔔
    Passive Buzzer + Red/Yellow/Green LEDs — Local audible + visual alert independent of Wi-Fi
  • 🔋
    Li-Po 2000 mAh + TP4056 + Solar Panel — Outdoor solar-powered for remote deployment
  • 📦
    IP65 Waterproof Enclosure — Protects electronics near water; cable glands for sensors

⚙️ How It Works

  1. 1

    Ultrasonic Distance Measurement

    Trigger HC-SR04 with a 10 µs pulse on TRIG pin. Measure ECHO pulse duration with pulseIn(ECHO_PIN, HIGH, 30000). Distance (cm) = duration × 0.0343 / 2. Subtract from tank height to get water depth. Average 5 readings every 2 s to eliminate false readings from turbulence. Temperature compensation (optional): speed = 331 + 0.6 × temp_celsius.

  2. 2

    Float Switch Threshold Logic

    Pull float switch pins HIGH via 10 kΩ pull-up resistors. When water lifts the float: pin reads LOW. Three levels: WARN (SW1 at 30 cm depth), ALERT (SW2 at 60 cm), CRITICAL (SW3 at 90 cm). Float switches provide reliable backup if ultrasonic sensor fails (e.g., from wave interference). Implement software debounce: require stable LOW for 500 ms before triggering.

  3. 3

    Send Tiered Telegram Alerts

    Maintain lastAlertLevel state. Only send a new alert when the level increases (not on every sampling loop). Format message with severity emoji, depth reading, timestamp, and location name. Use WiFiClientSecure + HTTPS GET to api.telegram.org/bot endpoint. Add alert cooldown of 15 minutes per level to prevent spam flooding.

  4. 4

    ThingSpeak Cloud Logging

    Every 5 minutes, POST to api.thingspeak.com/update with API key and field values: field1=depth_cm&field2=sw1&field3=sw2&field4=sw3. ThingSpeak auto-charts the data — set up alert analysis with ThingHTTP+React to get secondary email alerts. Use client.stop() after each HTTP POST to free heap — ESP8266 has only 80 KB RAM.

💻 Core Code Snippet

#include <ESP8266WiFi.h>
#include <WiFiClientSecure.h>

#define TRIG_PIN D6
#define ECHO_PIN D7
#define SW1 D1   // low level
#define SW2 D2   // medium level
#define SW3 D3   // high level (critical)
#define TANK_HEIGHT_CM 100

float measureDepth() {
  digitalWrite(TRIG_PIN, LOW); delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH); delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);
  long dur = pulseIn(ECHO_PIN, HIGH, 30000);
  float dist = dur * 0.0343 / 2.0;
  return TANK_HEIGHT_CM - dist;  // water depth
}

void sendTelegram(String msg) {
  WiFiClientSecure client; client.setInsecure();
  if (client.connect("api.telegram.org", 443)) {
    String url = "/botTOKEN/sendMessage?chat_id=CHATID&text=" + msg;
    client.print("GET " + url + " HTTP/1.1\r\nHost: api.telegram.org\r\nConnection: close\r\n\r\n");
    delay(300); client.stop();
  }
}

int lastAlertLevel = 0;
void loop() {
  float depth = measureDepth();
  int alert = 0;
  if(!digitalRead(SW3)) alert = 3;
  else if(!digitalRead(SW2)) alert = 2;
  else if(!digitalRead(SW1)) alert = 1;

  if(alert > lastAlertLevel) {
    String emojis[] = {"","⚠️ WARN","🔴 ALERT","🚨 CRITICAL"};
    sendTelegram(emojis[alert] + "+Flood+Level:+" + String(depth,1) + "cm");
    lastAlertLevel = alert;
  }
  if(alert == 0) lastAlertLevel = 0;
  delay(2000);
}
